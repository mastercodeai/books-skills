# 任天堂的体验设计：创造不知不觉打动人心的体验

> 作者：玉树真一郎（前任天堂员工，参与Wii开发）  
> 核心命题：任何人都能创造打动人心的体验

---

## 核心框架：三种体验设计

本书将体验设计分为三种类型，构成完整的设计体系：

```
直觉设计（基础）→ 惊喜设计（持续）→ 故事设计（意义）
  假设→尝试→高兴     误解→尝试→惊讶     吸引→成长→意志
```

**体验→感情→记忆**：这是驱动人类行为的根本链条。能打动人心的体验会被转化为长期记忆，而记忆构成了人生的意义。

## 第1章 直觉设计——人为什么会不由自主地行动

### 核心案例：《超级马里奥兄弟》

游戏最重要的规则不是"打败酷霸王"或"收集金币"，而是**"向右前进"**——一个玩家从开头画面中凭直觉就能感受到、却无法用语言表达的规则。

### 直觉设计的结构

1. **假设**：自发建立"要××吗？"的假设（不知道是否正确）
2. **尝试**：自发决定"试试××"并行动（不知道是否正确）
3. **高兴**：自发确认"××是正确的"（第一次确信）

在马里奥出场到酷栗宝登场的几秒内，玩家内心就完成了这个循环。关键：**通过自己的体验学到的东西，一辈子都无法否定**——就像骑自行车一样。

### 核心原理：示能（Affordance）与能指（Signifiers）

- **示能**：看到某物时脑中浮现"要做××吗？"——环境赋予的意义
- **能指**：为传达示能而特别处理的信息（马里奥的位置、山、草、云）
- 超级马里奥开头画面**牺牲了"有趣"的信息**，集中传达"向右前进"的示能
- 设计师最大的考验：**放弃漂亮、放弃有趣，集中精力告诉用户该做什么**

### 直觉设计的连接要点

1. **在一定时间内用直觉设计来填补**（直到用户意识到"有趣"）
2. **每个直觉设计都要简短**（假设带来的不安不超过10秒）
3. **提高愉悦体验的概率**——关键原则：**简单、容易**

### 关键设计手法

- **初始效应**：在体验开始时注意力和学习效率最高，把需要学习的内容集中在初期（4种道具集中在第一关）
- **酷栗宝的设计**：在开发最后阶段创造的最简单敌人，横着走让玩家正面看到恐怖表情，确认"向右前进"是正确的
- **十字键设计**：控制器与游戏画面融为一体，共同传达示能

### 真正的"贴近用户"

**"了解"比"好、正确"更重要。** 优先让用户直观地了解产品，而不是先传达产品有多好。

> 人为什么要玩游戏？——不是因为游戏本身有趣，而是因为自身的直觉体验有趣。

## 第2章 惊喜设计——人为什么会不由自主地着迷

### 核心案例：《勇者斗恶龙》系列

直觉设计的致命缺点：**连续的直觉设计会让玩家产生疲劳和厌倦**。"假设→尝试"中的不安反复出现，加上心理饱和（相同刺激重复多次反应变弱），体验必须被打破。

### "惊喜刺激"的本质

表面上是无聊的段子，实际上是精心设计的体验：

- **登场时机**：在玩家疲劳厌倦达到顶峰时（学完所有指令后、打败第一个强敌后）
- **功能**：消除疲劳和厌倦，让玩家继续沉浸
- **本质**：出乎意料的体验——大脑无法预测未来时会激活学习动力

### 惊喜设计的结构

1. **误解**：自发做出错误假设（坚信假设正确）
2. **尝试**：自发尝试并行动（坚信尝试正确）
3. **惊讶**：自发发现"××是错误的"，大吃一惊

### 两种"坚信"的战略

1. **对前提的坚信**："这个游戏是××" → 故意背叛（如严肃游戏突然插入荒诞内容）
2. **对日常的坚信**："禁忌不会出现" → 让禁忌登场

### 10种禁忌主题

**积极主题（本能追求）**：
- 性、饮食、得失、认可

**消极主题（本能回避）**：
- 肮脏、暴力、混乱、死亡

**复杂主题**：
- **侥幸与偶然**：概率设计（赌场、会心一击），让用户"下注并祈祷"
- **个人隐私**：暴露用户内心世界（取名、选择结婚对象），"这个体验能体现出个性吗？"

### 惊喜设计的操作要点

- **花时间构建误解**：让玩家慢慢形成错误认知
- **设计暴露误解的剧情**：两种坚信同时落空
- **简化版**：不颠覆前提，只打破对日常的坚信（用禁忌主题即可）

> "游戏不是生活必需品，所以需要产生惊喜。"——岩田聪

## 第3章 故事设计——人为什么会不由自主地想叙述

### 核心问题：游戏有意义吗？

玩游戏本质上是浪费时间。但游戏至今存在，说明体验中一定隐藏着意义。

### 故事 = 叙事 = 故事内容 + 故事叙述

- **故事内容**："发生了什么"（情节）
- **故事叙述**："如何表达"（手段）
- 游戏通过**"玩游戏的体验"**来叙述故事——人类历史上新颖的叙事方式

### 故事设计三步

#### 第1步：吸引

利用大脑的**叙述本能**（大脑是讲故事的器官），让玩家自己成为解说员：

- **环境故事**：分散信息碎片，让玩家自行拼凑"发生了什么"
- **节奏和对比**：影像（被动/信息多）→ 探索（半主动）→ 战斗（主动/信息少）构成波浪
  - 消除疲劳厌倦 + 让未来预测变得简单容易
- **伏笔**：隐藏真正含义，埋下暗示，利用时间差让人意识到真正含义
  - "那个场景的那个东西，原来是这个意思！"——让人忍不住向别人叙述的快感

#### 第2步：成长

游戏的意义在于**让玩家成长**（而非主人公成长）。

**三个成长主题：**

1. **收集和重复** {空缺和整体 → 收集和重复 → 成长}
   - 人发现空缺就想填补（蔡格尼克效应）
   - 节奏 = 时间轴上均匀分布的空缺，消除节奏上的空缺让人感受到时间
   - 俄罗斯方块：不留停顿，让方块持续掉落，保持紧张感

2. **选择和斟酌** {风险和回报 → 选择和斟酌 → 成长}
   - 低风险低回报 vs 高风险高回报，玩家自行选择
   - 玩家调整难度 = 游戏给每个玩家最适合的挑战
   - **反馈**：对玩家行为做出反应（游戏最基本的结构 = 交互媒体）
   - "失败是你的错"——让玩家产生"我做得好/我做错了"的自我意识

3. **逆转和共鸣** {麻烦的同行者 → 逆转和共鸣 → 成长}
   - 同行者故意给玩家制造麻烦
   - **共鸣三条件**：对主人公感兴趣 + 相信"对方和自己有相同想法" + 非憎恨的情感
   - 镜像神经元：强烈打动主人公的心 = 强烈打动玩家的心
   - 同行者的危机 → 克服憎恨 → 真正的共鸣和成长

#### 第3步：意志

- **生命交换**：用自己的双手决定珍视之物的命运
- **未知体验**：在从未经历的状况下做决定
- **包含解释余地**：保留未明确部分，引导玩家拥有"自己怎么想"的意志

### 英雄之旅与"回到起点"

故事在起点结束——**为让玩家意识到自己的成长**。通过与经历故事前的自己比较，玩家才能切实感受到变化。

> 故事的使命是让接收者成长。正因为如此，英雄之旅的最后一步是"回家"。

## 第4章 体验设计的本质——体验→感情→记忆

### 记忆的分类

- **长时记忆**：陈述性记忆（语义记忆 + 情景记忆）与非陈述性记忆
- **语义记忆**：语言意义本身（"1492年哥伦布发现美洲"）
- **情景记忆（体验记忆）**：具体的5W1H（"在社会课上学过……"）
- **形成路径**：体验 → 情景记忆 → 语义记忆（长期保存）

### 核心法则

**情景记忆能否长期保存，取决于体验者的感情是否强烈。**

> "体验→感情→记忆"这一流程，经常推动着我们的人生向前发展。

你现在记得的，应该是强烈震撼、激发你情感的体验。以确实激发个人感情的体验为基础，就能设计出打动无数人心灵的体验。

### 跨学科性

体验设计跨越心理学、认知科学、脑神经科学、行为经济学、设计、文学、哲学等无数学科领域，是一个跨学科研究领域。

## 第5章 实践篇：体验设计的应用

### 判断指标

1. **问题令人难以理解** → 运用直觉设计
2. **问题让人疲劳和厌倦** → 运用惊喜设计
3. **问题没有价值** → 运用故事设计

### 应用1：思考/策划

- 问题根源：等待创意降临只会产生不安 → 疲劳厌倦 → 逃避思考
- **惊喜设计**：暴露个人隐私（禁忌主题），让自己惊讶兴奋
- **直觉设计**：将思考碎片放在眼前，从共同点中发现重要事情
- **故事设计**：讲述"失去重要东西陷入危机"的故事，把策划等同于寻找自己的幸福

### 应用2：讨论/引导

- **打破"必须说好点子"的坚信**：讨论"没用的策划"而非"好的策划"
- **建立团队认同**：让成员发现"我们是这样的团队"（直觉设计的假设→尝试→高兴）
- **伏笔手法**：从过去发言中找出深层含义，让成员成为英雄

### 应用3：传达/演示

- **用连接词预告下一张幻灯片**（避免"接下来"这个无法预想的连接词）
- **定期插入禁忌主题/沉默**（打破"演示者是说话者"的前提）
- **最后再展示开始的幻灯片**——让听众感受到自己的成长

### 应用4：设计/产品设计

- **优先考虑第一次使用的用户**（初始效应，人生只有一次的最高效学习机会）
- **产品是配角，用户的人生才是主角**
- **刻意让体验停止的设计**：不使用惊喜设计的回归日常情节
- **提供作弊选项**：让用户自由选择，越自由越能感受成长

### 应用5：培养/管理（育儿启示）

- 孩子不收拾玩具 → 不是因为懒，而是**不知道放到哪里**（直觉设计问题）
- 孩子不想刷牙 → **感觉不到效果、感到厌倦**（惊喜设计问题：故意用牙刷柄刷牙）
- 孩子不想听读故事 → **爸爸已经知道内容了，真无聊**（故事设计问题：一起体验未知）
- **教的一方和被教的一方一起成长**

## 设计师的心法

1. **比有趣更重要的是让人了解**——了解了自然觉得有趣
2. **简单、容易是绝对条件**——复杂谁都能做，简单才是真正的难点
3. **考虑用户的心情，而非功能**——"玩家的心情如何"是体验设计的核心问题
4. **用户体验的顺序决定优先级**："了解"→"好、正确"
5. **产品是配角**——不抢夺用户的人生，让用户自由选择使用或不使用
