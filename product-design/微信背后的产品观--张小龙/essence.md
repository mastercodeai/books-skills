# 微信背后的产品观 —— 张小龙 读书精华

> 产品是技术和艺术的结合。——乔布斯

---

## 一、用户篇：用户是人

### 1. 产品经理的定位
- **产品经理是站在上帝身边的人**——创造一个虚拟世界的创造者
- 产品经理应该像上帝一样建立一个系统、制定规则，让群体在系统中**自我演化**
- 关键词：了解用户、了解群体心理、让人性驱动产品

### 2. 人性的六大特性

| 特性 | 产品启示 |
|------|----------|
| **人是环境的反应器** | 营造"环境"来决定用户行为，朋友圈拍照（当下环境触发）vs Qzone相册（电脑上传）本质不同 |
| **人是懒惰的** | 语音查找联系人等功能，降低一切操作门槛 |
| **时尚是驱动力** | "不用微信就落伍了"——产品不能太工具化，需要引起用户连锁反应 |
| **人是没有耐心的** | 如果不能让用户一分钟爱上你的产品，就不会再来第二次；不要试图教育用户 |
| **人是不爱学习的** | "马桶阅读"理论：给用户的内容如果马桶上时间看不完，就太多了 |
| **群体是"乌合之众"** | 群体智商必然低于个体；不能通过单个用户调研推导出千人整体行为 |

### 3. 核心用户观
- **重视"草根"用户群**：80%用户是草根，他们渴望存在感和自我价值
- **从日常体验中发现本质**：三星Note流行=时尚+实用性（长指甲打字舒适）
- **人性化就是以己推人**：靠自己的感觉做决定
- **需求是满足人们的贪嗔痴**：贪婪、嫉妒恨、痴迷——了解人性的多面性
- **让不上微博的用户来用你的产品**：真正的用户不是你周围的人
- **伟大的产品满足情感需求**，不仅仅是功利上的好处

## 二、需求篇：需求来自对用户的了解

### 1. 四个"不"
> 需求不来自调研；需求不来自分析；需求不来自讨论；需求不来自竞争对手。

### 2. 需求决策原则
- **对于新点子，99%的情况下否定是对的**
- **不要用户说什么就做什么**：用户反馈只是帮助你了解想法，解决方案是归纳抽象的过程
- **不要听从产品经理的需求**：产品经理往往比普通人更不了解大众心理
- **不要从同类产品里找需求**：照搬无法深刻理解需求
- **每天花一小时看普通用户在微博上谈论如何用你的产品**——感受用户潮流

### 3. 需求的真正来源

| 来源 | 案例 |
|------|------|
| **对群体效应的好奇** | 漂流瓶、附近的人、摇一摇——做了之后才知道会发生什么 |
| **自身的需要** | 蓝牙耳机、Web微信（连接键盘而非PC版）、待办置顶 |
| **对生活潮流的理解** | 每个时代有每个时代的产品（QQ→微信） |
| **优先满足自己的需求** | 不做"已读"功能（保护隐私的个人癖好代表了群体需求） |
| **对信息流的理解** | 微信=Push中心，所有信息推过来而非用户去找 |
| **对未来系统的理解** | 二维码普及、连接现实与手机 |

### 4. 需求背后的心理诉求
- **漂流瓶**：不是为了交友，而是满足倾诉和好奇的欲望→默认语音（更迅速直接，不假思索）
- **附近的人**：人天生对周围环境好奇→不是为了交友，是好奇心驱动
- **摇一摇**：满足人对"随机性"的好奇→既具心理驱动又有群体效应
- **朋友圈**：人需要在朋友的关注里存在→私密、舒适、无压力
- **"爽"胜过功能**：用户说"好玩"就对了，"省钱"没有传播力

### 5. 方法论
- **只抓主场景，不做全功能**：朋友圈主推照片（简单），纯文字入口隐藏
- **最后一刻才决定新版本特性**：互联网产品发展太快，每天都在想有什么好玩的
- **世界是新的**：忘记过去的数据甚至经验，洞察当前和未来趋势
- **对未来的理解比当前的需要更重要**

## 三、设计篇：进化而非规划

### 1. 核心设计哲学
- **产品是进化出来的，而非规划出来的**
- **有DNA的产品才会进化**：DNA是产品的价值观和认知
- **把用户分为高中低端是不道德的**：好的产品应该所有人都喜欢用

### 2. 设计方法论

| 原则 | 说明 |
|------|------|
| **先做产品结构，后做功能细节** | 先把"骨骼"梳理清楚，枝叶藏得再深也没关系 |
| **功能模块有机联系** | 朋友圈+个人相册替代多头像需求 |
| **设计就是分类** | 分得不好→产品不好；微信保证两年内只有4个Tab |
| **抽象方能化繁为简** | 100个需求→10个需求→1个需求；订阅平台用一个账号体系 |
| **越简单的分类越易被接受** | 过多变化引用户不适应 |
| **找到需求背后的本质需求** | 分组需求→本质只需屏蔽父母和上级 |

### 3. 设计原则
- **面向场景设计，而非功能列表**：语音自动播放只适合独处和开车，多数场景不合适
- **面向场景，才能取舍**：不做在线状态（手机永远在线）、不做PC版
- **让功能存在无形之中**：商家会员只有扫码后才出现
- **区别手机和PC**：手机是手的延伸、触觉的延伸
- **不过度设计**：附近的人做到极简，不做多余修饰
- **宁愿损失功能也不损失体验**：不加编辑按钮避免误操作
- **面向未来的场景做设计**：服务器将更稳定→不做"已送达"
- **操作的响应速度永远是第一体验**：简单+快捷=简捷

### 4. 进化法则
- **如果特性不能让人兴奋，不如不做**：好奇心是产品经理的驱动力
- **保留变化**：不要一步到位，1.0发布后从反馈获取2.0计划
- **如果没有自然增长，就不必推广**：KPI只是好产品的副产品
- **保持粗放，保持笨拙**：没想好就不要先做
- **如果解决方案太复杂，一定是问题错了**
- **让用户推动用户**：展示自己头像→自然触发设置，而非强制引导
- **极简方能不被超越**：摇一摇做到极简，竞品无法超越

### 5. 创新观
- **把用户体验做到极致就是创新**：微信"认识"耳朵（从Siri获得启发）
- **你的价值观决定产品特性**：滤镜该不该做？"真正的生活状态不应该有滤镜"
- **抛弃不人性的创新**：贝索斯——"你要不计一切代价展示聪明，还是选择善良？"
- **避免战略行为替代真实需求**："打通""整合""拉动"往往来自战略任务而非用户需求
- **从人出发，改变商业模式**：朋友影响朋友的品牌传播模式
- **没有设计，只有解决问题**
- **创新≠凭空发明**：用已有知识以新的方式做事；跟随用户而非跟随业界
- **做产品可以做"类型"而非"实例"**：一劳永逸

## 四、气质篇：做有灵魂的产品

### 1. 产品要有气质
- **改变旧世界的意愿**：微信立项前就期望用Kik替代短信、Google Voice替代电话、FaceTime替代视频
- **有自己的理念和愿景**："微信是一个生活方式"——不是"一种"，是"一个"
- **让技术产品传达人文意识**：QQ邮箱登录页放马丁·路德·金、海子的诗
- **微信启动画面**：能让用户有不同的感觉就对了；人类的孤独感是永恒主题

### 2. 产品人格
- **你没有竞争对手**：重要的是你想做什么，不是他们在做什么
- **是作品，而不是产品**：作品渴望完美，KPI是副产品
- **工匠，而不是设计师**：每天优化十个细节，每个细节都是一个完整的产品
- **做有灵魂的产品**：像人一样——结构是骨骼、功能是肌肉、气质来自价值观、反应要敏捷、文案是谈吐
- **精神分裂的产品**=来自不同想法的妥协拼凑；集体决策会导致平庸
- **保持对产品的坚持甚至独断才能使产品不分裂**

### 3. 气质表达
- **文字反映气质**：清晰易懂、不讨好用户、不抬高自己、用"你"不用"您"
- **不用的字词**：不用"吧"、不用"哦"、不用勉强用户的句式、不用"成功"
- **未来感的设计**：关键词——未来、科技、抽象、简约
- **新功能的罗列是坚决不可接受的**：欢迎页用来传达想法，不是罗列功能
- **不听摇滚的程序员不是好的产品经理**
- **即使做广告也要非常人性化**：引用用户的话比自创更生动

## 五、UI篇：条理清晰最重要

### 1. UI核心原则
- **UI最重要的是条理清晰**，不是绚丽
- **UI围绕特性，而非特性围绕UI**
- **禁止UI设计师使用竞争对手的产品**（产品经理可以体验并总结）

### 2. 具体做法
- 能用系统控件就用系统控件
- 保持一致：一种字体、一种控件、一种背景
- **每个界面只有一个主题**
- 每个对话框有明显默认按钮（蓝色完成按钮）
- **对最常用操作精益求精**
- **需要文字解释的功能不是好体验**——尽量不用Tips
- **隐藏数字**：进度条不需要精确到小数点后两位
- **探索手机上的新交互**：更多传感器、更多可能
- **于细微处用心**：下拉空白处展示微信LOGO
- **锻炼审美能力**：观察日常物品的设计历程
