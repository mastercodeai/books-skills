# SKILL.md — 上瘾模型 (Hook Model) 产品设计技能

## 概述

基于 Nir Eyal 的 Hook Model，掌握如何设计让用户形成使用习惯的产品。核心是一个四阶段循环：触发 → 行动 → 多变的酬赏 → 投入。该模型强调习惯养成是长期用户留存的核心引擎，而非短期增长黑客。

## 何时使用

- 设计新产品功能时
- 提升用户留存和参与度时
- 分析竞品为什么让人"上瘾"时
- 评估产品是否存在习惯养成潜力时
- 审视产品设计的道德边界时

## 场景A：习惯养成评估

**适用**：新功能/新产品概念阶段，判断是否有"上瘾"潜力。

```
评估清单：
□ 1. 是否存在高频内部触发？（用户每天/每周至少感受1次的负面情绪）
□ 2. 目标行为能否在1分钟内完成？（行动门槛足够低）
□ 3. 酬赏是否有多变性？（用户无法完全预测下次体验）
□ 4. 用户投入是否自然积累价值？（数据/关系/技能随使用增长）
□ 5. 投入是否自动加载下一次触发？（闭环是否存在）

判定：
  5/5 ✅ → 强习惯潜力，进入详细设计
  3-4/5 → 有潜力但需补短板，聚焦最弱环节
  ≤2/5 → 习惯路径弱，考虑其他留存策略（如工具价值/网络效应）
```

## 场景B：触发设计

**适用**：拉新阶段或用户回访率低。

```
外部触发优先级（从持久到临时）：
1. 自主型触发（最持久）→ 应用图标、订阅通知、邮件摘要
2. 人际型触发（最强传播）→ 口碑推荐、社交分享、@提及
3. 回馈型触发（补充）→ 媒体报道、应用商店推荐、KOL背书
4. 付费型触发（仅用于拉新）→ 广告投放、渠道合作

设计流程：
Step 1：列出用户旅程中每个"离开点"
Step 2：在每个离开点设计一个触发拉回机制
Step 3：确保触发与内部情绪关联（不是"来用产品"，而是"你还记得那个焦虑吗"）
Step 4：将外部触发逐步转化为内部触发

关键检查：
□ 触发是否在用户最需要的时刻出现？（而非产品需要的时刻）
□ 触发频率是否避免"狼来了"效应？（过多通知 → 关闭 → 永久流失）
□ 是否有退出机制？（用户可控制触发频率）
```

## 场景C：酬赏系统设计

**适用**：用户来了但留不住，缺少让人反复回来的"钩子"。

```
三种酬赏类型（通常组合使用）：

社交酬赏：来自他人的认同、互动、归属感
  → 点赞、评论、排行榜、社区认可、私信

猎物酬赏：对资源/信息的追逐
  → 信息流、搜索结果、新内容发现、限量资源

自我酬赏：操控感、成就感、终结感
  → 进度条、等级提升、清空待办、通关、连续打卡
```

**设计要点**：
- 酬赏必须与用户内部触发匹配（解决的"痒"一致）
- 保持"无穷多变性"（用户生成内容 > 固定内容）
- 保障用户自主权，避免强制行为引发逆反心理
- 不确定性是关键：让用户不知道下一次会得到什么

```
酬赏健康度检查：
□ 用户是否主动向朋友推荐酬赏体验？（口碑信号）
□ 使用30天后，酬赏新鲜感是否衰减？（多变性测试）
□ 酬赏是否创造了真实的用户价值？（非纯操控）
□ 酬赏是否与投入形成正反馈循环？
```

## 场景D：投入钩子设计

**适用**：用户活跃但不沉淀，切换成本低。

```
投入时机：在用户享受酬赏之后提出，而非之前

投入设计四原则：
1. 分解为小任务，从最小投入开始（如：先点赞，再评论，再发帖）
2. 每次投入必须能储存价值（内容/数据/关注者/信誉/技能）
3. 储存的价值必须提高下次使用的体验（更多好友 → 更好的信息流）
4. 投入必须加载下一个触发（发布内容 → 收到互动通知）

投入形式示例：
- 关注/收藏/点赞 → 储存偏好数据 → 算法推荐更精准
- 填写资料/完善信息 → 增加沉没成本 → 个性化体验
- 发布内容 → 加载互动通知（下一个触发）→ 社交酬赏
- 学习技能 → 提高迁移成本 → 竞品难以替代
- 邀请好友 → 扩大社交图谱 → 社交酬赏增强
```

```
投入闭环检查：
□ 用户的投入是否随时间增值？（而非被遗忘）
□ 投入是否创造了"离开就亏"的感觉？（沉没成本）
□ 投入是否自然加载了下一个触发？（闭环验证）
□ 新用户是否能在首次使用中完成最小投入？
```

## 核心工作流

### 1. 定义内部触发 — "用户为什么需要你？"

```
行动步骤：
1. 用5问法挖掘用户核心情感（问5个"为什么"直到触及情绪）
2. 确定目标用户的最高频负面情绪触发（无聊/孤独/焦虑/困惑）
3. 用一句话描述：每当用户感到 [内部触发]，他就会 [预期习惯行为]
```

**检验标准**：你能否用一句话说清产品解决的"痒"是什么？

### 2. 设计触发系统

（详见场景B）

### 3. 简化行动路径

```
福格行为模型：B = M × A × T

优先级：先简化能力(A)，再强化动机(M)

简化六要素检查清单：
□ 时间：能否在30秒内完成核心动作？
□ 金钱：是否有免费体验入口？
□ 体力：点击次数能否再减少？
□ 脑力：是否需要用户思考或学习？
□ 社会偏差：周围人是否接受这个行为？
□ 非常规性：是否需要用户改变已有习惯？
```

**关键技巧**：
- 利用启发法（稀缺效应、锚定效应、目标渐近效应）降低感知难度
- 用已有的用户习惯搭便车（如Facebook登录、一键分享）

### 4. 设计多变的酬赏

（详见场景C）

### 5. 设计投入环节

（详见场景D）

### 6. 习惯测试（迭代优化）

```
第1步：确定习惯用户
  → 定义"忠实用户的合理使用频率"
  → 基准：至少5%用户达到习惯标准

第2步：分析习惯路径
  → 从忠实用户中提取共同行为模式
  → 例：Twitter发现关注≥30人是关键行为

第3步：改进产品
  → 推动新用户朝习惯路径前进
  → 评估结果，持续迭代
```

## 失败模式诊断

当Hook循环不生效时，用以下框架定位根因：

### 症状 → 根因 → 修正

| 症状 | 可能根因 | 诊断问题 | 修正方向 |
|------|----------|----------|----------|
| 用户不知道产品存在 | 外部触发缺失 | 触发覆盖率是否>目标用户50%？ | 增加人际型/回馈型触发 |
| 用户知道但不尝试 | 行动门槛太高 | 首次核心动作是否<30秒？ | 简化注册/Onboarding |
| 用户试了但不回来 | 无内部触发建立 | 产品是否在用户"痒"的时刻出现？ | 强化触发-情绪关联 |
| 用户回来但很快流失 | 酬赏多变性不足 | 用户能否预测每次体验？ | 引入UGC/随机性/社交元素 |
| 用户活跃但不投入 | 投入环节缺失 | 用户投入是否自动加载下次触发？ | 设计投入-触发闭环 |
| 用户投入但逐渐沉默 | 价值衰减 | 用户的投入是否随时间增值？ | 让历史投入改善未来体验 |
| 用户增长停滞 | 触发传播断裂 | 用户是否有动机分享？ | 设计人际型触发激励 |

### 诊断流程

```
Step 1：定位Hook循环断点
  → 用习惯测试数据，看用户在哪一步流失最多

Step 2：对照上表，确定根因类别
  → 触发问题？行动问题？酬赏问题？投入问题？

Step 3：设计最小修正实验
  → 每次只改一个变量，A/B测试验证

Step 4：量化效果
  → 核心指标：习惯用户占比（目标≥5%）、DAU/MAU比、回访率曲线
```

## 操控模式自检

在设计习惯养成功能时，回答两个问题：

```
问题1：我自己会使用这个产品吗？
问题2：该产品会帮助用户大大提高生活质量吗？

结果：
  ✅ + ✅ = 健康习惯推广者（最可能成功）
  ❌ + ✅ = 兜售商（警惕脱离用户）
  ✅ + ❌ = 娱乐用户者（产品缺乏持久力）
  ❌ + ❌ = 经销商（不道德，不推荐）
```

**底线**：如果回答时需要为自己寻找正当理由，立刻停手。

**进阶伦理检查**：
- 产品是否利用了用户无法控制的弱点？（赌博心理、FOMO）
- 停止使用后，用户的生活质量是否下降？
- 是否有"健康使用"的提醒机制？
- 目标用户是否有足够的认知能力做出知情选择？

## 反模式与陷阱

| 反模式 | 问题 | 为什么常见 | 修正 |
|--------|------|-----------|------|
| 过度依赖付费触发 | 成本不可持续，用户无习惯 | 短期见效快 | 转向自主型/人际型触发 |
| 只做游戏化外壳 | 徽章/积分无实质吸引力 | 照搬成功案例表皮 | 酬赏必须匹配用户真实需求 |
| 强制用户行为 | 引发逆反心理 | 产品团队KPI导向 | 保障自主权，让用户觉得是自己的选择 |
| 有限多变性 | 新鲜感消退快 | 初期设计容易忽略 | 引入用户生成内容，创造无穷多变性 |
| 跳过投入直接要求行动 | 用户缺乏持续动机 | 急于求转化 | 先给酬赏，再要投入 |
| 忽视道德边界 | 可能伤害用户 | 增长压力 | 使用操控模式自检，识别并帮助上瘾用户 |
| 触发疲劳 | 通知过多 → 关闭 → 永久流失 | 短期指标好看 | 尊重用户触发频率偏好 |
| 虚假多变性 | 用随机数伪装变化，用户很快看穿 | 实现成本低 | 多变性必须源于真实差异（如UGC） |
| 投入无回报 | 用户积累数据但体验不变 | 投入与体验脱节 | 每次投入必须可见地改善体验 |
| 照搬Hook不看品类 | 低频品类强行做习惯循环 | 教条化应用 | 先评估品类是否有习惯养成空间 |

## 关键检查点

在Hook循环设计的每个阶段，用以下检查点确认质量：

### 设计前检查点
- [ ] 是否已完成用户访谈/调研，识别出真实内部触发？
- [ ] 是否定义了"习惯用户"的标准（频率+行为）？
- [ ] 是否评估过品类是否适合习惯养成？

### 设计中检查点
- [ ] 触发是否与内部情绪强关联？
- [ ] 核心行动是否<30秒可完成？
- [ ] 酬赏是否包含至少两种类型？
- [ ] 酬赏多变性是否可持续（非一次性惊喜）？
- [ ] 投入是否自然加载下一次触发？
- [ ] 是否设计了从外部触发到内部触发的转化路径？

### 设计后检查点
- [ ] 是否完成操控模式自检？
- [ ] 是否有数据埋点追踪Hook各阶段转化率？
- [ ] 是否有A/B测试计划验证核心假设？
- [ ] 是否设计了"健康使用"机制（如使用时长提醒）？

## 实战模板

### 用户习惯画布

```
产品名称：_______________

1. 内部触发：用户最频繁的负面情绪/需求是什么？
   → _______________

2. 外部触发：用户首次如何发现产品？
   → _______________

3. 预期行动：用户使用产品的第一个动作是什么？
   → _______________

4. 酬赏类型：产品提供哪种多变的酬赏？
   □ 社交酬赏  □ 猎物酬赏  □ 自我酬赏
   具体机制：_______________

5. 投入环节：用户投入了什么？储存了什么价值？
   → _______________

6. 下一个触发：投入如何加载下一次使用的触发？
   → _______________

7. 操控自检：
   □ 我自己会用吗？  □ 能提高用户生活质量吗？

8. 习惯用户定义：
   频率：每周___次
   关键行为：_______________
   基准线：___%用户达到
```
