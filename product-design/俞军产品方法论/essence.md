# 俞军产品方法论 — 精华提炼

> 俞军：百度前产品副总裁、滴滴前产品高级副总裁。中国互联网第一批顶级产品经理，主导了百度搜索、贴吧、知道等产品的从0到1。

---

## 核心公式

### 用户价值公式
```
用户价值 = 新体验 - 旧体验 - 替换成本
```
- **提升用户价值三条路径**：新体验最大化、旧体验最小化、替换成本最小化
- 该公式本质是主观价值判断工具，非精确计算公式
- 可推广到分析产品价值链上的所有角色：客户、政府、员工、股东、供应商等

### 相对价格公式（人间第一定律）
```
相对价格 = (直接成本 + 交易成本) / 效用组合
```
- 其他条件不变时，相对价格降低，需求量上升
- 产品经理可通过追加效用组合、改变效用组合或降低各类交易成本来降低相对价格，实现用户增长

### 产品经理价值公式
```
产品经理的价值 = 经验等级 × 平台匹配度 × 智慧等级
```

## 一、用户模型

### 用户的本质
- **用户不是自然人，是需求的集合**
- 同一个自然人可以是成百上千种产品的成百上千个用户
- 注册用户≠真正用户，关键看用户在有需求时是否使用该产品

### 用户五大属性
1. **异质性**：用户千差万别，不能统一为单一画像
2. **情境性**：有情境才有用户，同一用户在不同情境下行为不同
3. **可塑性**：用户偏好和认知会随外界信息刺激变化
4. **自利性**：用户追求个人总效用最大化
5. **有限理性**：用户判断经常出错，只能做到有限理性

### 用户行为机制
```
情境刺激 → 感知/解读/选择集合/模拟推演/价值判断 → 期望效用 → 行为 → 结果 → 经验反馈
```
- 用户约等于"偏好和认知的函数"
- 偏好同时受先天遗传和后天社会经验影响

### 系统1与系统2
- **系统1**（快思考）：本能、习惯、直觉，自动启动，节省资源，受认知偏见支配
- **系统2**（慢思考）：逻辑、理性，被动启动，消耗资源
- 人默认使用系统1；做重要决策时必须启用系统2

## 二、交易模型

### 核心理念：产品即交易
- 用户的每一次主动行为都是与世界的一次交易
- 交易≠只有金钱交换，时间、体力、心理成本、数据等都是支付
- **世界上不存在等价交换**——双方都主观判断自己受益才会完成交易

### 好产品的三个属性
1. **有效用**（对用户有使用价值）
2. **有收益**（企业能获利）
3. **可持续**（产业链各方利益分配公平合理）

### 交易模型设计精髓
- **利从何来**：如何创造利益
- **利往何去**：如何分配利益
- 本质是多边关系平衡的利益创造和利益分配模式

### 关键经济学概念（产品经理及格线）
1. **效用**：欲望的满足程度，主观评价
2. **成本**：直接成本、交易成本、机会成本
3. **边际**：边际效用递减、边际成本、边际利润
4. **供需定律**：其他条件不变时，价格下降则需求上升

### 交易成本
- **本质**：用户付出但企业没收到的成本 + 企业付出但用户没收到的成本
- **来源**：信息不对称、信息不确定、信息不完全、信息有成本
- **分类**：搜寻成本、度量成本、议价成本、决策成本、实施成本、保障成本
- **降低交易成本是产品设计的核心工作**

### 损失厌恶
- 同量损失的负效用约为同量收益正效用的2.5倍
- 用户换回的物品价值要大于支付代价的2.5倍，交易才能成立

### 激励相容
- 使行为人追求个人利益的行为正好与企业实现整体价值最大化的目标相吻合
- 让对方干得好比干得不好时获得更多收入

## 三、效用深度理解

### 效用的多样性
效用包括但不限于：货币、时间、身体需求、心理需求、信念、情绪、认知等。同一产品可叠加无数效用。

### 产品效用满足的四类需求
| 类型 | 说明 | 策略 |
|------|------|------|
| 底线需求 | 不能低于，一票否决 | 追求bad case最小化 |
| 越多越好 | 用户愿为更多效用多支付 | 做到边际收入=边际成本 |
| 越多越好 | 用户愿为更多效用多支付 | 做到边际收入=边际成本 |
| 惊喜 | 超过预期或参照系 | 惠而不费，性价比极高 |

### 产品定义
```
产品 = 一组约束条件下的一个效用组合
```

## 四、企业理解

### 企业的本质
1. 发现市场获利机会（洞察、试错、偶然性）
2. 生产效率高于市场

### 詹森生产函数
```
Q = f(L, K, M, C; T; r)
```
- r（外部规则）> T（技术）> C（内部规则）> L, K, M
- 外部规则和技术给定后，长期来说内部规则影响最大

### 企业做产品的四方面产出
1. 财务绩效
2. **认知**（know-how）
3. **团队**（磨合过的高效团队）
4. **无形资产**（信任积累，降低未来交易成本）

即使产品失败，后三项依然增长。

## 五、决策方法论

### 理性决策三要素（按重要性排序）
1. **理性的信念**：与真实世界一致的信念（对自我认知的认知）
2. **理性的目标**：约束条件下的价值最大化
3. **理性的行动**：给定目标下寻找最优解决方案

### 三种决策方式
| 方式 | 适用场景 | 特点 |
|------|----------|------|
| 数据决策 | 适合AB测试的在线产品 | 成本最低，但有局限性 |
| 逻辑决策 | 需要沟通共识的场景 | 成本较高，需对方有认知基础 |
| 主观判断决策 | 信息不足、环境不确定 | 最难，需强大的用户模型和交易模型 |

### 九个权衡视角
1. 对自我认知的认知
2. 对给定目标的批判性思考
3. 参照系
4. 成本（直接成本、交易成本、机会成本、风险成本）
5. 不确定性决策
6. 概率（风险决策）
7. 非货币价值（跨效用决策）
8. 外部性
9. 时间性（跨期决策）

### 决策落地
- **议程设置**：影响决策者去想什么，而非想的结果
- 推动能力 = 为他人创造价值的能力

## 六、产品经理选拔与成长

### ABC分类法（天赋潜力）
- **A类**：有深度思考能力或超常同理心（极少见，<10%）
- **B类**：逻辑清晰且有产品心（绝大多数）
- **C类**：不适合做产品经理（逻辑不清晰、同理心弱、功利心过重）

### 面试产品经理四个维度
1. **批判性思维**
2. **同理心**
3. **用户模型**（对做过产品的用户模型掌握程度）
4. **产品技能熟练度**

### 潜力判断三项
1. **自省**（第一潜力）：发现和承认自己短处的能力
2. 学习意愿和学习能力
3. 高成就动机

### 产品经理成长的关键标准

#### 视角一：用户模型 → 交易模型
- 用户模型：预判产品迭代后用户行为变化的准确率
- 交易模型：理解多边利益的价值判断和行为习惯

#### 视角二：科学方法 + 人文关怀 + 实践精神

#### 视角三：批判性思维

### 前五年成长建议
1. 从大量产品迭代中验证和思考
2. 加入一个有产品心的小团队

### 六个能力发展方向
1. **从0到1**（做一次有价值，边际递减）
2. **做深做透**
3. **做宽做杂**
4. **大用户量**（必须经历）
5. **商业闭环**（用户价值+商业价值一体化）
6. **团队建设**

### 五年后发展方向
1. 专业路线（权衡决策，交易模型）
2. 管理路线（识人用人，分优先级）
3. 业务路线（知识经验累加）
4. 做新产品/创业（高风险高收益）
5. 出海

### 产品经理的核心竞争力
```
输出决策的质量 = 逻辑 + 同理心 + 经验 + 异见 + 数据
```

## 七、关键认知

### 俞军产品经理十二条（2005年）
1. 产品经理首先是用户
2. 站在用户角度看待问题
3. 用户体验是一个完整的过程
4. 追求效果，不做没用的东西
5. 发现需求，而不是创造需求
6. 决定不做什么往往比决定做什么更重要
7. 用户是很难被教育的，要迎合用户而不是改变用户
8. 关注最大多数用户，在关键点上超越竞争对手
9. 给用户稳定的体验预期
10. 如果不确定该怎么做，就先看别人是怎么做的
11. 把用户当作傻瓜，不要让用户思考和选择
12. 不要给用户不想要的东西

### 产品工作本质
- 产品工作属于强实践性的社会科学
- 只可归纳，不可演绎——不能保证预测具体个人行为
- 经济学帮助理解宏观群体行为，心理学帮助理解微观个体行为

### 创造价值的五大路径
1. 劳动
2. 分工
3. 交易
4. 新技术
5. 制度

### 关于创新
- 创新 = 将新要素引入现有生产方式或生活方式，创造巨大新价值
- 新要素包括：新技术、新人群、新渠道、新方法、新工具等
- 人人都能创新，事无大小都能创新
