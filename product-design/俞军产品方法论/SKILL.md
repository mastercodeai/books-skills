## 核心心智模型

### 1. 用户价值公式
做任何产品决策前，用此公式评估：
- **用户价值 = 新体验 - 旧体验 - 替换成本**
- 提升路径：①新体验最大化 ②旧体验最小化 ③替换成本最小化
- 旧体验最小化 = 选择旧体验最差的被替代品用户（完美新用户）或创造局部垄断

### 2. 产品即交易
- 每次用户行为都是一次交易（不只是金钱）
- 交易成立条件：效用 - 成本 > 0，且优于其他选择
- 好产品三要素：有效用、有收益、可持续

### 3. 人间第一定律（相对价格）
```
相对价格 = (直接成本 + 交易成本) / 效用组合
```
降低相对价格 = 增加效用组合 或 降低各类交易成本

## 分析工具

### 用户分析框架
1. 用户不是自然人，是**需求的集合**
2. 用户五属性：异质性、情境性、可塑性、自利性、有限理性
3. 研究维度：生物的人、社会的人、认知的人、情境的人
4. 行为机制：情境刺激 → 偏好/认知函数 → 期望效用 → 行为 → 经验反馈

### 效用分析框架
将产品拆解为用户视角的各种效用，分析每个属性迭代对效用的影响：
- 底线需求（不能低于，一票否决）→ bad case最小化
- 够用就好（边际递减）→ 打磨到边际回报率最划算的点
- 越多越好（多多益善）→ 做到边际收入=边际成本
- 惊喜（超预期）→ 惠而不费的增量创造

### 交易成本分析
识别并降低以下六类交易成本来源：
1. **信息不对称**：一方有别人没有的信息
2. **信息不确定**：行动后才揭示相关信息
3. **信息不完全**：相关信息不为当事人所知
4. **信息有成本**：认知、发现、收集、处理、传播、鉴别都有成本
5. **有限理性**：身心智能限制
6. **投机主义**：欺诈和不信任

降低交易成本的常用方法：标准化、线上化、用户评价体系、品牌口碑、算法推荐

### 决策九维权衡视角
每次重要决策前，逐项检查：
1. ✅ 我的认知完备度打几分？可能有哪些认知偏差？
2. ✅ 我的目标理性吗？关键变量是否已变化？
3. ✅ 参照系是什么？对用户/各方的参照系有偏误吗？
4. ✅ 直接成本、交易成本、机会成本、风险成本分别是什么？
5. ✅ 不确定性有多大？是否敬畏了世界的不确定性？
6. ✅ 发生概率有多大？预期效用=(收益-成本)×概率
7. ✅ 是否考虑了非货币价值的跨效用权衡？
8. ✅ 是否考虑了外部性（对第三方的影响）？
9. ✅ 时间尺度是什么？短期正确≠长期正确

## 失败模式清单

做产品决策时逐条自检，命中任何一条都需要暂停并修正：

### 认知类失败
| # | 失败模式 | 症状 | 修正动作 |
|---|----------|------|----------|
| F1 | **自我偏好投射** | "我觉得用户需要…" | 回到用户五属性，要求数据/用户样本支撑 |
| F2 | **幸存者偏差** | 只看成功案例得出方法论 | 主动收集失败案例和反面数据 |
| F3 | **锚定效应** | 被第一个数据/方案锁定 | 强制列出至少3个独立参照系 |
| F4 | **沉没成本执念** | "已经投入这么多不能放弃" | 只看未来增量价值，忽略已投入成本 |
| F5 | **确认偏误** | 只收集支持自己观点的证据 | 指定一个人专门唱反调（红队） |

### 决策类失败
| # | 失败模式 | 症状 | 修正动作 |
|---|----------|------|----------|
| F6 | **指标替代** | 优化了指标而非真实用户价值 | 问"这个指标提升后用户真的更满意了吗？" |
| F7 | **单一维度优化** | 只看增长/留存/收入之一 | 用效用分析框架四层（底线→够用→越多→惊喜）全检 |
| F8 | **忽略替换成本** | 只算新体验好，不算用户迁移代价 | 强制计算替换成本三要素（认知+操作+关系迁移） |
| F9 | **短期正确长期错误** | 季度OKR驱动牺牲长期价值 | 九维第9项：强制回答"3年后这个决策还正确吗？" |
| F10 | **忽略外部性** | 对平台生态/第三方造成损害 | 画出所有利益相关方，逐一检查正/负外部性 |

### 执行类失败
| # | 失败模式 | 症状 | 修正动作 |
|---|----------|------|----------|
| F11 | **功能堆砌** | 做加法容易做减法难 | 每个新功能必须回答"替换哪个旧功能或砍掉什么" |
| F12 | **过度设计** | MVP变成全功能产品 | 只做验证核心假设所需的最小功能集 |
| F13 | **AB实验污染** | 实验组互相影响导致结论无效 | 检查分流独立性、时间效应、网络效应 |

## 反模式（Anti-patterns）

### 产品决策反模式

**"竞品做了我们也要做"**
- 问题：忽略了用户差异、情境差异、替换成本差异
- 正确做法：回到用户价值公式，用自己的用户数据评估，竞品仅作信息参考

**"用户调研说要就做"**
- 问题：用户说什么≠用户会做什么（有限理性 + 情境性）
- 正确做法：观察行为 > 听取言语，用AB实验验证真实行为变化

**"数据好就是好功能"**
- 问题：短期指标提升可能掩盖长期价值损耗（指标替代）
- 正确做法：区分虚荣指标和因果指标，结合用户满意度/留存曲线综合判断

**"先上线再迭代"**
- 问题：上线即有用户成本，坏体验的替换成本是负向的（用户留下心理阴影）
- 正确做法：底线需求必须达标才能上线，"够用就好"级别才能快速迭代

### 招聘反模式

**"大厂背景 = 能力强"**
- 问题：平台匹配度换场景后大幅降低，标签最大化效应
- 正确做法：追问具体项目中的个人贡献、决策过程、认知变化

**"经验丰富 = 值得高薪"**
- 问题：5年后经验边际递减，智慧等级才是跨领域保值项
- 正确做法：评估自省能力、学习意愿、成就动机（潜力三要素）

**"面试表现好 = 实际能力强"**
- 问题：面试技巧可以训练，但同理心和批判性思维难以伪装
- 正确做法：深入追问过去项目中的挫折和失败，观察反思深度

### 团队管理反模式

**"用KPI驱动一切"**
- 问题：指标替代，团队优化指标而非用户价值
- 正确做法：KPI只用于底线管理，创新和体验用用户反馈驱动

**"产品经理是需求传话筒"**
- 问题：沦为各方需求的收集转发器，丧失判断力
- 正确做法：产品经理必须独立做优先级判断，基于用户价值公式排序

## AB实验 / MVP检查清单

启动任何AB实验或MVP前，逐项确认：

### 实验设计阶段
- [ ] **假设明确**：一句话说清"如果我们做X，指标Y会变化Z，因为…"
- [ ] **用户价值公式验算**：新体验 - 旧体验 - 替换成本 > 0 的证据是什么？
- [ ] **分流独立性**：实验组之间是否互相影响？是否有网络效应？
- [ ] **最小功能集**：只保留验证假设所需的最少功能，其余全砍
- [ ] **底线需求检查**：实验版本的底线需求（一票否决项）是否达标？

### 实验执行阶段
- [ ] **观测周期**：是否有足够时间观察长期效应（不只是首日/首周）？
- [ ] **指标体系**：核心指标 + 护栏指标（防止单一优化损害其他维度）
- [ ] **用户分群**：不同用户群体（新/老、高频/低频）效果是否一致？

### 实验复盘阶段
- [ ] **因果 vs 相关**：指标变化真的是功能导致的吗？排除了外部因素吗？
- [ ] **用户真实价值**：指标提升是否对应用户满意度/留存的实际改善？
- [ ] **推广决策**：实验结论是否适用于全量用户？边界条件是什么？

## 产品经理选拔标准

### 面试方法
1. **深入追问做过的项目**：为什么做→怎么做→遇到什么问题→发现什么→认知有什么变化
2. **问一个他不熟悉的问题**（你熟悉而他不熟悉）：看思维框架和拆解方法
3. 对疑似A类人，进一步交流性格、志趣、三观

### 四维专业能力评估（五分制）
| 维度 | 说明 | 评估要点 |
|------|------|----------|
| 批判性思维 | 理性反思、求证 | 逻辑链是否严谨 |
| 同理心 | 站在别人角度思考 | 能否准确察觉判断他人感受 |
| 用户模型 | 预判用户行为的能力 | 对人性常识和大众心理的认知 |
| 产品技能 | 基础扎实程度 | 流程、方法、工具 |

### ABC分类
- **A类**：深度思考能力 或 超常同理心（极少见）
- **B类**：逻辑清晰 + 产品心（绝大多数合格产品经理）
- **C类**：逻辑不清晰/同理心弱/功利心过重 → 不适合

### 潜力三要素
1. **自省**（第一潜力）：发现和承认自己短处的能力
2. **学习意愿和能力**
3. **高成就动机**

### 简历判断技巧
- **标签最大化效应**：候选人用标签美化自己，实际可能是标签范围内最弱的
- **针对性**：产品经理应为不同岗位重新设计简历，否则同理心/勤奋不合格

## 产品经理成长路径

### 前五年：基础积累期
1. 找有大量迭代机会的产品岗位
2. 加入有产品心的小团队
3. 积累用户样本量，建立用户模型

### 六个能力发展方向
1. 从0到1（做一次即可，边际递减）
2. 做深做透（细分场景深入研究）
3. 做宽做杂（中小公司优势，全局观）
4. **大用户量**（必须经历，理解用户异质性）
5. **商业闭环**（用户价值+商业价值一体化）
6. 团队建设（识人用人）

### 五年后：四种方向
| 方向 | 适合人群 | 核心能力 |
|------|----------|----------|
| 专业路线 | A类天赋 | 权衡决策、交易模型 |
| 管理路线 | 专业天赋一般 | 识人用人、分优先级 |
| 业务路线 | 求稳 | 领域经验持续积累 |
| 新产品/创业 | 机会成本低 | 冒险和试错 |

## 招聘决策框架

### 产品经理价值公式
```
价值 = 经验等级 × 平台匹配度 × 智慧等级
```
- 经验等级：5年后边际递减
- 平台匹配度：换公司/业务后大幅降低
- 智慧等级：换领域后仍能保持个人价值

### 空降高阶产品经理
```
成功率天然低（熟悉成本+磨合成本高）
成功率高的场景：复制产品、新产品、灾后重建、巨大新要素
```

### 应届生招聘优先级
- **当前好 vs 未来好**：潜力优先
- 带你的人 > 业务 > 平台
- 选人才质量长期正向上升的企业

## 经典案例思维模式

### 动态调价 vs 排队模式
- 问题本质：满足更快打到车的诉求（预期+准确性+速度）
- 权衡：公平 vs 效率，受社会文化约束
- 方法：MVP实验验证 → 迭代扩展

### 司乘纠纷判责
- 从"跷跷板"价值观（偏袒一方）→ "弹簧"价值观（判责不清，平台承担）
- 上游问题上游解决
- 用户认为不公平就是不公平（以大众评价为参照系）

### 拼车价值分配
- 供大于求 → 价值向乘客倾斜（降价+拼成奖励）
- 供小于求 → 价值向司机倾斜（共乘计价模式）
- 核心：在约束条件下求相对最优解，随供需变化动态调整
