# SKILL.md — 李想产品实战方法论

## 适用场景

产品经理、创业者、CEO制定产品战略、打造团队、从0到1做产品、从1到10复制爆品。

## 核心心智模型

### 模型1：产品方法论双轮驱动

```
用户价值轮：关注用户价值 → 超越用户需求 → 打造产品力
组织成长轮：把组织当产品 → 对抗惰性/惯性/无知 → 支撑产品迭代
```

### 模型2：品牌三问（做产品前必答）

| 问题 | 示例（理想汽车） | 示例（汽车之家） |
|------|-----------------|-----------------|
| 我们是谁？ | 智能电动车品牌 | 汽车信息网站 |
| 我们为谁服务？ | 家庭用户 | 汽车消费者 |
| 创造什么价值？ | 移动的家，幸福的家 | 帮助看车、买车、用车 |

**验证**：说给10个人听，想象一致=清晰

### 模型3：产品标准三维度

```
安全感（底线）→ 不会劝退用户 → 突破行业惯性
价值感（竞争力）→ 用户选择你 → 聚焦目标群，完整体验
向往感（金线）→ 口碑传播 → 唤起向往场景
```
排序：安全 > 舒适 > 便捷

### 模型4：产品价值公式

```
产品价值 = 产品力 ÷ 产品价格 × 产品质量
```
用途：在给定价格区间，精确计算如何最大化用户可感知价值

### 模型5：体验设计决策框架

```
是否高频使用？ ──否──→ 谨慎投入
       │是
竞争对手是否已做好？ ──是──→ 需要差异化突破
       │否
投入是否务实（不带来更多麻烦）？ ──否──→ 放弃
       │是
→ 执行
```

## 实操清单

### 定位选择

- [ ] 在价格等硬指标外，增加用户人生阶段等新维度重新看市场
- [ ] 验证：差异化定位 + 高速增长市场 + 用户需求未被充分满足
- [ ] 优先找正在上升的细分市场，而非眼前规模最大的
- [ ] 集中力量只做一件事，不要"两条腿走路"

### 品牌定义

- [ ] 产品动工前回答品牌三问
- [ ] 答案清晰无歧义（10人测试法）
- [ ] 在产品设计、宣传触达中让用户感知品牌定位
- [ ] 从"我们擅长什么"出发，不只看"想成为谁"

### 文化建设

- [ ] 等团队运行1-3年后再提炼文化，不提前强行规定
- [ ] 团队共创：所有人写成功原因+真实案例 → 投票选前三
- [ ] 在具体事件上厘清灰度（做选择题不做对错题）
- [ ] CEO持续讲、不厌其烦讲，新员工培训必讲

### 用户需求挖掘

- [ ] 少做一对一访谈，多做多人共创（分组讨论避免意见领袖主导）
- [ ] 共创会：先问场景回忆 → 问感性期待 → 不问具体方案 → 不现场收敛
- [ ] 少问多观察：进入用户生活/购买现场
- [ ] 需求验证：回到真实场景，执行足够彻底

### 技术/方案选择

- [ ] 技术服务于目标用户的产品价值，不为新技术而新技术
- [ ] 非此即彼时考虑组合方案，取长补短
- [ ] 看失败经验要分析关键问题，不看表面结论
- [ ] 成本效率最优 ≠ 成本最低

### 定价策略

- [ ] 精确做数学题：给定价格区间对标竞品，计算产品力最大化
- [ ] 不低于用户心理价位（身份象征类产品尤其注意）
- [ ] 评估企业长期经营需求（毛利率目标）
- [ ] 不能逃避竞争

### 产品标准与取舍

- [ ] 功能取舍看高频，不看大众/小众
- [ ] 原子级对标竞品配置
- [ ] 务实原则：新功能是否带来更多麻烦？
- [ ] 紧盯用户价值，不刷存在感（NPS系统指导研发权重）

### 复盘

- [ ] 先总结成功经验，再讨论改进
- [ ] 对照PEA计划复盘（6要素：问题、目标、方案、风险、资源、指标）
- [ ] 每个问题制定To-Do落到具体人
- [ ] CEO/一号位必须参与，专人主持，按时嵌入工作流

### 团队打造

- [ ] 统一度量衡：统一语言和工具，工具选与团队文化匹配的
- [ ] 嵌入流程僵化执行：不允许"聪明人"跳过步骤
- [ ] 培养成年人：《高效能人士七个习惯》纳入考核+强制举真实案例
- [ ] 不惩罚新错误，只惩罚重复犯错

### 流程建设

- [ ] 还原用户价值流：从用户角度倒推，剔除不创造价值的环节
- [ ] 区分路（流程）和车（专业模型）：路有问题修路，车有问题修车
- [ ] 定义主流程+支线流程，关键节点对齐互锁
- [ ] 重量级PDT团队做拉通，不是个人催进度

### 节奏把控

- [ ] 从0到1：目标=长板做到天花板（3%市占率）；资源=集中办大事
- [ ] 从1到10：目标=基于未来淘汰赛必要性（非惯性外推）；资源=宁可过度准备
- [ ] 从1到10不是简单复制，打法完全不同

### 门店/销售

- [ ] 低频大额消费品必须线下体验
- [ ] 快速开店（有无门店市占率差8倍）：少SKU + 培训考试 + 数字化
- [ ] 总部收（数据管理）+ 一线放（自主分配资源），培养经营者意识
- [ ] 按周复盘，及时调整资源组合

### 利润管控

- [ ] 原子级拆分：从最小单元推算人效→店效→整体利润
- [ ] 提炼最佳实践→分享→全员逼近→半年刷新
- [ ] NPS系统记录功能感知权重，指导研发取舍
- [ ] 毛利率20%目标（保证10%研发投入）

## 反模式（常见陷阱）

| 陷阱 | 正确做法 |
|------|---------|
| 更大众的定位带来更多销量 | 差异化+高速增长+未满足需求 |
| 文化是缺什么补什么 | 从团队成功经验中提炼 |
| 用户调研就是一对一访谈 | 多人共创+实地观察 |
| 用户说要什么就做什么 | 追溯底层动机，场景验证 |
| 追新技术热点 | 技术服务于目标用户价值 |
| 复盘找人背锅 | 先总结成功经验，对照计划复盘 |
| 从1到10就是复制 | 基于必要性重新定目标，全面备战 |
| 定价越低越好 | 心理价位+企业经营需求+产品力数学题 |
| 功能越多越好 | 高频+务实+不刷存在感 |
| 惩罚所有错误 | 不惩罚新错误，只惩罚重复犯错 |

## 失败模式

| 失败模式 | 信号 | 应对 |
|---------|------|------|
| **定位漂移** | 品牌三问答案半年变3次；团队对"我们是谁"说法不一 | 锁定三问，贴墙上，每次决策回溯 |
| **需求幻觉** | 用户调研满意度高但产品没人买 | 回到真实场景，观察行为而非听表态 |
| **流程空转** | 流程文档齐全但没人照做；"聪明人"总跳步 | 僵化执行 + 管理层以身作则 + 跳步即问责 |
| **文化口号化** | 价值观墙上好看但决策时没人提 | 绑定具体事件厘清灰度，CEO反复讲案例 |
| **规模惯性** | 从1到10照搬从0到1打法；资源线性外推 | 重新做必要性分析，打法重新设计 |
| **复盘走过场** | 复盘会变成甩锅会或庆功会 | 专人主持 + 先成功后改进 + PEA对照 + 落人落时间 |
| **定价恐惧** | 不敢定高价；越低越好的直觉 | 做数学题，对标竞品，测算心理价位 |
| **技术自嗨** | 团队兴奋于新技术但用户无感 | 回到目标用户的产品价值，问"这对用户意味着什么" |

## 反模式深层解析

### 产品决策类
- **伪差异化**：改了外观/包装就算差异化 → 差异化必须在用户核心价值维度上
- **功能堆砌**：每个需求都说"很重要" → 用高频度和务实原则做减法
- **数据迷信**：完全依赖数据决策 → 数据是参考，用户价值才是锚点
- **完美主义**：产品永远不够好不上线 → 从0到1关键是长板够长，不是短板够短

### 组织管理类
- **流程崇拜**：流程万能 → 流程是路，专业是车，分清问题在哪
- **文化速成**：刚创业就定文化 → 文化是成功经验的提炼，不是提前规定
- **惩罚导向**：所有错误都罚 → 只罚重复犯错，鼓励探索新错误
- **度量衡混乱**：每人用自己的语言和工具 → 先统一再开工

### 用户洞察类
- **意见领袖绑架**：共创会被一两个人主导 → 分组讨论，去领袖化
- **方案前置**：调研时直接问方案 → 先问场景和感受，方案是产品经理的事
- **办公室调研**：坐在公司分析用户 → 必须进入用户真实生活场景

### 规模化类
- **线性外推**：把从0到1的经验直接放大 → 从1到10是全新命题
- **资源不足就凑合**：从1到10资源不够也硬上 → 宁可过度准备，不做半吊子
- **开店=铺量**：门店越多越好 → 少SKU + 培训 + 数字化 + 按周复盘

## 关键检查点

### 产品立项检查点
1. 品牌三问是否清晰（10人测试通过）？
2. 是否处于高速增长的细分市场？
3. 用户需求是否未被充分满足？
4. 有没有"两条腿走路"的问题？

### 产品开发检查点
1. 每个功能是否通过"高频度"筛选？
2. 每个功能是否通过"务实原则"（不带来更多麻烦）？
3. 技术选择是否服务于目标用户价值？
4. 定价数学题做完了吗？心理价位确认了吗？

### 上市前检查点
1. 安感感维度是否突破行业惯性？
2. 价值感是否让用户在对比中选你？
3. 向往感场景是否能驱动口碑传播？
4. 门店/销售渠道准备好了吗？

### 团队健康检查点
1. 团队是否使用统一语言和工具？
2. 流程是否在僵化执行（没有"聪明人"跳步）？
3. 文化是从成功经验提炼的，还是口号？
4. 复盘是否产出具体To-Do并落到人？

### 规模化检查点
1. 是否基于必要性重新设目标（非惯性外推）？
2. 资源是否过度准备？
3. 最佳实践是否提炼并全员逼近？
4. 原子级利润模型是否建立？

## 一句话总结

> 做产品没有秘密，就是回归两个本质问题：**用户是谁，创造什么价值**；然后**把组织当产品做，用成长支撑一切**。常识的力量 + 敢于坚持常识的团队 = 最大的竞争力。
