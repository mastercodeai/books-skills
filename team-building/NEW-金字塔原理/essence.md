# 金字塔原理 —— 思考、表达和解决问题的逻辑

> **作者**: 芭芭拉·明托 (Barbara Minto)  
> **定位**: 麦肯锡40年经典培训教材  
> **核心**: 一种重点突出、逻辑清晰、主次分明的思考方式、沟通方式和规范动作

---

## 一、核心思想

### 金字塔的四个基本原则

1. **结论先行**：每篇文章只有一个中心思想，放在最前面
2. **以上统下**：每一层次的思想必须是对下一层次思想的总结概括
3. **归类分组**：每一组中的思想必须属于同一逻辑范畴
4. **逻辑递进**：每一组中的思想必须按照逻辑顺序排列

> 先重要后次要，先总结后具体，先框架后细节，先结论后原因，先结果后过程，先论点后论据。

### 两条核心构建路径

| 方向 | 操作 |
|------|------|
| **自上而下表达** | 结论先行，从金字塔顶端向下展开 |
| **自下而上思考** | 列出要点 → 找出逻辑关系 → 得出结论 |

## 二、金字塔的三大子结构

### 1. 纵向关系：疑问/回答式对话

- 每个层次的思想都会引发读者疑问
- 下一层次的思想必须回答这些疑问
- 通过不断"疑问→回答"的对话，引导读者逐层深入

### 2. 横向关系：演绎推理与归纳推理

**演绎推理（线性推理）：**
- 阐述已存在的某种情况
- 阐述相关情况
- 说明两者同时存在时的隐含意义
- 例：所有人都会死 → 苏格拉底是人 → 苏格拉底会死

**归纳推理（分组概括）：**
- 将具有共同点的事实、思想归类分组
- 概括其共同性
- 关键技能：正确定义该组思想 + 剔除不相称的思想
- 例：法国坦克、德国坦克、俄国坦克 → 波兰将受入侵

**核心区别**：演绎法第二点是对第一点主语或谓语的论述；归纳法同组思想具有类似主语或谓语。

### 3. 序言结构：SCQA框架

| 要素 | 含义 | 作用 |
|------|------|------|
| **S** (Situation) | 背景 | 读者已知的事实 |
| **C** (Complication) | 冲突 | 推动情节发展，引发疑问的因素 |
| **Q** (Question) | 疑问 | 由冲突自然引发的读者疑问 |
| **A** (Answer) | 答案 | 文章的核心思想/中心论点 |

**序言的四种顺序风格**：
- 标准式：S→C→A（背景→冲突→答案）
- 开门见山式：A→S→C（答案→背景→冲突）
- 突出忧虑式：C→S→A（冲突→背景→答案）
- 突出信心式：Q→S→C→A（疑问→背景→冲突→答案）

## 三、思考的逻辑

### 三种逻辑顺序

| 顺序类型 | 含义 | 适用场景 |
|----------|------|----------|
| **时间（步骤）顺序** | 第一、第二、第三 | 流程、行动步骤 |
| **结构（空间）顺序** | 东、中、西 | 组织结构、地理划分 |
| **程度（重要性）顺序** | 最重要、次重要 | 问题、原因、因素 |

### MECE原则（Mutually Exclusive, Collectively Exhaustive）

- **相互独立**：各部分之间没有重叠
- **完全穷尽**：所有部分加起来没有遗漏
- 应用于结构划分、问题分解、原因分析

### 概括各组思想的要点

**行动性思想**（步骤、措施、流程）：
→ 应说明采取行动后取得的**结果/目标**
→ 措辞必须明确、具体，而非模糊笼统

**描述性思想**（背景、信息、原因）：
→ 应说明这些思想具有的**共同点的含义**

**避免"缺乏思想"的句子**：
- ❌ "该公司存在3个问题"
- ✅ "该公司的主要组织问题是缺乏授权"

## 四、解决问题的逻辑

### 界定问题框架

| 要素 | 含义 |
|------|------|
| 切入点/序幕 | 问题发生的背景 |
| 困扰/困惑 | 导致问题出现的事件 |
| R1（现状） | 非期望结果 |
| R2（目标） | 期望结果 |

**问题 = R1 与 R2 之间的差距**

### 七种常见问题类型

1. 不知道如何从R1到R2
2. 知道方案但不确定是否正确
3. 知道正确方案但不知如何实施
4. 方案已实施但行不通
5. 有多个方案不知选哪个
6. 知道R1但无法描述R2
7. 知道R2但不清楚是否处于R1

### 结构化分析的三种诊断框架

1. **呈现有形结构**：画出系统/组织的结构图
2. **寻找因果关系**：财务结构、任务结构、活动结构的树状图
3. **将可能原因分类**：按相似性分类，用是非题逐步排除

### 逻辑树

- 系统地展开所有可能的解决方案
- 遵循MECE原则
- 可用于：寻找方案、评估方案、发现缺陷

## 五、演示的逻辑

### 书面呈现

| 方法 | 适用场景 |
|------|----------|
| 多级标题法 | 长篇文章，层次清晰 |
| 下划线法 | 中等长度，突出要点 |
| 数字编号法 | 便于查找引用 |
| 行首缩进法 | 短文，简洁区分层次 |
| 项目符号法 | 项目进度小结 |

**标题规则**：
- 每一层级标题不可能只有一个
- 相同思想使用相同句型
- 标题用词应提炼思想精髓
- 不要滥用标题

### PPT演示

- 图表幻灯片占90%，文字占10%
- 文字幻灯片只包含最重要的、经分组和总结的思想
- 每次只说明一个论点
- 每张不超过6行或30个单词
- 标题应直接传递信息（用完整句子）

### 写作的图像思维

- 先在大脑中"看见"思想之间的关系
- 将图像复制成文字
- 读者从文字中重新构建图像
- 图像比文字更高效，更易理解和记忆

## 六、关键金句

1. "想清楚，说明白，知道说什么、怎么说"
2. 条理不清的根源不是写作风格，而是**文章结构**
3. 大脑短期记忆无法一次容纳约7个以上的记忆项目
4. 读者的大脑只能逐句理解，会自动寻找逻辑关系
5. 写作时的思路是自下而上，表达时的顺序是自上而下
6. 每一组思想都隐含一个总结性思想，必须明确指出
7. 用明确语言表达目标，否则无法判断行动是否完整
