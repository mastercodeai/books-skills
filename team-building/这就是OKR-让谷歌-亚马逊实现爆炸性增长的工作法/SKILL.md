# OKR目标管理与团队协同

## Description
OKR（目标与关键结果）是一套源自英特尔、经谷歌发扬光大的目标管理框架，用于帮助团队聚焦优先事项、实现跨部门协同、基于数据追踪进展、并持续挑战极限。适用于从初创企业到大型组织的各类场景。

## When to Use
- 团队目标不清晰、缺乏聚焦，多头并进却无重点
- 部门之间协同差，各自为战、重复劳动
- 目标设定流于形式，年终才回顾，无法及时调整
- 需要从年度KPI转向更敏捷的目标管理
- 推动组织文化变革，建立透明、责任驱动的工作方式
- 员工敬业度低，不清楚自己的工作如何与公司使命关联

## How to Use

### 第一步：理解OKR基本结构

**目标（O）= 方向**：鼓舞人心、具体、行动导向
**关键结果（KR）= 里程碑**：可衡量、有时限、有挑战性

```
O：成为华东区市场份额第一的SaaS产品
  KR1：Q1新增付费客户200家（当前150家）
  KR2：客户月留存率从85%提升到92%
  KR3：NPS评分从35提升到50
```

**规则：**
- 每周期3~5个目标，每个目标≤5个关键结果
- KR必须有数字，没有数字不算KR
- 所有KR完成 → O自然达成
- OKR ≠ 日常任务清单，是精心策划的优先事项

### 第二步：区分两种OKR

| 类型 | 承诺型OKR | 挑战型OKR |
|------|----------|----------|
| 定义 | 必须完成的运营目标 | 大胆的突破性目标 |
| 完成标准 | 100%完成 | 60~70%完成即为成功 |
| 来源 | 通常自上而下 | 可来自任何层级 |
| 示例 | 本季度发布V2.0版本 | 年底用户量增长10倍 |
| 失败代价 | 不可接受 | 可接受，是学习机会 |

### 第三步：设定OKR的流程

```
1. 领导层确定公司最高目标（1~3个）
   ↓
2. 团队围绕公司目标设定部门OKR（自上而下协同）
   ↓
3. 个人设定个人OKR，约50%自下而上涌现
   ↓
4. 团队会议对齐：确认依赖关系、消除冲突
   ↓
5. 公开发布，全员可见
```

**关键技巧：**
- **数量匹配**：产出型KR配质量型KR（如"处理100张工单" 配 "客户满意度≥4.5"）
- **时间框架**：季度OKR为主，年度OKR为辅
- **自下而上**：鼓励一线员工提出OKR，管理层选择性提升为公司级目标

### 第四步：OKR的节奏与追踪

```
周节奏：每周站会回顾进度（15分钟）
月节奏：月度检查，识别障碍，调整资源
季节奏：季度评分 + 总结反思 + 设定下季度OKR
```

**评分标准（谷歌0~1.0制）：**
- 0.7~1.0 = 绿色（目标完成）
- 0.4~0.6 = 黄色（有进展但未完成）
- 0~0.3 = 红色（目标失败）

**季度总结三步：**
1. **客观评估**：计算KR完成率
2. **自我评估**：结合情境判断（0.67的数据分可能值得0.9的评价分）
3. **反思**：学到了什么？下周期如何改进？

### 第五步：OKR + CFR = 持续性绩效管理

**CFR三要素：**

**对话（Conversation）**——经理与员工的定期一对一
- 五个核心问题：
  1. 你正在做什么？
  2. 你的OKR进展如何？
  3. 有什么阻碍？
  4. 需要我提供什么帮助？
  5. 需要什么帮助实现职业目标？
- 频率：每1~2周一次，每次30~60分钟
- 关键：以下属为主导，主管倾听并指导

**反馈（Feedback）**——双向沟通
- 具体、及时、建设性
- 负面反馈示例："上周会议组织晚了导致混乱"
- 正面反馈示例："演讲开场吸引全场注意力，结尾的下一步规划很棒"
- 推动360度反馈文化

**认可（Recognition）**——对等表彰
- 鼓励同事间互相认可
- 对小成就也予以赞扬
- 分享成就背后的故事

**关键原则：OKR与薪酬分离**
- OKR评分不直接决定奖金
- 谷歌：OKR仅占绩效考核的1/3
- 评分周期结束后清零

### 第六步：用OKR推动文化变革

**健康OKR文化的五要素（谷歌"亚里士多德"项目）：**
1. 结构与清晰度：目标、角色、计划明确
2. 心理安全感：能安心承担风险
3. 工作意义：做对每个人重要的事
4. 可靠性：彼此信赖，按时高质量完成
5. 工作影响：确信工作有意义

**文化变革路径：**
```
1. 先在管理层试行1~2个季度
2. 高管以身作则，公开自己的OKR
3. 指定OKR导师，推动全员参与
4. 逐步扩大到全组织
5. 将OKR融入日常对话和决策
```

## Failure Modes

> OKR落地中最常见的失败模式、诊断信号和修复策略。

### FM1：目标膨胀——OKR数量失控
- **信号**：团队有6+个O，每个O下4+个KR，周会时无法逐一回顾
- **根因**：把日常运营任务和战略突破混为一谈；不敢做取舍
- **修复**：强制上限（3个O / 每O≤3KR）；区分"必须做"和"想做"，多余项放入backlog

### FM2：KR变成任务清单
- **信号**：KR写成"完成XX功能上线"、"召开3次会议"——做完即满分，但业务指标没变化
- **根因**：混淆了"产出（output）"和"成果（outcome）"
- **修复**：每个KR追问"so what?"——"上线XX功能→so what?→用户转化率提升15%"

### FM3：评分通胀/全员满分
- **信号**：季度末所有人自评0.8+，但业务数据无明显改善
- **根因**：目标设定太保守；或评分与薪酬挂钩导致"安全打法"
- **修复**：引入挑战型OKR（60~70%完成即成功）；评分与薪酬解耦；校准会议横向对比

### FM4：设定后束之高阁
- **信号**：季度初热闹设定，之后再无人提及，季度末才"回忆"打分
- **根因**：缺少周/月节奏；OKR工具与日常工作流脱节
- **修复**：周站会必看KR进展（15分钟）；OKR嵌入项目管理工具；月度红绿灯预警

### FM5：部门墙——对齐断裂
- **信号**：A部门的KR依赖B部门，但B部门OKR中无对应承诺
- **根因**：设定流程缺少跨团队对齐环节
- **修复**：季度初召开"对齐会"，明确依赖关系并写入双方OKR；设置跨部门KR owner

### FM6：推行半年后热度骤降
- **信号**：前2个季度积极参与，第3季度开始敷衍，第4季度名存实亡
- **根因**：缺少高管持续背书；看不到"OKR帮我赢"的正反馈
- **修复**：高管每季度公开分享自己的OKR和评分；展示OKR驱动的胜利故事；指定OKR champion持续推动

## Anti-Patterns

> 看起来像OKR，但实际上违背OKR精神的做法。

### AP1：KPI换皮
- **做法**：把原有KPI指标直接改名叫KR，考核方式不变
- **问题**：失去了OKR的挑战性和灵活性，员工继续"保底思维"
- **正解**：OKR聚焦"突破性方向"，KPI管"运营底线"，两者共存但不混淆

### AP2：自上而下一刀切
- **做法**：高管写好所有OKR，逐级下达，基层只负责执行
- **问题**：失去自下而上的创新涌现；一线员工无ownership
- **正解**：约50%自上而下对齐，50%自下而上涌现；鼓励一线提出"如果我来定目标"

### AP3：OKR与奖金强绑定
- **做法**：KR完成率直接决定奖金系数
- **问题**：员工趋向保守目标、隐藏真实进度、博弈评分
- **正解**：OKR用于方向对齐和成长学习，绩效考核通过CFR对话综合评估

### AP4：完美主义设定
- **做法**：花2周反复打磨OKR措辞，追求"完美表述"
- **问题**：过度设计导致行动延迟；OKR本就是迭代的
- **正解**：先快速设定（"足够好"即可），在追踪中持续调整

### AP5：只看数字不看故事
- **做法**：季度末机械计算KR完成百分比，不分析背后原因
- **问题**：0.7分可能是巨大成功（市场剧变下保住了70%），0.9分可能是目标太保守
- **正解**：评分必须伴随反思——"为什么是这个分数？学到了什么？"

### AP6：用OKR管日常琐事
- **做法**：把"每周写周报"、"按时打卡"写成KR
- **问题**：OKR被琐碎任务稀释，失去战略聚焦意义
- **正解**：日常运营用checklist/流程管理；OKR只承载战略性突破目标

## Checkpoints

> 每个阶段的自检清单，确保OKR不走样。

### 设定阶段检查点（季度初）
- [ ] 每个O是否回答了"我们想去哪里？"（方向性、鼓舞人心）
- [ ] 每个KR是否有明确数字？没有数字不算KR
- [ ] 每个KR是否回答了"怎么知道到了？"（可验证）
- [ ] 承诺型OKR和挑战型OKR是否区分明确？
- [ ] 是否控制在3个O、每O≤3~5个KR？
- [ ] 是否进行了跨团队对齐，确认依赖关系？
- [ ] 个人OKR中是否有至少50%是自下而上提出的？
- [ ] 所有OKR是否已公开、全员可见？

### 追踪阶段检查点（每周/每月）
- [ ] 周站会是否在15分钟内完成KR进展回顾？
- [ ] 是否有KR处于"红灯"状态（严重落后）？如何干预？
- [ ] 月度检查是否识别了新的障碍和资源需求？
- [ ] 是否有KR因为外部环境变化需要调整？
- [ ] 团队是否在日常工作中引用OKR做决策优先级判断？

### 复盘阶段检查点（季度末）
- [ ] 每个KR是否用0~1.0进行了客观评分？
- [ ] 是否区分了"数据分"和"评价分"（结合情境判断）？
- [ ] 是否写下了"本季度最大收获"和"最大教训"？
- [ ] 评分是否伴随着反思而非仅填数字？
- [ ] 下季度OKR是否基于本季度学习进行了调整？

### 推行阶段检查点（组织层面）
- [ ] 高管是否公开了自己的OKR并带头评分？
- [ ] 是否指定了OKR导师/champion持续推动？
- [ ] 是否在管理层试行了至少1~2个季度再全面铺开？
- [ ] OKR是否与薪酬解耦？（至少不强绑定）
- [ ] 是否建立了"OKR胜利故事"的分享机制？

## Common Mistakes & Solutions

| 常见错误 | 正确做法 |
|---------|---------|
| 设定太多OKR（7~8个） | 聚焦3~5个，少即是多 |
| KR无法衡量（"提升品牌形象"） | 量化："品牌知名度调查得分从60提升到80" |
| OKR与薪酬直接挂钩 | 分离，用CFR进行持续性绩效管理 |
| 全部自上而下 | 50%自上而下，50%自下而上 |
| 设定后从不回顾 | 至少每月检查一次 |
| 所有目标都是承诺型 | 区分承诺型（100%完成）和挑战型（60~70%） |
| 目标太保守 | 设定让自己"不舒服的兴奋"的目标 |
| 忽视跨部门依赖 | 季度初确认依赖关系，消除冲突 |
| 第一次推行失败就放弃 | 需要4~5个季度才能完全适应 |

## Key Principles

1. **想法很容易，执行最重要** — OKR是执行力的放大器
2. **少即是多** — 聚焦才能产生实质性差异
3. **透明是力量** — 公开目标让政治失去毒性
4. **敢于失败** — 挑战性目标推动创新，允许失败是前提
5. **数据驱动** — 没有数字就没有关键结果
6. **持续对话** — OKR + CFR才构成完整系统
7. **文化先行** — 没有健康文化，再好的工具也会失效
8. **耐心坚持** — 需要4~5个季度才能真正内化OKR
