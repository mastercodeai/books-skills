> 来源：《给你一个亿你能干什么》第二部
> 适用场景：创业融资、VC沟通、商业计划书撰写、公司估值谈判

## 🧭 场景路由 (Scene Router)

```
用户需求 → 识别场景 → 进入对应技能模块

┌─────────────────────────────────────────────────────────────────┐
│  场景识别                        │  推荐路径                    │
├─────────────────────────────────────────────────────────────────┤
│  "帮我写BP/商业计划书"           │  → 技能一 (21页框架)         │
│  "不知道找哪些投资人"            │  → 技能二 (VC筛选矩阵)       │
│  "怎么谈估值/估值太高怎么办"     │  → 技能三 (估值谈判)         │
│  "融资邮件怎么写/面谈怎么准备"   │  → 技能五 (投资人沟通话术)   │
│  "公司是不是快不行了"            │  → 技能四 (生存检查)         │
│                                    │    + 技能六 (失败信号)       │
│  "Term Sheet拿到了怎么处理"       │  → 技能五 §Term Sheet       │
│  "怎么算股权稀释"                │  → 核心公式                  │
└─────────────────────────────────────────────────────────────────┘

复合场景：
- 首次融资 → 技能一 → 技能二 → 技能五 (顺序执行)
- 估值谈判 → 技能三 + 核心公式
- 融资受阻 → 技能四 + 技能六 (诊断) → 技能二 (重新匹配) 或 技能三 (调整估值)
```

## 技能一：商业计划书21页框架

### 使用场景
需要向VC提交商业计划书时使用。

### 执行步骤
1. **第1页：执行摘要** — 一句话说清产品、一句话说清市场、一句话说清团队、一句话说清融资需求
2. **第2页：产品/服务** — 你做什么、怎么赚钱、为什么是你
3. **第3页：市场分析** — 宏观数据（引用iResearch等权威来源）+ 微观目标市场
4. **第4页：竞争分析** — 列出3-5个竞品，说明差异化优势
5. **第5页：团队** — 核心成员背景，突出实战经验而非学历
6. **第6页：里程碑** — 已完成什么、下一步做什么、关键时间节点
7. **第7页：财务计划** — 3年预测（第一年详细），包含假设、利润表、现金流
8. **第8页：股权结构** — 现有股东 + 投资后稀释情况
9. **第9页：组织架构** — 公司注册地、部门设置
10. **第10页：已有投资** — 自己投了多少（体现诚意）
11. **第11-21页：补充** — 产品截图、路线图、商业模式图等

### 关键原则
- CEO亲自写，不要外包
- 总页数控制在14页以内（加上封面、封底）
- VC用15秒决定是否继续看——第一页是生死线

### ✅ BP检查点 (Checkpoints)
```
□ 执行摘要是否在30秒内能读完？
□ 市场数据是否引用权威来源（非自己编造）？
□ 竞品分析是否诚实（不要说"没有竞争对手"）？
□ 财务预测是否有合理的假设支撑？
□ 股权结构是否清晰（含期权池预留）？
□ 团队介绍是否突出"为什么是我们做这件事"？
```

### ❌ BP常见反模式 (Anti-Patterns)
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| PPT超过30页 | VC没耐心看 | 控制在14-21页 |
| 只讲技术不讲市场 | VC投资的是市场机会 | 先讲市场规模，再讲技术 |
| 用"颠覆""革命"等大词 | 空洞无物 | 用具体数据和案例 |
| 没有竞品分析 | 要么市场太小，要么不了解市场 | 诚实列出竞品并说明差异化 |
| 财务预测太乐观 | 第一年就盈利不现实 | 展示合理假设和敏感性分析 |
| 让秘书/外包写BP | CEO不理解自己的业务 | CEO亲自写，至少亲自改 |

### ⚠️ BP失败模式 (Failure Modes)
```
失败模式1: "故事不连贯"
  症状: 市场很大，但产品与市场之间缺乏逻辑连接
  后果: VC觉得你在硬凑
  修复: 用"问题→解决方案→市场→商业模式"的逻辑链重写

失败模式2: "数据造假被识破"
  症状: 引用的数据来源模糊，或与公开数据矛盾
  后果: 信用破产，融资失败
  修复: 所有数据必须可追溯，宁可保守也不要夸大

失败模式3: "忽视第一页"
  症状: 执行摘要写了半页废话
  后果: VC直接跳过，后面白写
  修复: 用"一句话公式"打磨执行摘要：我们为[目标客户]提供[产品]，解决[问题]，市场规模[X]，已实现[Y]
```

## 技能二：VC筛选与匹配矩阵

### 使用场景
不知道该找哪些VC时使用。

### 筛选维度
```
第一步：确定你的阶段
  → 种子期/天使轮 → 找早期VC（10万-100万美元）
  → A轮/B轮 → 找成长期VC（100万-1000万美元）
  → C轮及以上 → 找扩张期VC/PE（1000万+美元）

第二步：匹配行业
  → 列出目标VC清单
  → 上VC官网查看投资方向
  → 过滤掉不投你行业的VC

第三步：确认投资规模
  → 你需要300万？不要找只投种子期10万的VC
  → 你需要10万？不要找C轮基金

第四步：获取推荐
  → 通过已投企业创始人推荐（最有效）
  → 通过律师/会计师推荐
  → 通过创业活动认识
```

### VC拒绝话术识别表
| VC说的话 | 真实含义 | 应对 |
|---------|---------|------|
| "很感兴趣，最近太忙" | 不投 | 礼貌感谢，不再追 |
| "方向偏了，再观察" | 不投 | 不要追问时间表 |
| "可以介绍给其他VC" | 不投 | 可以接受介绍 |
| "我们需要内部讨论" | 可能有兴趣 | 等1-2周后跟进一次 |
| "请发详细BP" | 有兴趣 | 立即发送 |

### ✅ VC筛选检查点
```
□ 是否已明确自己所处的融资阶段？
□ 目标VC清单是否至少包含10家？
□ 每家VC的投资方向是否与你的行业匹配？
□ 是否找到了至少1个有效的推荐人？
□ 是否了解目标VC的决策周期和投资节奏？
```

### ❌ VC筛选反模式
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 海投所有VC | 效率低，且会留下不好印象 | 精准匹配，优先级排序 |
| 只找知名VC | 竞争激烈，小项目不受重视 | 早期项目找专注早期的中小基金 |
| 没有推荐人就直接联系 | 成功率极低 | 先建立人脉，获取推荐 |
| 同时跟太多VC | 精力分散，每家都聊不深 | 同时深度接触3-5家 |

### ⚠️ VC筛选失败模式
```
失败模式1: "阶段错配"
  症状: 种子期项目去找B轮基金
  后果: 浪费时间，可能收到误导性反馈
  修复: 严格按阶段匹配，种子期找天使/早期VC

失败模式2: "行业错配"
  症状: 电商项目去找医疗基金
  后果: 即使VC有兴趣，也无法提供行业资源
  修复: 研究VC过往投资案例，确认行业覆盖

失败模式3: "期望错配"
  症状: 需要100万但只接触投1000万的基金
  后果: 项目太小，VC不值得花时间
  修复: 按融资金额筛选，找单笔投资额匹配的VC
```

## 技能三：创业公司估值谈判框架

### 使用场景
与VC谈判估值时使用。

### 估值方法

**方法一：对标法**
```
找到3-5家同行业、同阶段的被投公司
→ 取估值中位数
→ 根据自身优劣势调整（±20%）
```

**方法二：VC回报反推法**
```
VC期望回报 = 10x-50x（早期）或 3x-5x（后期）
→ VC投入金额 × 期望倍数 = 退出时公司价值
→ 退出时价值 ÷ 期望倍数 = 当前合理估值
```

**方法三：里程碑法**
```
列出未来12-18个月的关键里程碑
→ 每个里程碑对应一个估值台阶
→ 当前估值 = 下一个里程碑的价值 × 折现系数
```

### 估值谈判策略
1. **先让VC出价** — 不要自己先报价
2. **准备3个数字** — 理想值、可接受值、底线
3. **用数据说话** — 市场规模、增长数据、竞品估值
4. **留出空间** — 不要把估值抬到让B轮投资人无法接盘

### 估值死亡陷阱
```
A轮估值过高 → B轮估值倒挂 → 无人接盘 → 公司"死在估值上"
```
**原则：** 合理估值，给后续投资人留出赚钱空间

### ✅ 估值检查点
```
□ 是否准备了3个估值数字（理想/可接受/底线）？
□ 是否收集了3-5家可比公司的估值数据？
□ 是否计算了VC的预期回报倍数？
□ 估值是否为后续融资轮次留出了空间？
□ 是否有支撑估值的可量化里程碑？
□ 是否了解反稀释条款对估值的实际影响？
```

### ❌ 估值反模式
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 先报价 | 可能报低了，或吓跑VC | 让VC先出价 |
| 估值越高越好 | B轮接盘困难，死在估值上 | 合理估值，留空间 |
| 只看估值不看条款 | 低估值+好条款可能优于高估值+苛刻条款 | 综合评估Term Sheet |
| 用未来梦想估值 | 无法支撑的天价估值 | 用当前数据+合理假设 |
| 拒绝谈判 | 僵持不下，错失机会 | 准备3个数字，有让步空间 |

### ⚠️ 估值失败模式
```
失败模式1: "估值泡沫"
  症状: A轮估值远超同行，B轮无法维持增长预期
  后果: B轮估值倒挂，老投资人不愿让步，新投资人不愿接盘
  修复: A轮估值 = 可比公司中位数 × (0.8~1.2)

失败模式2: "条款陷阱"
  症状: 估值很高，但签了苛刻的反稀释条款
  后果: 下一轮融资时，创始人股权被严重稀释
  修复: 请律师逐条审阅，特别是反稀释、优先清算权、一票否决权

失败模式3: "自欺欺人"
  症状: 用最乐观的假设计算估值
  后果: 与VC的认知差距太大，谈判破裂
  修复: 准备基准/乐观/悲观三种估值方案
```

## 技能四：SURVIVAL创业生存检查清单

### 使用场景
每月/每季度自我检查创业状态。

### 检查清单
- [ ] **S** — 我清楚当前市场环境和竞争格局吗？
- [ ] **U** — 我有没有盲目追求速度而忽略质量？
- [ ] **R** — 我记住公司的核心定位了吗？有没有跑偏？
- [ ] **V** — 我是否在用意志力克服恐惧和困难？
- [ ] **I** — 我能否在变化中快速应变？
- [ ] **V** — 我创造了什么实际价值？
- [ ] **A** — 我今天采取了什么具体行动？
- [ ] **L** — 我珍惜每一天的创业机会了吗？

### Plan A/B 双轨检查
- Plan A（融资路线）进展如何？VC反馈是什么？
- Plan B（自力更生）进展如何？现金流还能撑多久？
- 如果Plan A失败，Plan B能否独立支撑公司生存？

### ✅ 生存检查点
```
□ 现金流还能撑几个月？（低于6个月为红色警报）
□ 本月是否有新增收入来源？
□ 团队核心成员是否稳定？
□ 是否有明确的Plan B并定期更新？
□ 上次全面SURVIVAL检查是什么时候？
```

### ❌ 生存检查反模式
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 只关注融资不关注收入 | 融资失败就死 | Plan A/B并行，自力更生优先 |
| 检查流于形式 | 发现问题不行动 | 检查后必须有Action Item |
| 回避现金流问题 | 麻痹大意 | 每月更新现金流预测 |

## 技能五：投资人沟通话术

### 使用场景
与VC进行正式沟通（邮件、面谈、谈判）时使用。

### 初次接触（邮件模板要素）
```
主题：[公司名] - [一句话描述] - 融资沟通

正文：
1. 一句话介绍自己和公司
2. 一句话说清产品/服务
3. 一句话说清市场规模
4. 一句话说清当前进展（用户数/收入/里程碑）
5. 一句话说清融资需求（金额、阶段）
6. 附件：Executive Summary（一页纸）
```

### 面谈准备
1. 准备15分钟版本的PPT（不是21页全部讲完）
2. 准备3个最难的问题及回答
3. 准备财务模型的Excel备份
4. 准备竞品对比表
5. **最重要的：** 准备一个让投资人记住你的故事

### 拿到Term Sheet后的动作
1. 不要立即签字
2. 请律师审阅关键条款（估值、反稀释、优先清算权、董事会席位）
3. 确认投资到账时间表
4. 确认投后管理方式（汇报频率、董事会参与度）

### ✅ 沟通检查点
```
□ 邮件是否能在1分钟内读完？
□ 是否准备了15分钟精简版PPT？
□ 是否预演了3个最难问题的回答？
□ 是否准备了一个让人记住的故事？
□ Term Sheet是否请律师审阅过？
□ 是否了解投资到账的时间承诺？
```

### ❌ 沟通反模式
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 邮件写成长篇大论 | VC不会看 | 5句话+一页摘要 |
| 面谈时照念PPT | 枯燥无味 | 讲故事，PPT只是辅助 |
| 过度承诺 | 无法兑现，失去信任 | 保守承诺，超额交付 |
| 被拒绝后纠缠 | 损害口碑 | 礼貌感谢，保持关系 |
| Term Sheet不看就签 | 可能有苛刻条款 | 必须律师审阅 |

### ⚠️ 沟通失败模式
```
失败模式1: "过度热情"
  症状: 被拒绝后反复追问，或同时跟太多VC暧昧
  后果: 投资圈很小，口碑受损
  修复: 被拒后礼貌结束，同时推进2-3家即可

失败模式2: "信息不对称"
  症状: 隐瞒关键问题（如团队冲突、法律风险）
  后果: 尽职调查时暴露，融资失败且信用破产
  修复: 主动披露风险，展示你有应对方案

失败模式3: "条款盲签"
  症状: 拿到Term Sheet太兴奋，不仔细审阅
  后果: 被苛刻条款套牢，后续融资被动
  修复: Term Sheet必须律师审阅，至少48小时冷静期
```

## 技能六：创业失败的五个信号

### 使用场景
定期自我诊断，判断是否需要调整方向。

### 信号清单
1. **团队没有足够经验** — 核心成员没有行业积累
2. **想法太多** — 什么都想做，什么都做不精
3. **没有差异化** — 产品和竞品没有本质区别
4. **没有投资** — 自己不投钱，只等VC
5. **现金流紧张** — 每月都在为工资发愁

### 应对策略
- 信号1-3：调整产品方向或团队
- 信号4：先自己投入真金白银，展示诚意
- 信号5：立即启动Plan B，控制成本，寻找收入来源

### ✅ 失败信号检查点
```
□ 过去3个月是否出现过以上任何信号？
□ 出现信号后是否采取了具体行动？
□ 团队是否对信号有共识？
□ 是否有人在回避问题？
```

### ❌ 信号应对反模式
| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 集体沉默 | 问题被掩盖 | 定期坦诚讨论信号 |
| 只诊断不行动 | 问题持续恶化 | 每个信号必须有对应Action |
| 把失败归咎于外部 | 无法改进 | 先从自身找原因 |

## 核心公式

```
公司估值 = 投资金额 ÷ VC获得股权比例

VC期望回报 = 投资金额 × (10x ~ 50x)  [早期]
VC期望回报 = 投资金额 × (3x ~ 5x)    [后期]

投资回报率 = (退出价值 - 投资金额) / 投资金额 × 100%

股权稀释 = 新投资金额 / (公司估值 + 新投资金额)
```

## 📋 融资全流程检查清单 (Master Checklist)

```
阶段一：准备期
  □ BP已完成（技能一检查点全部通过）
  □ 财务模型已建好（含敏感性分析）
  □ 目标VC清单已列出（至少10家）
  □ 推荐人已联系（至少1个有效推荐）
  □ 团队对融资目标有共识

阶段二：接触期
  □ 邮件已按模板发送
  □ 面谈PPT已准备（15分钟版）
  □ 最难问题已预演
  □ 竞品对比表已准备

阶段三：谈判期
  □ 估值方案已准备（3个数字）
  □ 可比公司数据已收集
  □ 律师已介入审阅条款
```
