# SKILL.md — 创新者的解答 思维工具箱

## 场景路由 (Scene Routing)

> **先选路，再走路。** 根据你当前面对的场景，跳转到对应 Skill 组合。

### 场景 A：评估一个新构想是否值得做
→ **Skill 1** (破坏性试金石) → **Skill 7** (资金好坏) → **Skill 6** (策略流程)
- 核心问题：这是破坏性机会还是延续性陷阱？
- 如果是延续性创新 → 停止，让大公司去做，你没有优势

### 场景 B：开发新产品 / 进入新市场
→ **Skill 2** (客户任务) → **Skill 3** (产品架构) → **Skill 8** (品牌任务匹配)
- 核心问题：客户到底在什么处境下雇佣你的产品？
- 如果客户任务定义不清 → 所有后续设计都是空中楼阁

### 场景 C：规划企业增长 / 选择价值链位置
→ **Skill 4** (高利润定位) → **Skill 3** (产品架构) → **Skill 5** (组织能力)
- 核心问题：利润正在往哪里转移？我们应该整合什么？
- 如果不理解货品化-反货品化周期 → 守着正在下沉的利润区等死

### 场景 D：为新业务组建团队 / 选择组织形式
→ **Skill 5** (RPV诊断) → **Skill 6** (策略流程) → **Skill 7** (资金评估)
- 核心问题：现有组织能不能执行这个任务？差在哪里？
- 如果用主流组织做破坏性创新 → 必然被资源分配流程扼杀

### 场景 E：CEO 战略决策 / 投资者评估
→ **Skill 6** (策略流程) → **Skill 7** (资金评估) → **Skill 1** (试金石复核)
- 核心问题：我们用对了策略流程吗？钱的性质对吗？
- 如果对早期业务用谋划型策略 → 预测不准但团队照做 → 烧钱

## Skill 1: 破坏性创新试金石测试

### 何时使用
评估一个新业务构想、产品策略或市场切入点是否具备破坏性潜力。

### 操作步骤

1. **新市场破坏测试**（至少一个为"是"）:
   - 是否有大量人群因缺乏资金/技术/设备而无法完成某项任务？
   - 客户是否需要克服不便到集中地点才能使用现有产品/服务？

2. **低端市场破坏测试**（两个都为"是"）:
   - 是否有客户愿意以更低价格购买"够用就好"的产品？
   - 能否建立一种商业模式，以低价服务低端客户但仍保持良好赢利？

3. **不对称动机测试**（必须为"是"）:
   - 该策略对**所有**主要先入者都是破坏性而非延续性的吗？
   - 若对其中任何一家只是延续性创新 → 先入者必赢，你必输

### 判断标准
- 通过三项测试 → 破坏性策略，值得推进
- 未通过 → 只能作为延续性创新，需要重新塑造构想
- 延续性创新策略适合在已有优势的企业内推进，不适合创业

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 自欺式通过 | 每项测试都勉强说"是"，忽略反例 | 进入伪破坏性市场，被真正的先入者碾压 |
| 只测新市场不测不对称动机 | 找到零消费人群，但对某家先入者是延续性 | 先入者快速复制，创业公司被收购或出局 |
| 将"降维打击"误认为破坏性 | 用先进技术降维进入低端市场 | 本质上仍是延续性创新，先入者跟进无压力 |
| 忽略商业模式可行性 | 低端客户存在，但服务成本太高 | 低价策略无法赢利，资金链断裂 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **技术驱动思维**: "我们的技术更好，所以是破坏性的" → 破坏性看的是市场定位，不是技术优劣
- ❌ **规模幻想**: "先亏钱做大再赢利" → 破坏性创新应从第一天就追求赢利
- ❌ **对标陷阱**: "Uber 就是破坏性创新" → 分析每个案例时重新走三步测试，别套标签
- ❌ **选择性测试**: 只看通过的那项，忽略未通过的 → 三项测试是"且"关系，不是"或"

### ✅ 检查点 (Checkpoints)
- [ ] 三个测试全部独立评估，不是一次性"感觉通过"
- [ ] 至少找到了一个具体的"零消费"场景或被过度服务的低端客户群
- [ ] 对每家主要先入者分别验证了不对称动机
- [ ] 确认了即使先入者反击，你的商业模式仍然可持续

## Skill 2: 客户任务分析（Jobs to Be Done）

### 何时使用
开发新产品、细分市场、设计营销策略时，避免"基于属性"的无效市场细分。

### 操作步骤

1. **观察真实行为**:
   - 到客户使用产品的场景中观察18小时以上
   - 记录: 何时购买、何时使用、和谁一起、是否打包、同时购买什么

2. **挖掘任务**:
   - 问: "你用这个产品来完成什么？"
   - 问: "不用它的时候，你用什么替代品完成同样的事？"
   - 问: "这个产品在哪些方面没能帮到你？"

3. **定义任务的三个维度**:
   - **功能维度**: 实际要完成的物理任务
   - **情感维度**: 使用时的感受需求
   - **社会维度**: 想在他人面前呈现的形象

4. **识别真正的竞争对手**:
   - 竞争对手不是同类产品，而是所有能完成同一任务的替代方案
   - 奶昔的竞争者: 香蕉、百吉饼、无聊

5. **针对性改良**:
   - 只改良与任务相关的特性
   - 忽略与任务无关的改进（即使客户在座谈中提到）

### 避坑清单
- ❌ 按人口统计细分市场（年龄、收入、行业）
- ❌ 按产品类别细分市场（"我们在手持设备市场竞争"）
- ❌ 试图改变客户的任务（"如果客户愿意……"）
- ❌ 用历史数据推断新市场（数据反映过去，不反映任务）
- ✅ 按客户处境和要完成的任务细分

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 任务定义太宽 | "客户需要沟通" → 任务大而无当 | 无法指导具体设计决策，产品什么都想做 |
| 任务定义太窄 | "客户需要更快的马" → 只看表面需求 | 被满足同一任务的完全不同形态产品颠覆 |
| 假客户验证 | 用焦点小组代替真实场景观察 | 客户说的是理想化的自己，不是真实行为 |
| 忽略情感/社会维度 | 只看功能，忽略客户的情绪和社交需求 | 产品功能好但没人用（如社交场景下的学习工具） |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **问卷迷信**: 发问卷问客户想要什么 → 客户不知道自己要什么，只知道什么让自己痛苦
- ❌ **数据替代观察**: 用销售数据推断任务 → 数据告诉你"是什么"，不告诉你"为什么"
- ❌ **任务万能论**: 认为找到任务就成功了一半 → 任务只是起点，商业模式和执行同样关键
- ❌ **忽略替代品**: 只和同类产品比 → 你的真正竞争者可能完全不在你的行业里

### ✅ 检查点 (Checkpoints)
- [ ] 至少做了18小时的真实场景观察，不是闭门造车
- [ ] 任务陈述包含具体处境 + 具体目标（"当___时，我需要___以便___"）
- [ ] 列出了至少3个非同类产品的替代方案
- [ ] 任务的三个维度（功能/情感/社会）都有覆盖

## Skill 3: 产品架构选择矩阵

### 何时使用
决定产品设计应采用交互式（专利/整合）还是模块化（开放/外包）架构。

### 决策框架

```
客户对产品的满意度如何？

"不够好"（性能缺口大）          "过于好"（性能过剩）
    ↓                                ↓
交互式/专利性架构               模块化/标准架构
    ↓                                ↓
垂直整合（内部生产关键组件）     分散外包（专业分包商组装）
    ↓                                ↓
竞争优势: 性能、差异化            竞争优势: 速度、灵活性、成本
```

### 三个外包前提条件（缺一则不可外包）
1. 供应商和客户清楚了解产品规格和接口标准
2. 供应商有能力按规格生产
3. 接口处不存在未定义的相互依赖关系

### 整合-分散周期判断
- 客户需求断层式上升 → 回到"不够好" → 整合再次成为优势
- 技术超越需求 → 进入"过于好" → 模块化成为主流
- 核心原则: **向未来的财富增长点整合，而非固守过去的**

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 误判成熟度 | 性能"不够好"时就搞模块化 | 产品性能输给整合型竞争对手 |
| 过早外包 | 三个前提条件不满足就外包 | 产品质量失控、供应链断裂 |
| 拒绝模块化 | 性能"过于好"时仍坚持整合 | 成本高、速度慢，被模块化对手蚕食 |
| 静态判断 | 一次评估后不再更新 | 市场已从"不够好"变"过于好"，架构未跟上 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **整合洁癖**: "核心组件必须自己做" → 到了"过于好"阶段，自己做是在烧钱
- ❌ **外包惯性**: "外包总是更便宜" → 前提条件不满足时，外包的质量灾难更贵
- ❌ **忽略周期**: 认为当前架构是永久的 → 整合-分散是周期，会反复交替

### ✅ 检查点 (Checkpoints)
- [ ] 明确判断了当前产品在"不够好—过于好"光谱上的位置
- [ ] 外包前验证了三个前提条件全部满足
- [ ] 评估了未来2-3年客户需求和技术供给的变化趋势
- [ ] 整合方向指向未来利润增长点，而非过去的利润区

## Skill 4: 高利润定位导航器

### 何时使用
在价值链上寻找最有利可图的定位，决定企业应整合价值链的哪些环节。

### 核心定律
**高利润守恒定律**: 当价值链某阶段因货品化而利润消失时，相邻阶段会出现新的高利润专利性产品。

### 导航步骤

1. **画出你的行业价值链**（从原材料到终端客户）
2. **标注每个环节的"完善度"**:
   - 哪些环节性能"不够好"？→ 高利润区（交互式、专利性）
   - 哪些环节性能"过于好"？→ 低利润区（模块化、货品化）
3. **定位利润增长点**:
   - 整合"不够好"的环节
   - 对"过于好"的环节采用模块化外包
4. **预测利润转移方向**:
   - 破坏者从低端向上突破
   - 每突破一层，该层利润下降，上一层子系统利润上升
5. **提前布局**: 滑向冰球即将到达的位置

### 反货品化信号
- 终端产品开始货品化 → 寻找性能导向的子系统
- 子系统开始标准化 → 寻找更高层级的整合机会
- 品牌价值从产品转移 → 关注渠道和客户接口的品牌建设

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 守死利润区 | 明知当前环节在货品化，仍固守不动 | 利润持续蒸发，被迫在最低点转型 |
| 追逐上一波利润 | 进入别人正在离开的高利润区 | 进入时利润已开始下降 |
| 忽略品牌转移 | 产品货品化后仍押注产品品牌 | 品牌价值随利润一起蒸发 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **利润近视**: 只看当前利润率，不看变化方向 → 高利润区正在消失不是好消息
- ❌ **全产业链幻想**: 想整合所有环节 → 资源分散，每个环节都做不好
- ❌ **忽略破坏者视角**: 只从在位者视角看价值链 → 破坏者正在重新定义价值链结构

### ✅ 检查点 (Checkpoints)
- [ ] 画出了完整的价值链，不是只看自己所在的环节
- [ ] 标注了每个环节的"完善度"状态和变化趋势
- [ ] 整合方向指向未来高利润区，而非当前高利润区
- [ ] 监控了破坏者从低端突破的信号

## Skill 5: RPV组织能力诊断

### 何时使用
为新业务选择组织形式、评估并购标的、判断组织能否执行某策略。

### 三个诊断问题

| 问题 | 若答案为"否" |
|------|-------------|
| 1. 我们有执行此任务所需的**资源**吗？ | 招聘/获取资源 |
| 2. 现有**流程**（工作方式）适合此任务吗？ | 创建重量级团队或独立组织 |
| 3. 我们的**价值观**（优先级标准）能让此事获得足够重视吗？ | 建立独立自治的业务部门 |

### 组织选择矩阵

```
            流程匹配度
            高          低
价值观  高  → 主流组织内     重量级团队
匹配度  低  → 独立部门       独立自治组织
```

### 关键决策规则
- **延续性创新** → 利用现有组织能力
- **破坏性创新** → 必须创建独立组织:
  - 独立的成本结构
  - 独立的决策流程
  - 独立的损益表
  - 不要求快速做大（规模与主流组织匹配）

### 选人指南（经验学校理论）
- 不看"胜任属性"（果断、善于沟通）
- 看**过去解决的问题类型**是否匹配新业务面临的挑战
- 问: "此人经历过什么'课程'？这些课程是否为新任务所需？"
- 从失败中学习的管理者往往比一路顺风的更可靠

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 用主流组织做破坏性创新 | 新业务挂在大部门下面 | 资源分配被优先级更高的主流业务吃掉 |
| 伪独立 | 组织独立但预算/人事仍受母公司控制 | 决策受制于母公司的价值观和赢利标准 |
| 选人看属性不看经验 | 招了"聪明、有领导力"的人 | 缺少应对新业务具体挑战的实战经验 |
| 独立组织做大后回并 | 破坏性业务成功后并入主流组织 | 回到母公司流程后丧失灵活性和斗志 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **资源迷信**: "我们有钱有人，肯定能做好" → 资源只是三个条件之一，流程和价值观更致命
- ❌ **文化万能论**: "只要CEO重视就行" → 价值观是组织的隐性决策规则，不是口号
- ❌ **猎头偏见**: 从大公司挖"明星经理人" → 他们的经验是在大公司流程中形成的，未必适合创业环境

### ✅ 检查点 (Checkpoints)
- [ ] 对新业务逐一评估了资源、流程、价值观三个维度
- [ ] 破坏性创新有独立组织、独立P&L、独立决策权
- [ ] 选人基于"经验学校"匹配度，而非简历光环
- [ ] 为独立组织设置了合理的规模预期（不要求快速做大）

## Skill 6: 策略流程匹配器

### 何时使用
决定新业务应采用谋划型（自上而下）还是应急型（自下而上）策略流程。

### 匹配规则

| 情境 | 策略流程 | CEO角色 |
|------|----------|---------|
| 延续性创新 | 谋划型 | 授权，不干预 |
| 破坏性创新初期（策略未知） | 应急型 | **亲自介入**，保护资源，管理探索 |
| 破坏性创新后期（策略已明） | 切换为谋划型 | 大胆执行，停止其他项目 |

### "以发现为主导"的规划（应急型策略加速器）

1. **先定目标财务指标**（利润率必须有吸引力）
2. **列出使数字成真的关键假设**（按重要性排序）:
   - 破坏性定位是否成立？
   - 目标客户会用产品完成什么任务？
   - 价值链定位是否能捕获未来利润？
3. **设计学习实验**测试假设
4. **验证后才投入**大量资源

### CEO的三个杠杆点
1. **控制初始成本结构** → 决定哪些客户有吸引力
2. **以发现为主导规划** → 加速试错学习
3. **判断每个业务用哪种策略流程** → 不让习惯/文化左右

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 早期用谋划型 | 破坏性初期就做详细5年计划 | 计划基于错误假设，团队照做浪费资源 |
| 应急期不切换 | 策略已验证但仍保持探索模式 | 错失执行窗口，竞争对手跟上 |
| CEO放手不管 | 对破坏性创新不亲自介入 | 资源被主流业务吸走，探索半途而废 |
| 假设不排序 | 所有假设同等对待 | 在低优先级假设上浪费时间，关键假设未验证 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **计划崇拜**: "没有计划就不是好管理" → 对未知市场做精确计划是自欺欺人
- ❌ **敏捷万能论**: "持续迭代就好" → 没有财务目标约束的迭代是无方向的探索
- ❌ **一刀切策略**: 所有业务用同一种策略流程 → 不同业务需要不同流程

### ✅ 检查点 (Checkpoints)
- [ ] 每个新业务都明确标注了"谋划型"还是"应急型"
- [ ] 应急型业务有清晰的假设清单和验证实验
- [ ] 验证通过后及时切换为谋划型执行
- [ ] CEO亲自参与了应急型业务的关键决策

## Skill 7: 资金好坏评估器

### 何时使用
为新业务选择融资来源，或评估是否接受某笔投资。

### 核心原则

```
好钱: 对成长有耐心 + 对赚钱没耐心
坏钱: 对赚钱有耐心 + 对成长没耐心
```

### 评估清单

| 检查项 | 好信号 | 坏信号 |
|--------|--------|--------|
| 对利润的态度 | 要求尽快赢利 | 允许长期亏损 |
| 对规模的态度 | 接受从小做起 | 要求快速做大 |
| 对市场的态度 | 接受零消费/小市场 | 要求进入最大市场 |
| 资金量 | 适度（刚够验证策略） | 过多（导致不珍惜） |

### 三条政策建议
1. **及早开始**: 核心业务健康时就播种，而非衰退时才反应
2. **保持小单位运营**: 大企业拆分为小业务单元，使破坏性机会有吸引力
3. **追求早期赢利**: 迫使团队快速验证策略，建立生存防线

### 坏钱螺旋（及早识别）
1. 企业成功 → 2. 成长缺口 → 3. 资金急于追成长 → 4. 忍受亏损 → 5. 损失攀升 → 回到3

> 识别信号: 当管理层说"我们得先赔几年钱"时，往往意味着破坏性技术正被硬塞进主流市场打延续性战争。

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| VC陷阱 | 接受大量VC资金，被迫追求高速增长 | 去打主流市场的延续性战争，烧完资金 |
| 母公司输血 | 亏损由母公司持续补贴 | 团队失去赢利压力，策略验证停滞 |
| 成长缺口驱动 | 核心业务增长放缓后才投资新业务 | 时间紧迫，被迫选坏钱、大市场 |
| IPO冲动 | 为上市做大收入规模 | 关注规模而非赢利，商业模式未验证 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **烧钱换规模**: "先亏损做大再赢利" → 破坏性创新的核心是找到可持续的小模型
- ❌ **资金充裕幻觉**: "钱多总比钱少好" → 钱多会降低决策纪律，允许低质量假设存活
- ❌ **只看金额不看性质**: 忽略资金背后的期望和约束 → VC的钱和自有资金有完全不同的压力

### ✅ 检查点 (Checkpoints)
- [ ] 资金来源的性质（好钱/坏钱）已明确评估
- [ ] 团队被要求在合理时间内展示赢利路径
- [ ] 资金量刚好够验证关键假设，不是"充裕的"
- [ ] 核心业务健康时就开始播种，不是衰退时才补救

## Skill 8: 品牌任务匹配法

### 何时使用
为破坏性产品设计品牌策略，或评估现有品牌是否适合新产品。

### 原则
- 品牌应定位在**任务**上，而非客户属性上
- 当客户身处特定情境时，能本能想到该品牌
- 能出色完成任务的品牌，客户愿付高价

### 破坏性产品品牌策略
- **创建针对性副品牌**: 专为特定任务打造
- **不要用主品牌直接覆盖**: 避免破坏性产品用错场景砸了品牌
- **案例**:
  - 柯达Funsaver（一次性相机）vs 柯达主品牌
  - 万怡酒店 vs 万豪酒店
  - 老海军 vs 盖普

### 品牌价值转移规律
- 产品"不够好"时 → 品牌在**产品**上（IBM主机品牌溢价30-40%）
- 产品"过于好"时 → 品牌转移至**子系统**（Intel Inside）和**渠道**（Talbot's、盖普）

### 🔴 失败模式
| 失败模式 | 表现 | 后果 |
|----------|------|------|
| 主品牌滥用 | 用主品牌覆盖破坏性产品 | 破坏性产品的低价定位拉低主品牌价值 |
| 品牌定位在人口属性 | "专为年轻人设计" | 无法在具体任务场景中被想起 |
| 忽略品牌转移 | 产品已货品化仍押注产品品牌 | 品牌投入无回报，利润已转移到子系统/渠道 |

### ⚠️ 反模式 (Anti-Patterns)
- ❌ **品牌延伸贪婪**: "品牌知名度高就能覆盖任何品类" → 品牌是任务联想，不是万能贴纸
- ❌ **只做广告不做任务**: 花大量预算做品牌推广但没匹配真实任务 → 知名度高但没人用
- ❌ **模仿大品牌策略**: 照搬大公司的品牌打法 → 大品牌的品牌资产是几十年积累的，你没有

### ✅ 检查点 (Checkpoints)
- [ ] 品牌名/定位能直接唤起客户的任务联想
- [ ] 破坏性产品使用了独立副品牌，没有和主品牌混淆
- [ ] 品牌价值转移规律已评估（产品→子系统→渠道）
- [ ] 品牌策略与"不够好—过于好"光谱位置一致

## 快速决策流程图

```
新构想进入
    ↓
通过三块试金石？ → 否 → 重新塑造或放弃
    ↓ 是
有"零消费"市场？ → 是 → 新市场破坏策略
    ↓ 否
有被过度服务的低端客户？ → 是 → 低端市场破坏策略
    ↓ 否
→ 此构想不适合破坏性创新，考虑延续性创新或放弃
    ↓
客户任务清晰？ → 细分市场 → 选择产品架构 → 选择组织形式
    ↓                              ↓
针对性品牌设计              评估资金类型 → 好钱才要
    ↓
以发现为主导的规划 → 验证假设 → 切换为谋划型执行
```
