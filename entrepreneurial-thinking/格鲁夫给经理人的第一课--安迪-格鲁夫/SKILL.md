# SKILL.md — 格鲁夫高产出管理技能卡

## 技能名称
格鲁夫高产出管理 (High Output Management)

## 一句话总结
用生产管理的杠杆率思维管理团队：找到高杠杆率活动，通过会议、决策、目标管理和绩效评估放大个人产出。

## 适用场景
- 中层管理者提升团队产出
- 建立会议体系（一对一、部门会、运营总结）
- 目标设定与追踪（OKR 原型）
- 绩效评估与反馈
- 组织架构设计（混血型/双重报告）
- 激励知识工作者
- 新经理培训

## 核心技能清单

### 1. 杠杆率思维
**做什么：** 识别并优先投入高杠杆率管理活动
**怎么做：**
- 列出当前所有管理活动
- 为每项活动评估杠杆率（高/中/低/负）
- 每周审视：高杠杆率活动占比是否增加？
- 砍掉或授权低杠杆率活动
- 警惕负杠杆率：愁云惨雾、举棋不定、上级干涉

**自检问题：** "如果我只能做一件事，哪件对组织产出影响最大？"

---

### 2. 一对一会议
**做什么：** 与每个直接下属定期进行结构化的一对一会议
**怎么做：**
- 频率：根据下属的 TRM 决定（低TRM=每周，高TRM=每月）
- 时长：最少1小时
- 地点：下属办公室或离他近的地方
- 议程：**下属负责准备**，含绩效指标、潜在问题、未来规划
- 上司角色：学生+教练，不是主讲人
- 秘诀：**再多问一个问题**，直到彼此"知无不言"
- 双方做笔记；建立"存档"留待下次讨论
- 每次会议结束前确定下次时间

**频率指标：** 1.5小时/两周/人 × 管理杠杆 ≈ 极高回报

---

### 3. 部门会议
**做什么：** 定期召开部门全员会议促进同事间互动
**怎么做：**
- 议程：涉及两个以上部属的事项
- 经理人角色：**协调者**（控制进度、化解纷争），禁止"一言堂"
- 有计划进行 + 保留开放讨论时间
- 两人之间的深入讨论 → 打断，建议私下再谈

---

### 4. 运营总结会议
**做什么：** 让不常合作的同事互动、分享经验
**怎么做：**
- 四个角色：召集人（提案人上司）、总结负责人（不预读资料）、提案人、与会者
- 视觉辅助：每4分钟用一张幻灯片
- 与会者必须参与发问，不许打瞌睡

---

### 5. 任务导向会议
**做什么：** 针对特定问题召开的决策会议
**怎么做：**
- 开会前先问：有没有必要？有没有替代方案？
- 主席在会前确认目的、参会人、议程
- 人数上限：6~7人（8人绝对上限）
- 发会议记录越快越好
- 目标：过程导向会议处理80%问题，任务导向会议≤20%

---

### 6. 目标管理（OKR 原型）
**做什么：** 设定短期目标 + 沿途验收成果
**怎么做：**
- 两个问题："我想去哪里？"（O）+ "我如何知道在前进？"（KR）
- 规划三步：预测需求 → 现状分析 → 缩小差距
- 目标短期（月/季），数量不超过3~5个
- 目标层层衔接：下属目标达成 → 上司目标达成
- 定期检视成果验收（用线性指标追踪进度）

**关键洞察：** 规划的产出是行动方案，不是文件本身

---

### 7. 决策制定
**做什么：** 建立结构化的决策流程
**怎么做：**
- 三阶段：自由讨论 → 清楚地决策 → 全力支持
- 决策前回答六个问题（内容、时限、决策人、咨询谁、决定权、告知谁）
- 决策权下放到**离问题最近的人**
- 同级群体会议需要"再加一"（高一级的人控制会议进度）
- 追求"不同意但可以接受"，不必强求共识
- 不要让决策早产，但也不要拖延

---

### 8. 绩效评估
**做什么：** 定期评估并反馈部属绩效
**怎么做：**

**评估阶段：**
- 兼顾产出评估（结果）和流程评估（活动/未来铺路）
- 警惕时间滞后：今年的漂亮数字可能是去年活动的结果
- 避免"潜力陷阱"：评估绩效而非潜力
- 评估经理人 = 个人贡献 + 部门整体表现

**告知阶段：**
- 三大原则：坦诚、倾听、忘了你自己
- 一次传达不超过3~4个关键信息
- 操作：自由列出所有事项 → 找关系 → 归纳信息 → 删减
- 书面报告在讨论**前**交给部属
- 冲突评估的关键：只要部属**愿意行动改进**就可接受，不必完全同意

**对明日巨星：** 多花时间提改进建议（高杠杆率）

---

### 9. TRM 感知与管理风格切换
**做什么：** 根据下属工作成熟度调整管理风格
**怎么做：**

| 判断 TRM | 管理风格 |
|----------|---------|
| 新任务/低经验 | 明确指示：做什么、何时、如何 |
| 有经验但不稳定 | 双向沟通、情绪支持、鼓励 |
| 熟练且可靠 | 方向确认即可，降低干涉频率 |

**自检：**
- TRM 针对**特定工作**，不是一般能力
- 环境剧变 → TRM骤降 → 管理风格须即时回调
- 不要以为"耳提面命"就是不好的管理方式
- 目标：尽快提升下属TRM → 授权 → 提高杠杆率

---

### 10. 激励与竞赛
**做什么：** 创造竞技环境，让自我实现成为驱动力
**怎么做：**
- 竞赛四要素：游戏规则 + 衡量指标 + 对手 + 跑道
- 目标设定：50%成功概率（高于一般能力）
- 金钱在高层需求中 = 衡量成就的标准，不是物质需求
- 经理人角色 = 教练：不居功、训练严格、了解规则

---

### 11. 培训
**做什么：** 经理人亲自负责团队培训
**怎么做：**
- 培训具有极高杠杆率（16小时投入 → 2万小时产出提升1%）
- 内容必须与公司实务紧密结合
- 系统化持续进行，不等火烧眉毛
- 先开短期课程（3~4堂），从最紧要项目开始
- 每堂课后收集匿名评价
- 第一堂课不会太成功——接受它，改进它
- 在前几堂课中培训出讲师替你代劳

## 失败模式详解

| 失败模式 | 根因 | 早期信号 | 纠正措施 |
|----------|------|----------|----------|
| 杠杆率倒挂 | 陷入低价值事务 | 忙碌但产出无增长 | 每周审计活动杠杆率，砍掉/授权低杠杆活动 |
| 决策瘫痪 | 追求完美共识 | 议题反复讨论无结论 | 采用"不同意但可接受"原则，明确决策人 |
| 信息黑洞 | 一对一敷衍或取消 | 下属问题堆积爆发 | 严格按TRM频率执行，下属准备议程 |
| 目标漂移 | OKR变成文件游戏 | 季末才发现目标未推进 | 建立线性指标，每月检视进度 |
| 评估失真 | 只看漂亮数字 | 高产出但团队疲惫/离职 | 兼顾产出+流程，警惕时间滞后 |
| TRM误判 | 把一般能力当工作成熟度 | 授权后频繁出错 | 针对具体任务评估TRM，而非整体印象 |
| 培训荒废 | 等火烧眉毛才培训 | 同类错误反复出现 | 系统化持续培训，经理人亲自上阵 |

## 检查点清单

### 日常检查（每周）
- [ ] 高杠杆率活动占比是否增加？
- [ ] 一对一会议是否按计划执行？
- [ ] 是否有决策在拖延？

### 月度检查
- [ ] OKR进度是否在线性轨道上？
- [ ] 下属TRM是否有变化需要调整管理风格？
- [ ] 部门会议是否促进了同事间互动？

### 季度检查
- [ ] 绩效评估是否兼顾产出和流程？
- [ ] 培训计划是否在执行？
- [ ] 决策权是否下放到离问题最近的人？

## 反模式（避免）

| 反模式 | 为什么有害 | 替代方案 |
|--------|-----------|----------|
| 上级干涉 | 剥夺下属执行机会，使其变成缩头乌龟 | 决策权下放到离问题最近的人 |
| 拖延决策 | 绿灯不亮=红灯，组织工作停顿 | "不同意但可接受"即可推进 |
| 情绪传染 | 经理人沮丧→全员士气崩塌 | 意识到情绪是高杠杆率活动 |
| 评估只看产出 | 忽视时间滞后，被漂亮数字迷惑 | 产出+流程双维度评估 |
| 评估只看潜力 | "表演"出的精明≠实际绩效 | 评估实际绩效而非潜力 |
| 培训外包 | 内容脱离公司实务，讲师无权威性 | 经理人亲自负责培训 |
| 授权无监督 | 等于渎职 | 授权≠放任，定期检视 |
| 目标过多 | 焦点放在每件事上=没有焦点 | 不超过3-5个目标 |
| 一言堂会议 | 扼杀同事间互动，信息单向流动 | 经理人做协调者，不做主讲人 |
| 一刀切管理 | 忽视TRM差异，低效或过度管理 | 针对特定工作评估TRM |

## 延伸影响
- **OKR 的直接源头**：John Doerr 在 Intel 亲历格鲁夫的目标管理体系，后将其发展为 OKR 框架推广至 Google 等公司
- **技术委员会 (Technical Board)**：格鲁夫描述的"同级群体"制造经理聚会制度，被台积电、联电等企业完整学习
- **知识管理的先驱**：Intel 在1983年前就推动制程创新配方上传至工程技术知识管理系统
