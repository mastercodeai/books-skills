# 精益创业：新创企业的成长思维

**作者**：埃里克·莱斯（Eric Ries）
**核心定位**：创业是一种可以习得的管理方式，而非天赋或运气。通过科学方法（假设→实验→验证→迭代）提高新创企业成功概率。

---

## 一、五项基本原则

1. **创业者无处不在** — 任何在极端不确定情况下开发新产品/服务的人都是创业者，不论企业规模、行业或部门
2. **创业即管理** — 新创企业需要一种新型管理方式，能应对极端不稳定
3. **经证实的认知（Validated Learning）** — 新创企业的存在是为了学习如何建立可持续业务，这种学习必须有实证支撑
4. **开发—测量—认知（Build-Measure-Learn）** — 核心反馈循环，所有活动应以加速此循环为目标
5. **创新核算（Innovation Accounting）** — 新创企业需要全新的核算体系来衡量进展、建立责任制

---

## 二、核心概念详解

### 1. 经证实的认知
- 不是失败后的借口，而是衡量新创企业进展的严格方式
- 必须以真实顾客的实证数据为基础，而非主观判断
- 新创企业的真正生产力 = 开发了多少经证实的认知（而非开发了多少功能）
- 例：IMVU早期6个月看似浪费了大量代码，但获得的认知将公司引向成功道路

### 2. 开发—测量—认知反馈循环
- **核心引擎**：新创企业把点子转化为产品 → 衡量顾客反馈 → 认知应转型还是坚持
- 计划顺序与执行顺序相反：先确定需要知道什么 → 需要衡量什么 → 需要开发什么
- 关键是缩短循环总时间，而非优化某一环节

### 3. 最小化可行产品（MVP）
- **定义**：用最少精力、最短时间完成一次完整的开发—测量—认知循环的产品版本
- 目的不是回答设计或技术问题，而是验证基本商业假设
- MVP形式多样：视频式（Dropbox）、贵宾式（Food on the Table）、人工后台式（Aardvark的Wizard of Oz测试）、冒烟测试
- **关键原则**：放弃对你需要的认知没有直接用处的一切功能、流程或努力
- 质量原则：如果不知道谁是顾客，就不知道什么是质量

### 4. 信念飞跃（Leap of Faith）
- 每个商业计划中风险最大的假设，企业成败系于此
- **两大核心假设**：
  - **价值假设**：产品是否真正为顾客创造了价值
  - **增长假设**：新顾客如何发现并使用产品
- 需要用实验而非分析来验证

### 5. 创新核算三步走
1. **确定基准线**：用MVP获取真实数据（转化率、注册率、留存率等）
2. **调整引擎**：每次产品/营销活动都应提升增长模式的某个驱动因素
3. **转型还是坚持**：无法推动驱动因素 → 说明需要转型

### 6. 虚荣指标 vs 可执行指标
- **虚荣指标**（总注册数、总点击量）→ 展示美好假象，无法指导决策
- **可执行指标**（同期群分析、对比测试）→ 清晰显示因果关系
- **三个"可"**：可执行（清晰因果）、可使用（简单易懂、广泛分发）、可审查（数据可信可查证）

### 7. 同期群分析
- 不看累积总量，而是按每组顾客分别评估行为表现
- 每个同期群 = 一张独立的产品成绩报告
- 是新创企业分析中最重要的工具之一

## 三、转型（Pivot）

### 什么是转型
- 有条理的方向性改变，测试新产品/战略/增长引擎的基础假设
- 不是盲目改变，而是基于已知认知的战略调整
- 转型需要勇气：虚荣指标遮蔽真相、缺乏清晰假设导致无法经历完整失败、恐惧心理

### 转型加速度
- 每次MVP开发时间应越来越短（Votizen：8个月→4个月→3个月→1个月）
- 加速来自对顾客、市场和战略的累积认知，而非已开发的功能
- **跑道重新定义**：不是现金÷月消耗，而是剩余的转型次数

### 十种转型类型
| 类型 | 说明 |
|------|------|
| 放大转型 | 原产品的一个功能成为全部 |
| 缩小转型 | 原产品成为更大产品中的一项功能 |
| 客户细分市场转型 | 产品对路但顾客群不对 |
| 客户需求转型 | 发现顾客真正需要解决的是另一个问题 |
| 平台转型 | 应用产品↔平台产品之间转换 |
| 商业架构转型 | 高利润低产量↔低利润高产量 |
| 价值获取转型 | 改变货币化/收入模式 |
| 增长引擎转型 | 病毒式/黏着式/付费式之间转换 |
| 渠道转型 | 改变产品交付顾客的途径 |
| 技术转型 | 用不同技术获得相同解决方案 |

---

## 四、三种增长引擎

| 引擎类型 | 核心驱动 | 关键指标 | 代表案例 |
|----------|---------|---------|---------|
| **黏着式** | 高顾客保留率 > 流失率 | 复合增长率 = 获客率 - 流失率 | 数据库软件、收藏品平台 |
| **病毒式** | 顾客使用产品自然带动传播 | 病毒系数（>1.0则几何级增长） | Hotmail、Facebook |
| **付费式** | 顾客生命周期价值(LTV) > 每取得成本(CPA) | 边际利润决定引擎转速 | IMVU、传统零售 |

- 建议每次专注一种增长引擎
- 每种引擎最终都会耗尽燃料，需提前培育新增长来源

## 五、小批量的力量

- **单件流优于大批量**：装信封实验——一次封装一个比全部折好再封更快
- 好处：更早发现问题、减少返工、降低在制品存货、更快获取反馈
- IMVU实践：每天平均50次产品改动，持续部署
- **大批量死亡螺旋**：每次推进批量 → 更多返工和延误 → 试图用更大批量降低间接成本 → 批量无限增长
- **拉动而非推动**：由对顾客的假设拉动产品开发，而非顾客需求直接拉动

## 六、自适应组织

### "五个为什么"（5 Whys）
- 丰田大野耐一创立的系统化问题解决工具
- 反复问五次"为什么"，从表面技术问题追溯到根本人为原因
- **按比例投入**：问题轻则投入少，问题重则投入多
- 作用：天然的速度调节器——问题越多越投入防范，防范见效后重新加速
- 避免"五大罪状"：目标是发现系统缺陷，而非指责个人

### 创新沙盒
- 为创新团队划定受控实验空间
- 规则：限定实验范围/时间/顾客数量，使用统一可执行指标
- 既保护上级组织不受冲击，也保护创新团队不受干扰

## 七、创新管理组合

企业同时管理四类工作：
1. **创新孵化** → 颠覆式新产品开发
2. **增长扩张** → 攻占新市场、建立公众形象
3. **优化运营** → 商品化、产品线扩充、降低成本
4. **基础维护** → 外包、自动化、基础设施保障

- 创新团队架构三要素：稀少但稳定的资源、独立开发权、与绩效挂钩的个人利益
- "创业企业家"应是一个正式的职业头衔

## 八、关键心智模型

| 概念 | 核心要义 |
|------|---------|
| 现地现物（Genchi Gembutsu） | 亲自观察，基于第一手信息做决策 |
| 走出办公大楼 | 顾客信息只存在于办公大楼之外 |
| 早期使用者 | 接受不完美产品的特殊顾客群，不需要完美解决方案 |
| 零的愚勇 | 零收入反而比低收入更容易筹资（有想象空间），但延迟数据收集造成巨大浪费 |
| 产品/市场契合 | 增长引擎概念可将其量化评估 |
| 持续部署 | 小批量、快速迭代、自动化检测，类似丰田"安灯"拉绳 |

---

## 九、一句话总结

> **创业不是天赋，而是一门可以科学管理的学科。核心方法：将愿景分解为可测试的假设，用最小化可行产品快速实验，通过经证实的认知而非虚荣指标衡量进展，在数据驱动下决定转型或坚持，最终建立可持续的业务。**
