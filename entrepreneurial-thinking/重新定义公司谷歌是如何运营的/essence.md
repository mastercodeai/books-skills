# 重新定义公司：谷歌是如何运营的 (How Google Works)

**作者**: Eric Schmidt, Jonathan Rosenberg, Alan Eagle
**出版**: 2015 (中信出版社)
**类型**: 企业管理 / 科技公司运营

---

## 核心论点

互联网时代的企业成功之道：聚集创意精英（Smart Creative），营造合适氛围和支持环境，充分发挥创造力，快速感知客户需求。传统的管理理念不适用于创意精英，甚至适得其反。

---

## 一、创意精英 (Smart Creative)

创意精英是互联网时代取得成功的关键员工群体，他们具备以下特征：

- **专业知识 + 实践经验**：不仅是理论家，还能设计概念并建造模型
- **分析头脑**：善用数据做决策，但不被数据牵着走
- **商业头脑**：理解技术、产品与商业成功的关联
- **竞争头脑**：追求卓越，实干积累
- **用户头脑**：自己就是超级用户，身先士卒试用产品
- **好奇心**：总在提问，不满足于守常不变
- **冒险精神**：不惧失败，能从失败中学习
- **自动自发**：不会坐等别人指方向

> 创意精英无处不在——每个城市、每所高校、每个班级中都有。

## 二、文化：相信自己的口号

### 2.1 文化建设原则

- **创始人是文化源头**：创始人为实现大计而物色的团队是企业文化的最佳体现
- **真实性**：使命宣言必须真实反映企业行为，否则毫无意义（反面案例：雷曼兄弟）
- **言出必行**：员工的"测谎仪"已被磨炼得异常灵敏
- **不作恶**：谷歌的核心口号，赋予员工道德指针，允许任何员工在发现不道德行为时拉绳中止

### 2.2 办公环境

- **拥挤出成绩**：办公室应鼓励交流，不要制造阻隔、强调地位
- **杂乱是美德**：混乱往往是自我表达和创新的衍生品
- **不同职能整合**：产品经理与工程师应一同吃住、并肩工作
- **在家办公是瘟疫**：面对面的偶遇机会不可替代（AdSense 就是打桌球时的灵感）

### 2.3 "河马"问题

- **河马 = Highest-Paid Person's Opinion（高薪人士的意见）**
- 薪金高低与决策能力完全无关
- 应营造"提议不问出处"的环境
- 必须创造强调"质疑"的文化，将提出疑义作为硬性规定

### 2.4 组织结构

- **7的法则**：每位管理者至少7份直接报告，保持扁平
- **按职能划分部门**：避免以业务线为基础造成"各成一家"
- **贝佐斯的两个比萨原则**：团队规模不能大到两个比萨还吃不饱
- **以最有影响力的人物为中心**：不看头衔和经验，看表现和热情
- **驱逐恶棍，保护明星**：恶棍的恶行是人品不端，明星的行为是出类拔萃

### 2.5 说"好"的文化

- 管理者条件反射般说"不"是创意精英的大敌
- 添加流程或增设门槛的前提条件一定要严格
- YouTube 高清重播功能案例：说"好"让几百万用户提前几周享受到便利

## 三、战略：你的计划是错误的

### 3.1 核心战略原则

> 如果你有商业计划，那你的计划一定是错误的。——但基础原理必须正确。

1. **信赖技术洞见，而非市场调查**
   - 技术洞见 = 用创新方式应用科技，达到成本显著降低或功能大幅提升
   - 所有成功产品背后都有技术洞见（AdWords、Chrome、Google News）
   - 失败产品往往缺少技术洞见支撑（iGoogle、Wave、Knol）

2. **为成长而优化，而非为收入**
   - 全球扩张必须成为基础核心
   - 平台思维 > 产品思维
   - 谷歌早期选择提升搜索引擎而非急着赚钱

3. **开放为王**
   - 开放有助于吸收创意、降低成本、加速扩张
   - 但封闭有时也有道理（如搜索算法保密）
   - 用户应享受来去自如的自由

4. **不被竞争对手牵着鼻子走**
   - 紧盯竞争对手 = 渐渐落入平庸
   - 竞争者可以促使你前进，但不要追随

### 3.2 组合创新

当今信息、连接、计算能力三大要素丰沛，创新者可以：
- 将可用科技及数据集中起来解决问题
- 从具体问题的解决方案出发进行拓展（SafeSearch → 图像搜索）
- 找到极客，看他们在研究什么

## 四、人才：招聘是你最重要的工作

### 4.1 招聘理念

- **招聘是管理者最重要的工作**，不是HR的活
- **羊群效应**：A级人才招A级人才，B级人才招BCD级人才
- **同事评估制度**：摒弃层级制，通过委员会定夺
- **宁缺毋滥**：宁可漏聘，也不误聘

### 4.2 寻找什么样的人

- **有激情的人不把"激情"挂嘴边**：激情在心中、在行动中
- **雇用学习型动物**：看重潜力而非已有知识，拥有"成长型思维模式"
- **机场测试**：被困机场6小时，你能和此人开心聊天吗？
- **谷歌范儿（Googleyness）**：上进心、团队精神、服务精神、创造力、品行
- **客观评价人才**：不因性别、种族、肤色而有偏见

### 4.3 面试原则

- **面试是最重要的商业技能**
- 面试时间设为30分钟（不够可以再约）
- 每位应聘者至多5位面试官（4次之后增量价值 <1%）
- 面试官必须有自己的明确立场：行，或不行
- **四大评估维度**：领导力、职务相关知识、一般认知能力、谷歌范儿
- **加大光圈甄才**：不局限于当前职位匹配，考虑未来发展潜力

### 4.4 招聘委员会

- 录用决定权不在用人部门经理手中
- 委员会以数据为根据做决策
- 信息包 = 流通货币，必须全面、规范、以事实支撑观点
- 每份录用决定都需经过拉里审核

### 4.5 留住人才

- 用新的想法保持工作趣味性
- 为最有价值的员工创造突破机会
- **换出巧克力，留下葡萄干**：管理者应把最好的员工轮换出来
- 薪酬策略：起点放低，表现越突出回报越高

## 五、决策：共识的真正含义

### 5.1 决策原则

- **共识 ≠ 一致同意**：共同达成对公司最有利的决策，并围绕决策共同努力
- **需要异议**：如果所有人一致点头，说明下面坐了一群"摇头娃娃"
- **用数据做决策**：不以"我觉得"服人，而以"请看数据"服人
- **该响铃时就响铃**：设立最后期限，按时完成决策
- **少做决策**：CEO应将决定权交给别人，只在重大问题上亲自出马

### 5.2 会议管理

- 会议应有一位决策者或主持
- 与会者不超过8人，10人已算上限
- 会议应很容易取消（如果目标不明确）
- 开会时就认真开会（关闭电脑和手机）

### 5.3 马背原则

- 不是每次决策都需要50页法律声明
- 快速环视四周，然后继续上路

## 六、沟通：当最牛的路由器

### 6.1 信息共享

- **"预设模式"应是与人共享一切**
- 董事会报告与全体员工分享
- OKR在公司内公开透明
- **"摘要"（Snippets）工具**：个人每周动态，全公司共享

### 6.2 沟通原则

- 沟通能强化核心理念
- 沟通要包含新鲜内容
- 沟通要有趣、鼓舞人心
- 沟通要发自肺腑
- **祷文不会因重复而失色**：一件事需要重复约20遍才能被真正听进去

### 6.3 安全环境

- 为讲真话营造安全的环境
- TGIF大会 + "多莉"（Dory）匿名提问机制
- **"爬升—报告—遵从"模式**：有人带坏消息来找你时，用心倾听、竭力相助

## 七、创新：缔造原始的混沌

### 7.1 创新定义

创新 = 新颖 + 出人意料 + 非常实用

### 7.2 创新环境

- **首席执行官必须兼任首席创新官**
- 创新不可把握、无法强制、不能事先安排
- 营造让创意因素自由碰撞的"原始混沌"环境
- **第一追随者原则**：将一个孤独的疯子变成领袖的，就是第一个追随者

### 7.3 聚焦用户

> "聚焦用户，一切水到渠成。"

- 为终端用户服务是第一要务
- 用户的信赖与任何货币一样重要
- 如果客户利益与用户利益冲突，以用户为重

### 7.4 往大处想

- "把想法放大10倍"（10x thinking）
- 挑战越大，越能吸引顶尖人才
- 较大的问题通常也较容易解决（因为必须从头开始）

### 7.5 OKR 系统

**目标与关键成果 (Objectives and Key Results)**：

1. **完备的OKR** = 大"目标" + 易于衡量的"关键成果"
2. **合理的OKR应有一定难度**：完成70%的好OKR优于完成100%的漏洞OKR
3. **几乎人人可用**：撇开职位差异
4. **分数不作他用**：唯一的用途是让员工诚实评判自己
5. **并非包罗一切**：只用于需要特别关注的领域
6. **避免被竞争者牵着鼻子走**

### 7.6 70/20/10 资源配置原则

- **70%** 给核心业务
- **20%** 给新兴产品
- **10%** 给全新产品（高风险高回报）

### 7.7 20% 时间制

- 允许工程师拿出20%的时间研究自己喜欢的项目
- 重点在于**自由**，而不在时间长短
- 产物：Google Suggest、Google News、Google Maps交通信息等
- 最宝贵之处：产生更多精干的创意精英

### 7.8 交付与迭代

- **完美是优秀的敌人**：产品应尽早交付市场
- 交付后根据数据不断改进
- 负面评价是有效的迭代激励
- 不要在产品上市时就大做营销——先让它凭实力飞起来

### 7.9 败得漂亮

- 失败的项目也会衍生宝贵信息
- 不要拿失败的团队问罪
- **速战速决**：一旦发现没有前途，以最快速度喊停
- **反脆弱**：经受失败和冲击后变得越来越强大

## 八、数据驱动决策

贯穿全书的核心理念：

- **"请看数据"而非"我觉得"**
- 多数会议室配两台投影仪：一台视频会议，一台投射数据
- 最了解数据的是第一线的员工，不是管理层
- 数据不针对个人，有利于广开言路
- 每个OKR都必须有可衡量的关键成果
- 以数据评判产品质量，而非以沉没成本决定去留

## 十、实操清单

### 对管理者的建议
- [ ] 每位管理者至少7份直接报告
- [ ] 招聘是最重要工作，亲自参与面试
- [ ] 用OKR设定（近乎）遥不可及的目标
- [ ] 每季度公开自己的OKR
- [ ] 营造说"好"的文化
- [ ] 用数据而非直觉做决策
- [ ] 每天与团队沟通核心理念
- [ ] 为员工创造20%时间项目空间
- [ ] 建立招聘委员会制度
- [ ] 速战速决地处理组织中的"恶棍"

### 对创业者的建议
- [ ] 先确定企业文化，再招聘
- [ ] 以技术洞见而非市场调查为产品基础
- [ ] 为成长而非收入优化
- [ ] 选择大市场，不要找无人问津的市场
- [ ] 交付、迭代，不要等到完美
- [ ] 失败要快，期限要长
- [ ] 聚焦用户，赚钱水到渠成
