## 核心方法论

### 一、互联网七字诀（小米方法论基石）

**专注 · 极致 · 口碑 · 快**

#### 专注
- **做什么**：清晰使命愿景 → 深刻洞察 → 明确目标 → 克制贪婪
- **不做什么**：决定不做什么跟决定做什么一样重要（乔布斯）
- **关键原则**：少就是多；单点突破，逐步放大；核心业务一元化，不存在二元
- **校验三问**：①我的业务增长为公司核心战略贡献了什么价值？②是否带动了其他业务连带增长？③消耗了公司哪些资源？

#### 极致
- **定义**：做到自己能力的极限，做到别人做不到的高度
- **两个层面**：心智上无限投入 + 无限追求最优解
- **三个自检**：①是不是用户真正需要的？②能不能成为核心竞争力？③能不能形成长期竞争壁垒？
- **警惕**：极致不是自嗨，不是自我感动，不是秀情怀

#### 口碑
- **核心公式**：只有超过预期的产品才能带来口碑
- **口碑第一，销量第二**：口碑总是领先销量一步
- **做全栈口碑**：产品、服务、沟通——与用户所有触点的总和
- **三条验证原则**：多路交叉验证；区分核心用户（关注意见点）和大众用户（关注印象面）；直面真实口碑

#### 快
- **四种能力**：洞察快、响应快、决策快、改善快
- **辩证**：局部的慢才是全局的快。战略积累快不得，战术演进慢不得
- **警惕**：不要用战术上的勤奋掩盖战略上的懒惰

### 二、铁人三项商业模式

```
硬件（爆品） + 新零售（高效交付） + 互联网服务（"小费"盈利）
```

- **硬件**：紧贴成本定价，硬件综合净利润率永远不超过5%
- **新零售**：自有电商+小米之家，目标对标电商效率的线下零售
- **互联网服务**：应用分发、游戏、阅读、云服务等提供利润

**增长飞轮**：好产品 → 用户口碑 → 流量 → 更多产品 → 更多用户 → 更大生态

---

### 三、爆品模式

**爆品定义**：产品力超群，具有一流口碑，最终实现海量长销。四个特征：单款、精品、海量、长周期。

**三个产生条件**：
1. 高效的工业化大生产
2. 迎合大众消费能力的定价
3. 产品体验导向的创新

**四项关键能力**：
1. **洞察未来**：产品要有"明天属性"——用过就不想放手
2. **洞察用户**：精准取舍功能，满足80%用户的80%需求
3. **创新实现**：重组技术和供应链资源
4. **精准触达**：直达目标用户

**小白模式**：代入普通用户视野，不从专家角度出发。回到第一性原理思考。

---

### 四、效率模型

**核心矛盾**：好 ≠ 贵，便宜 ≠ 差。效率提升是解决这对矛盾的唯一出路。

**定价五要素**（都是分摊成本）：
1. 制造和服务分摊成本
2. 研发分摊成本
3. 市场推广及广告分摊成本
4. 销售及渠道分摊成本
5. 利润

**小米策略**：
- 爆品模式 → 降低全链条分摊成本
- 新媒体+新零售 → 降低推广和渠道成本
- 自我控制利润 → 硬件净利润率不超5%

**ROI公式**：
```
投资回报率 =（毛利率 – 费用率）× 年周转次数
```

**关键指标**：小米综合运营成本（OPEX）长期约10%，世界级水平。

## 实操指南

### A. 如何做产品（爆品思维）

1. **找到一个明确且用户迫切需要的产品**
   - 一句话说清楚解决什么问题
   - 选择有普遍性的需求
   - 切口越小，压强越大

2. **追求最优解，不是"还行"**
   - 每个品类都存在当下的最优解
   - 从第一性原理出发，不照搬既有经验
   - 哪怕只好1%，也愿意多投入100%

3. **定价策略**
   - 紧贴成本定价（BOM定价）
   - 性价比 ≠ 低价，是比较优势
   - 令人惊喜的定价可以是有理由的低价或有理由的高价

4. **持续迭代**
   - 产品卖出去才是跟用户关系的开始
   - 改改改改再改改

### B. 如何与用户建立关系

1. **全员泡在用户里**
   - 从CEO到基层，直接倾听用户声音
   - 每个管理者的手机里都有不同类型用户的联系方式

2. **听意见就要有行动**
   - 一旦确认用户意见有价值，马上就改
   - 没有行动的倾听比不听更糟糕

3. **新媒体是沟通阵地，不是营销工具**
   - 用户在哪里就去哪里
   - 偶尔说错话是"必要代价"，不能因此放弃沟通

4. **让口碑自传播、自转化**
   - 好产品 + 超预期 + 真诚 = 自来水
   - 不要急于做营销，先服务好一批核心用户

### C. 如何做效率

1. **减少中间环节**
   - 尽可能压缩商业流通领域的中间环节
   - 小米之家：厂家到用户只有一层零售商

2. **用爆品分摊固定成本**
   - 单款、海量、长周期
   - 研发成本被海量出货摊薄到极致

3. **数字化全链路**
   - 交易即数据
   - 实时数据驱动业务决策

4. **周转为王**
   - 高周转可以弥补低毛利
   - 小米之家年周转17次，ROI达30%

### D. 如何建设组织

1. **找人第一**：三十顾茅庐的诚意
2. **工程师文化**：包容失败、鼓励创新、相信工程师、放手让工程师干
3. **使命驱动**：上下同欲者胜
4. **OKR对表**：全员参与、逐级分解、互相协同
5. **品牌建设是CEO工程**：品牌战略 = 技术产品战略 = 用户经营战略

## 失败模式与反模式

### 🚨 五大失败模式

| # | 失败模式 | 表现 | 根因 | 解法 |
|---|----------|------|------|------|
| 1 | **伪专注** | 同时推进5+条业务线，每条都"重要" | 贪婪 + 不敢做减法 | 回答"校验三问"，砍到只剩1-2个核心 |
| 2 | **自嗨式极致** | 团队感动自己，用户无感 | 闭门造车，脱离用户 | 用"小白模式"检验：普通用户会为此买单吗？ |
| 3 | **口碑断裂** | 营销声量大，但复购率低/差评多 | 产品力撑不住承诺 | 先把产品做到超预期，再谈营销 |
| 4 | **效率假象** | 砍成本砍到产品力下降 | 混淆"降成本"和"提效率" | 效率提升靠创新和模式优化，不是偷工减料 |
| 5 | **组织膨胀** | 人均产出下降，官僚化 | 追求规模而非效能 | OPEX红线 + 定期审视组织效率 |

### ⛔ 反模式清单（看到就该警觉）

| 反模式 | 正确做法 |
|--------|----------|
| 先想怎么赚钱，再想做什么产品 | 先想怎么创造价值，盈利是结果 |
| 用补贴换用户 → 期待后续转化 | 用产品超预期换口碑 → 用户自来 |
| 产品线越铺越广 = "生态" | 生态是能力延伸，不是盲目扩张 |
| 高端 = 高价 + 高毛利 | 高端 = 高品质 + 高效率，不一定高毛利 |
| 听用户的话做产品 | 听用户的痛，用专业能力给方案 |
| 战略变来变去 = "灵活" | 战略要定力，战术要灵活 |
| "我们跟XX不一样" = 创新 | 创新要有用户可感知的价值差异 |
| KPI达标 = 做得好 | 口碑和复购才是终极KPI |

## 决策检查点（Checkpoints）

### 产品决策检查点

```
□ 一句话能说清楚这个产品解决什么问题？
□ 目标用户群是否足够大（大众需求 vs 小众需求）？
□ 是否找到了当下品类的"最优解"？
□ 定价是否紧贴成本？性价比是否成立？
□ 有没有"明天属性"（用过就回不去）？
□ 功能取舍是否满足了80%用户的80%需求？
```

### 用户运营检查点

```
□ 团队里有多少人直接接触过真实用户？
□ 最近一次根据用户反馈做了什么改变？
□ 核心用户的NPS（净推荐值）是多少？
□ 用户是否在自发传播你的产品？
□ 有没有把"用户满意"和"用户口碑"混为一谈？
```

### 效率检查点

```
□ OPEX（综合运营成本）是多少？目标<15%
□ 硬件综合净利润率是否控制在5%以内？
□ 库存周转次数是否在提升？
□ 中间环节是否压缩到了最少？
□ 是否有数字化全链路的数据支撑决策？
```

### 战略检查点

```
□ 增长飞轮是否在转？转速是否在加快？
□ 新业务是否为公司核心战略贡献了价值？
□ 品牌战略、技术战略、用户战略是否"三位一体"？
□ 是否在用战术勤奋掩盖战略懒惰？
□ 当前最大的效率瓶颈在哪里？
```

### 组织检查点

```
□ 关键岗位是否找到了最合适的人（而非"还行"的人）？
□ 工程师是否有足够的自主权和容错空间？
□ OKR是否做到了全员参与、逐级分解？
□ 组织是否在膨胀？人均产出趋势如何？
```

## 可迁移的通用原则

| 原则 | 含义 | 适用场景 |
|------|------|----------|
| 效率是一切的基石 | 商业的本质是提升效率 | 所有行业 |
| 超越预期才有口碑 | 好不够，要比预期好 | 产品/服务设计 |
| 少就是多 | 克制贪婪，专注核心 | 产品线管理、战略规划 |
| 分摊思维 | 固定成本被海量出货摊薄 | 定价策略、商业模型 |
| 第一性原理 | 从基本原理出发，不类比 | 创新、问题解决 |
| 用户参与共创 | 用户不是旁观者，是参与者 | 产品开发、社区运营 |
| 品牌 = 信任积累 | "闭着眼睛买"是终极追求 | 品牌战略 |
| 投资工程师 = 投资未来 | 技术是长期竞争力之源 | 研发投入决策 |

## 思考框架（可用于分析任何企业）

```
1. 这家公司的效率如何？OPEX是多少？
2. 它的产品是否有"明天属性"？是否超越用户预期？
3. 它与用户的关系是博弈还是朋友？
4. 它的技术积累是否形成了体系？
5. 它的增长飞轮是否成立？正循环还是恶性循环？
6. 它的品牌战略、技术战略、用户战略是否"三位一体"？
```
