# 从0到1 — 思维技能手册

## 技能总览

本书提供了一套完整的创业与创新思维框架，核心是：**用独立思考发现秘密，用垄断思维创造价值，用明确计划塑造未来。**

## 场景路由（Scene Routing）

遇到以下情境时，直接跳转对应技能：

| 场景 | 主技能 | 辅助技能 |
|------|--------|----------|
| "大家都这么做，我要不要跟？" | → 技能一：逆向思维 | 技能四：秘密发现 |
| "我要创业/评估一个商业模式" | → 技能二：垄断构建 | 技能六：创业基础 |
| "如何分配时间、精力、资金？" | → 技能三：幂次法则 | — |
| "找什么方向/为什么没人做这个？" | → 技能四：秘密发现 | 技能一：逆向思维 |
| "战略规划/要不要做计划？" | → 技能五：未来四象限 | 技能七：销售策略 |
| "组建团队/治理架构" | → 技能六：创业基础 | — |
| "产品怎么卖出去？" | → 技能七：销售策略 | 技能二：垄断构建 |
| "AI/自动化怎么设计？" | → 技能八：人机互补 | — |
| "全面评估一个创业想法" | → 七问检验法 | 全部技能 |

## 技能一：逆向思维提问法

### 触发条件
- 面临战略决策、市场判断、人生方向选择时
- 需要评估一个想法是否有价值时

### 操作步骤

1. **问自己**："在什么重要问题上我与其他人有不同看法？"
2. **检验答案质量**：
   - ❌ 差答案：很多人已经认同的观点（"教育体制有问题"）
   - ❌ 差答案：常见辩论中一方的观点（"不存在上帝"）
   - ✅ 好答案模式："大多数人相信X，但事实却是X的对立面"
3. **从反面思考**：识别大众观点中哪些是基于恐惧、惯性或错误推理
4. **验证逻辑**：你的逆向观点是否能被证据支持，而非仅为叛逆

### 反面教材
- 泡沫后的4条"教训"（循序渐进、保持灵活、在改进中竞争、专注产品）的对立面可能更正确：
  - 大胆尝试胜过平庸保守
  - 坏计划也好过没有计划
  - 竞争性市场很难赚钱
  - 营销和产品同样重要

### 失败模式
- **为反而反**：逆向观点没有逻辑支撑，纯粹为了标新立异
- **选错战场**：在无关紧要的问题上逆向思考，浪费认知资源
- **确认偏误**：找到逆向观点后只寻找支持证据，忽略反面证据
- **孤立无援**：逆向观点正确但无法说服任何人，无法落地执行

### 反模式
- ❌ "所有专家都错了" → 专家共识在大多数情况下是对的，你需要解释**为什么这次不同**
- ❌ 把逆向思维等同于悲观主义 → 逆向思维可以是"别人看到风险，我看到机会"
- ❌ 仅凭直觉不做验证 → 逆向观点需要比常规观点**更强**的证据

### 检查点
- [ ] 你的逆向观点能否用一句话清晰表达？
- [ ] 你能否列出至少3条支持证据？
- [ ] 你能否诚实地列出反对你的最强论据，并解释为什么它不成立？
- [ ] 如果这个逆向观点是对的，它是否能产生实际价值（而非仅仅是知识性洞察）？

## 技能二：垄断构建框架

### 触发条件
- 创建新公司或评估商业模式时
- 思考竞争优势来源时

### 操作步骤

**第一步：从小市场开始**
- 目标市场要小到你能成为明确的领导者
- "如果你认为自己起步的市场可能太大，那就一定是太大了"
- 不是找不存在的市场，而是找到一小群确实需要你产品的人

**第二步：检验10倍改进**
- 你的产品比最接近的替代品好10倍吗？
- 三种实现路径：
  - 创造全新事物（如消灭睡眠的药物）
  - 彻底改进已有事物（如PayPal vs 邮寄支票）
  - 出众的综合设计（如iPad vs 之前所有平板）

**第三步：建立四重壁垒**
- [ ] 专利技术（比替代品好10倍）
- [ ] 网络效应（产品随用户增加而更有价值）
- [ ] 规模经济（边际成本趋近于零）
- [ ] 品牌优势（建立在实质优势之上）

**第四步：逐步扩大**
- 主导小市场 → 扩展到相邻市场 → 达成长远目标
- 案例：亚马逊（书→CD/DVD→全品类→AWS→一切）

### 注意事项
- 不要搞"破坏"——那意味着你透过旧企业的眼光看自己
- 不要把自己定义为已有公司的敌对者
- 先发优势不重要，后发垄断才重要

### 失败模式
- **伪垄断**：自认为垄断但市场定义过窄（"我是XX市最好的瑜伽教练"→市场太小无意义）
- **过早扩张**：小市场还没垄断就冲向大市场，两头落空
- **壁垒幻觉**：以为有专利/品牌就安全，实际上壁垒不够深
- **竞争否认**：声称"我们没有竞争对手"，实际是没看清竞争格局

### 反模式
- ❌ 从第一天就想做"平台" → 平台是垄断后的结果，不是起点
- ❌ 追风口 → 风口意味着拥挤竞争，与垄断背道而驰
- ❌ 把融资额当壁垒 → 钱不是护城河，别人也能融到钱
- ❌ 用"先发优势"安慰自己 → 先发者常被更高效的后来者取代

### 检查点
- [ ] 你能用一句话定义你垄断的那个小市场吗？
- [ ] 你的10倍改进是客观可衡量的，还是自说自话？
- [ ] 四重壁垒中你至少有1项是强项？（2项以上更佳）
- [ ] 你有清晰的从小市场到大市场的扩展路径吗？
- [ ] 如果一个资金10倍于你的团队进入同一市场，你还能赢吗？

## 技能三：幂次法则应用

### 触发条件
- 做投资决策时
- 分配时间、精力、资源时
- 选择职业方向时

### 操作步骤

1. **认识到分布不均匀**：
   - 不是正态分布（中间最多），而是幂律分布（极端集中）
   - 最好的一个 > 其余所有之和

2. **投资思维**：
   - 不要撒大网，集中押注少数真正有潜力的标的
   - 问自己：这个选择是否有成为"那一个"的可能？

3. **人生决策**：
   - 你不能把自己多元化——同时运营十几家公司或保留十几种职业
   - 找到你最擅长且未来最有价值的那一件事
   - "你做什么并不重要"是彻头彻尾的错误

4. **公司运营**：
   - 一个市场可能胜过所有其他市场
   - 一种分销策略通常优于所有其他策略
   - 某些关键时刻远比其他时刻重要

### 失败模式
- **假集中**：口头上说集中，实际上同时追5个方向
- **错配**：把精力集中在"安全"选项上，而非"幂次最高"的选项上
- **沉没成本陷阱**：在一个明显不是"那一个"的方向上持续投入，因为已经投了很多
- **过早放弃**：幂次需要时间显现，在拐点到来前就退出

### 反模式
- ❌ "分散风险"的本能 → 在幂律世界里，分散意味着平庸
- ❌ 用正态分布思维做决策 → "中等努力会有中等回报"在幂律世界不成立
- ❌ 忙碌≠高效 → 10件平庸的事不如1件关键的事
- ❌ 模仿成功者的"多元化" → 他们是在一个方向成功后才多元化的

### 检查点
- [ ] 你现在最重要的那一件事是什么？能明确说出来吗？
- [ ] 你的时间/精力分配是否与"那一件事"匹配？（看日历，不是看愿望）
- [ ] 如果你必须放弃现在做的80%的事情，剩下20%是什么？
- [ ] 你投入的方向是否有成为"赢家通吃"的潜力？

## 技能四：秘密发现法

### 触发条件
- 寻找创业方向时
- 评估一个领域是否有价值时
- 试图理解为什么某些事情被忽视时

### 操作步骤

1. **区分秘密与无解之谜**：
   - 秘密 = 重要但无人知晓，困难但可为
   - 无解之谜 = 不可能完成的任务

2. **寻找秘密的两个方向**：
   - **自然的秘密**：自然没有告诉你的（物理学、生物学未解问题）
   - **人的秘密**：人类隐藏的或回避的（被禁止谈论的话题、禁忌）

3. **探索秘密的最佳地点**：
   - 无人关注的领域
   - 未被标准化和制度化的领域
   - 营养学 vs 天文学——重要但缺乏学术声望的领域

4. **验证秘密**：
   - 问：开公司的人忌讳说什么？（垄断者隐藏垄断，竞争者夸大独特性）
   - 问：人们把什么视为理所当然但实际上可能是错的？

5. **保守秘密**：
   - 发现秘密后，告诉越少人越好
   - 但与你的创业团队分享——他们成为你的"谋士"
   - 公司 = 在秘密基础上建立的密谋

### 失败模式
- **把秘密当幻想**：没有验证机制，纯粹是自己的臆想
- **秘密太小**：发现了对的事情，但市场太小不值得做
- **泄密**：在社交媒体/会议上炫耀自己的洞察，引来竞争
- **秘密过期**：发现得太晚，市场已经有人在做了

### 反模式
- ❌ 只在热门领域找秘密 → 热门领域意味着很多人已经在找了
- ❌ 把"别人不做"等同于"秘密" → 有时候别人不做是因为没价值
- ❌ 等待"完美秘密" → 秘密不需要是颠覆性的，小秘密也能构建好生意
- ❌ 独自探索太久 → 秘密需要在真实世界中验证，闭门造车容易走偏

### 检查点
- [ ] 你能区分你的"秘密"是真正的洞察还是一厢情愿吗？（有验证数据吗？）
- [ ] 如果这个秘密是真的，它能支撑一个多大的生意？
- [ ] 有多少人知道这个秘密？知道的人越少越好
- [ ] 这个秘密为什么现在才成立？（时机因素）
- [ ] 你有把秘密转化为产品的具体路径吗？

## 技能五：未来四象限诊断

### 触发条件
- 评估个人/企业/国家的战略方向时
- 决定是否需要制定具体计划时

### 四种未来观

|  | 乐观 | 悲观 |
|--|------|------|
| **明确** | 有计划，未来更好（17世纪-1960年代西方） | 知道未来不好，提前准备（当代中国） |
| **不明确** | 知道未来更好，但无计划（1982年后美国） | 未来不好，束手无策（当代欧洲） |

### 操作步骤

1. **诊断自己处于哪个象限**
2. **如果不明确乐观**→ 这是最不可持续的状态
3. **转变为明确乐观**：
   - 制定具体、可执行的计划
   - 不要依赖"精益创业"式的无限迭代——精益是方法，不是目标
   - 苹果案例：乔布斯从创立起就规划未来，而非听用户反馈

4. **对自己说**："你不是一张彩票。"

### 失败模式
- **计划僵化**：制定计划后完全不调整，无视环境变化
- **计划空洞**：有"愿景"但没有具体可执行的步骤
- **象限错判**：自认为"明确乐观"，实际上是"不明确乐观"（用计划掩盖不确定性）
- **悲观瘫痪**：诊断为悲观后陷入无力感，不做任何行动

### 反模式
- ❌ 用"拥抱不确定性"当借口不做计划 → 不确定性需要更强的计划，而非没有计划
- ❌ 精益创业万能论 → 无限迭代是缺乏远见的表现，不是方法论
- ❌ 只做年度计划 → 真正的明确乐观需要10-20年的长远视角
- ❌ 模仿大公司的规划方式 → 创业公司需要创始人愿景，不是委员会共识

### 检查点
- [ ] 你能否写下你/公司5年后、10年后的具体状态？
- [ ] 你的计划是否包含可衡量的里程碑？
- [ ] 你是否知道下一步要做什么？（如果不知道→象限诊断有问题）
- [ ] 你的计划是基于你自己的判断，还是基于市场共识？

## 技能六：创业基础检查清单

### 触发条件
- 正在创业或即将创业
- 审视现有公司的治理结构时

### 检查清单

**合伙人选择：**
- [ ] 创业前已有深厚交情
- [ ] 技术能力互补
- [ ] 合作默契度高
- [ ] 不是"在赌场老虎机前遇见的第一个人"

**治理结构：**
- [ ] 明确所有权、经营权、控制权
- [ ] 董事会3人（最多5人，上市前）
- [ ] 没有利益冲突的董事

**薪酬设计：**
- [ ] CEO年薪不超过15万美元
- [ ] 用股权而非高薪激励
- [ ] 现金奖励设置上限
- [ ] 股权分配细节保密

**文化建设：**
- [ ] 每人只负责一件事
- [ ] 全职投入（无兼职、远程等模糊地带）
- [ ] 招聘基于使命和团队匹配，非简历最优
- [ ] 介于邪教和咨询公司之间

### 失败模式
- **合伙人撕裂**：没有提前约定退出机制，分手时公司崩溃
- **股权均分**：三个创始人各33%→无人有决定权→决策瘫痪
- **治理真空**：早期不设规矩，等到出问题时已经来不及
- **文化稀释**：快速招人时降低标准，团队文化从"使命驱动"变成"打工心态"

### 反模式
- ❌ 找"明星合伙人" → 能力匹配和信任比履历重要
- ❌ 用高薪吸引人才 → 高薪来的人会因更高薪离开
- ❌ 模仿硅谷福利（免费午餐、游戏桌）→ 文化≠福利，使命才是核心
- ❌ 过早引入职业经理人 → 早期需要使命驱动的创始人心态，不是管理技巧

### 检查点
- [ ] 你和合伙人是否讨论过"如果公司失败/成功后怎么办"？
- [ ] 股权结构是否确保了清晰的决策权？
- [ ] 你的团队中每个人是否都知道公司的核心使命？
- [ ] 你最近一次招人，是基于简历还是基于对使命的认同？

## 技能七：销售策略选择

### 触发条件
- 设计产品分销策略时
- 决定如何触达客户时

### 决策树

```
你的产品单价是多少？
│
├─ >$1,000,000 → 复杂销售
│  └─ CEO亲自卖，每年成交几笔
│  └─ 参考：SpaceX（NASA合同）、Palantir
│
├─ $10,000-$1,000,000 → 人员销售
│  └─ 建销售团队和流程
│  └─ 参考：Box（从斯坦福睡眠诊所起步）
│
├─ $100-$10,000 → 营销盲区
│  └─ 广告太散，销售太贵
│  └─ 这是最难卖的价位——需要找到特殊渠道
│
├─ <$100 → 市场营销/广告
│  └─ 电视、社交媒体、内容营销
│  └─ 参考：宝洁、Warby Parker
│
└─ 产品自带传播 → 病毒式营销
   └─ 每个用户带来>1个新用户
   └─ 参考：PayPal（付钱注册+推荐返利）、Facebook
```

**核心原则**：销售的幂次法则——找到一条最有效的渠道，而非同时尝试多种。

### 失败模式
- **营销盲区困局**：产品定价在$100-$10,000区间，进退两难
- **渠道错配**：用市场营销的方式卖高价产品，或用复杂销售的方式卖低价产品
- **忽视销售成本**：只看收入不看获客成本，卖得越多亏得越多
- **病毒系数幻觉**：以为产品会自然传播，实际上病毒系数<1

### 反模式
- ❌ "好产品会自己说话" → 产品再好也需要分销策略
- ❌ 同时尝试5种渠道 → 幂次法则：找到最有效的1种，全力投入
- ❌ 只关注获客不关注留存 → 留存是增长的基础
- ❌ 低估销售的价值 → Thiel的原话："销售和产品一样重要"

### 检查点
- [ ] 你的产品单价落在决策树的哪个区间？
- [ ] 你是否明确了主要分销渠道？（只能有1个主力）
- [ ] 你的获客成本是否低于客户终身价值的1/3？
- [ ] 如果你的主力渠道明天失效，你有备选方案吗？

## 技能八：人机互补设计

### 触发条件
- 设计产品或系统时
- 评估AI/自动化策略时
- 思考技术如何增强人类能力时

### 操作步骤

1. **不要用技术替代人类**，而是增强人类
2. **识别任务类型**：
   - 人类擅长：复杂情境判断、计划、决策、理解人
   - 计算机擅长：海量数据处理、模式识别、重复计算
3. **设计混合系统**：
   - 计算机处理数据→标记可疑结果→人类审核决策
   - 案例：PayPal反欺诈系统"Igor"（算法标记+人工审核）
   - 案例：Palantir（软件分析+分析师判断）
4. **避免大数据迷信**：
   - 数据本身是沉默的
   - 只有人类分析师能给出可行见解
   - 最有价值的公司是帮助人类解决问题，而非用计算机替代人类

### 失败模式
- **全自动化陷阱**：试图用算法替代所有人类判断，结果系统脆弱
- **数据迷信**：收集大量数据却不知道要回答什么问题
- **人机错配**：让计算机做人擅长的事（创意），让人做计算机擅长的事（重复计算）
- **技术炫技**：为了用AI而用AI，没有解决真实问题

### 反模式
- ❌ "AI会取代一切" → 最有价值的是人机协作，不是替代
- ❌ 数据越多越好 → 无方向的数据收集是浪费资源
- ❌ 追求完全自动化 → 保留人类判断的"人在回路"设计更稳健
- ❌ 忽视人机界面 → 技术再好，如果人类无法有效使用就是废物

### 检查点
- [ ] 你的系统中哪些决策留给了人类？为什么？
- [ ] 你是否明确了"人擅长什么、机器擅长什么"的边界？
- [ ] 你的数据收集是否服务于一个明确的问题？
- [ ] 如果AI判断错误，人类是否有能力和时间介入纠正？

## 综合决策框架：七问检验法

面对任何创业/商业决策，依次回答：

1. **工程**：你的技术有突破性（10倍改进）而非仅稍有改进？
2. **时机**：现在开创事业，时机合适吗？
3. **垄断**：是在一个小市场抢占大份额吗？
4. **人员**：你有合适的团队吗？
5. **销售**：除了创造产品，你有办法销售产品吗？
6. **持久**：未来10-20年，你能保住市场地位吗？
7. **秘密**：你发现了其他人没有发现的独特机会吗？

**7个全答是 → 必然成功**
**5-6个是 → 很可能成功**
**<5个是 → 需要重新审视**

### 七问检验的失败模式
- **自我欺骗**：每个问题都答"是"但缺乏客观依据
- **跳过问题**：只回答自己有把握的，回避困难问题
- **静态评估**：只评估现在，不考虑1-2年后的变化
- **权重错误**：7个问题不是等权的——秘密（7）和垄断（3）是核心

### 综合检查点
- [ ] 让一个诚实的朋友/导师独立回答这7个问题，与你的答案对比
- [ ] 对于每个"是"，你能否提供一个客观证据？
- [ ] 最弱的那1-2个问题是什么？你有改进计划吗？
- [ ] 如果7个问题中有3个以上答"否"，是否应该放弃或大幅调整方向？
