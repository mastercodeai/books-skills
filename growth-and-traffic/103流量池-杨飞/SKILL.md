## 一、品牌定位速查

### 选择定位方法的决策树

```
市场是否已饱和/有明确头部竞品？
├── 是 → 对立型定位（"更/没有/不是…而是…"）
│        例：神州"更安全的专车"、瓜子"没有中间商赚差价"
└── 否 → 产品是否有单一强功能？
         ├── 是 → USP定位（"…就用…"场景口号）
         │        例：OPPO"充电5分钟通话2小时"
         └── 否 → 升维定位（"开创者/重新定义/革命"）
                  例：小米"互联网电视"
```

### 口号检验清单
- [ ] 能否一句话/几个字说清楚？
- [ ] 能否朗朗上口、说人话？
- [ ] 电梯20秒内能否说清楚？
- [ ] 是否弱化了广告腔？
- [ ] 是否可念出声（不仅可看）？

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 口号记不住、传不开 | 超过10字、有生僻词或英文 | 砍到7字以内，用日常口语 |
| 定位与用户无关 | 只讲企业愿景不讲用户利益 | 改为"你能得到什么"句式 |
| 对标竞品但无人认知 | 选的对手太小众或太泛 | 选品类第一联想品牌作对手 |
| 升维定位变成空话 | "智能""赋能"等虚词堆砌 | 必须落到可感知的场景或功能 |

### 🚫 反模式
- ❌ "我们什么都好" → 定位必须有取舍，只打一个点
- ❌ 老板喜好代替用户调研 → 定位前做50人以上用户语义联想测试
- ❌ 一年换三次定位 → 定位至少坚持6个月才能建立认知
- ❌ 口号只有内部人懂 → 让非行业朋友试读，3秒不懂就重写

### ✅ 自检点
- [ ] 你的定位能否用"我是___，与___不同，我是___"填空？
- [ ] 员工能否在不看资料的情况下准确复述品牌定位？
- [ ] 定位是否同时回答了"选我不选别人的理由"？

## 二、品牌符号打造清单

### 视觉符号
| 维度 | 要求 | 反面教材 |
|------|------|----------|
| LOGO | 纯文字或简化图形，识别度>艺术性 | 花哨字体，辨识困难 |
| 主题色 | 80%以上纯色，抢占单一色彩联想 | 多色混搭，无记忆点 |
| 包装 | 特殊造型/主题色/辅助图形三选一强化 | 千篇一律无特色 |
| 代言 | 代言人+品牌专属造型（区别于其他代言） | 用明星但无强关联 |

### 听觉符号
- **口号**：说人话，有场景感，可口头传播。
- **Jingle**：在App提示音、客服等待音、广告结尾设置品牌声音锚点。

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 视觉符号无统一感 | 各渠道各做一套视觉 | 制定品牌视觉规范手册并强制执行 |
| 代言人抢了品牌风头 | 代言人形象过于强势 | 设计品牌专属造型/台词绑定 |
| 投入大量设计费但无记忆点 | 追求艺术性忽视识别度 | 用"5秒遮住LOGO测试"验证 |

### 🚫 反模式
- ❌ LOGO频繁改版 → 视觉符号至少稳定2年
- ❌ 跟风用渐变色/3D风 → 色彩风格应服务于品类心智而非流行趋势
- ❌ 线上线下视觉不统一 → 所有触点用同一套设计源文件

### ✅ 自检点
- [ ] 遮住LOGO，消费者能否通过颜色/形状/声音认出你？
- [ ] 品牌视觉在手机端缩略图状态下仍可辨识？
- [ ] 所有渠道的视觉符号是否100%一致？

## 三、场景营销落地法

### 场景切割四步法
1. **数据分析**：从用户行为数据中发现高频场景（如神州发现接送机订单占比高）
2. **利基切入**：避开主战场，选择一个窗口型场景集中突破
3. **产品配套**：开发独立功能入口（如神州App的"接送机""孕妈专车"按钮）
4. **补贴聚焦**：将补贴集中在这个场景而非全面撒钱

### 神州6场景模板（可迁移）
```
核心场景（高频刚需） → 接送机、会务用车
情绪场景（痛点驱动） → 夜晚加班"放心睡"、孕妇出行
长尾场景（品牌强化） → 带子出行、异地出差
```

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 场景切了但无人用 | 场景是伪需求/频率太低 | 用数据验证：该场景月订单≥总订单5%才值得切 |
| 场景补贴一停用户就走 | 只靠补贴无产品壁垒 | 场景必须配套专属功能/服务，形成使用习惯 |
| 场景太碎、资源分散 | 同时切3个以上场景 | 集中资源打透1个场景再扩第2个 |

### 🚫 反模式
- ❌ 凭直觉选场景 → 必须有用户行为数据支撑
- ❌ 场景只有营销没有产品配套 → 每个场景至少需要1个独立功能入口
- ❌ 撒钱式补贴覆盖全场景 → 补贴预算的80%聚焦主力场景

### ✅ 自检点
- [ ] 选中的场景是否有数据证明用户自然行为中已存在？
- [ ] 场景是否有独立的产品入口/功能模块？
- [ ] 补停后该场景的自然留存率是否>30%？

## 四、裂变营销操作手册

### 裂变系统搭建等式
```
流量裂变 = 平台 + 创意 + 福利 + 技术
```

### 选择裂变方式的决策矩阵

| 产品特征 | 推荐裂变方式 | 案例 |
|----------|------------|------|
| 高频+App | 拉新奖励 + 裂变红包 + IP裂变 | 神州专车"邀请有礼" |
| 高频+微信生态 | 分销海报 + 微信卡券 + 礼品卡 | 神州U+司机招募海报 |
| 低客单+社交电商 | 团购裂变 | 拼多多拼团 |
| 虚拟/高单价 | 个体福利裂变 | 喜马拉雅"分享免费听" |
| 有线下产品 | 包装裂变 + O2O红包 | 味全拼字瓶、青岛啤酒一瓶一码 |
| 低频产品 | 转化为高频福利/强福利刺激 | 保险赠险裂变、房产转介绍费 |

### 种子用户筛选标准
1. 活跃度高、影响力大（宁少勿滥）
2. 与产品调性吻合
3. 愿意反馈产品建议

### 裂变海报设计公式
```
一张海报 = 吸引眼球的头图 + 核心利益点 + 专属二维码 + 紧迫感/限时
```

### 防微信封号注意
- 页面不能出现"分享有福利"等诱导分享描述
- 分销层级不超过二级（超过定义为传销）
- 现金红包用技术手段防刷

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 裂变启动后无人转发 | 福利不够吸引/海报无传播力 | 福利价值需让用户觉得"不转亏了" |
| 传播链断裂（只裂一代） | 缺乏二级激励机制 | 设置阶梯奖励：邀请3人/10人/30人不同档 |
| 羊毛党刷量但无真实用户 | 福利门槛过低、无风控 | 增加实名/首单/活跃度等解锁条件 |
| 活动被微信封禁 | 诱导分享话术/三级分销 | 用"邀请好友一起享"替代"分享得XX" |
| 裂变来了用户但留不住 | 产品承接力不足 | 裂变落地页必须是产品核心体验页 |

### 🚫 反模式
- ❌ 只看拉新数不看留存 → 裂变用户7日留存率低于自然用户50%则活动无效
- ❌ 一次裂变用到底 → 每月迭代裂变玩法，避免用户审美疲劳
- ❌ 裂变福利与产品价值脱节 → 福利应是产品体验的一部分（如免费课程>现金红包）
- ❌ 不做A/B测试直接全量上线 → 至少测试海报2版、文案3版

### ✅ 自检点
- [ ] 裂变系数K值是否>1？（1个用户平均带来>1个新用户）
- [ ] 裂变用户的7日留存是否≥自然用户的50%？
- [ ] 是否配置了反作弊机制（设备指纹/IP限频/行为验证）？
- [ ] 裂变路径是否≤3步完成？（看到→参与→分享）

## 五、微信运营实战

### 服务号 vs 订阅号选择
```
资源有限？
├── 是 → 只做服务号（月4次精品推送，直达首屏）
└── 否 → 服务号为主 + 订阅号发紧急/临时信息
```

### 推送内容公式
```
10万+ = 创意驱动 + 利益驱动 + 技术驱动
       有趣好玩儿   福利/发券   H5/小程序/裂变技术
```

### 花式发券模板（神州实践）
- 热点借势发券（女排夺冠、发现新地球、影视剧热播）
- 节日节点发券（七夕、十一开工）
- IP联名发券（达康书记券、加勒比海盗券、哈根达斯券）

### 模板消息提频场景
1. 新用户注册 → 即时推送礼券
2. 会员卡绑定 → 增强留存
3. 积分变动 → 持续互动
4. 消费提醒 → 增强黏性
5. 卡券到期 → 降低流失
6. 配送状态 → 关联消费行为

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 公众号打开率持续下降 | 推送无差异化/频率过高 | 服务号≤4次/月，每次必有明确福利或价值 |
| 粉丝多但转化极低 | 内容与产品脱节、无转化路径 | 每篇内容底部必须有明确CTA按钮 |
| 模板消息被用户投诉骚扰 | 推送频率过高/内容无价值 | 同一用户同一类型消息≤1次/周 |
| 发券核销率<5% | 券面额过低/使用门槛过高 | 券价值需≥客单价的15%，门槛尽量低 |

### 🚫 反模式
- ❌ 把公众号当广告板 → 内容价值:促销信息 ≥ 7:3
- ❌ 模板消息用来群发营销 → 模板消息只能用于交易/服务通知
- ❌ 不看数据凭感觉推送 → 必须分析每篇打开率、分享率、取关率
- ❌ 所有粉丝收到同样内容 → 用标签分组做差异化推送

### ✅ 自检点
- [ ] 最近3篇推送的打开率是否≥5%？（行业均值约3-5%）
- [ ] 每篇推送是否有且仅有1个明确的CTA？
- [ ] 粉丝标签分组是否覆盖≥3个维度（活跃度/消费层级/兴趣）？
- [ ] 模板消息的退订率是否<1%？

## 六、事件营销"轻快爆"检查表

### 轻（内容轻量）
- [ ] 制作周期是否≤14天？
- [ ] 是否选择海报/短视频等轻量形式？
- [ ] 是否避免了过度复杂的H5/视频制作？

### 快（反应速度）
- [ ] 是否能在24-48小时内上线？
- [ ] 是否提前储备了热点模板？
- [ ] 竞争型话题是否选在周四（竞品来不及周五反应）？

### 爆（5个爆点检查）
- [ ] **热点**：是否借势了当下热点？
- [ ] **爆点**：是否有5字以内关键词 + 视觉符号？
- [ ] **卖点**：是否紧扣产品核心利益？
- [ ] **槽点**：是否埋设了可吐槽方向（控制引导）？
- [ ] **节点**：是否选在周二至周四？

### 转化检查
- [ ] 事件是否与品牌强关联（而非只火事件不火品牌）？
- [ ] 是否有即时转化路径（领券/下载/购买）？

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 事件火了但品牌没被记住 | 品牌植入太弱/与事件无关 | 事件创意必须从品牌核心价值出发 |
| 热点追上了但没水花 | 追得太慢（>48h）或角度太平 | 建立热点响应SOP，2小时内出创意 |
| 被舆论反噬 | 触碰敏感话题/消费灾难 | 设"红线清单"：政治/灾难/民族/性别一律不碰 |
| 制作复杂导致错过窗口 | 追求完美错过热点黄金期 | 用"60分速度优先"原则：先发后优化 |

### 🚫 反模式
- ❌ 所有热点都追 → 只追与品牌调性匹配的热点，不匹配就不出手
- ❌ 事件营销无转化设计 → 必须在事件中埋入产品入口
- ❌ 负面热点也蹭 → 负面事件只做公益回应，不做营销
- ❌ 一人决策无审核 → 事件营销内容必须经PR/法务双重审核

### ✅ 自检点
- [ ] 目标受众看到事件后，能否3秒内联想到你的品牌？
- [ ] 事件海报/视频去掉品牌LOGO后，是否仍能看出是你的品牌？
- [ ] 是否有完整的舆情监控和应急预案？

## 七、数字广告投放技能

### 三端数据打通清单
```
媒体端：展现量、点击量、点击率、关键词消费
    ↓ 对比验证
网站端：PV、UV、跳出率、停留时间
    ↓ 对比验证
销售端：留资量、潜客量、订单量、成单量
```

### KPI设定原则
- ❌ 不要只考核展现量/点击量/留资量（易造假）
- ✅ 重点考核潜客量/订单量/成单量（造假难度大）

### SEM优化四步法
1. 趋势分析：关注波峰波谷，记录消费/转化量/转化成本
2. 关键词更换：精准词+长尾词为主，结合热点定期更换
3. 优化激活/拉新成本：多维度（时段/设备/地区）分析
4. 持续优化落地页

### 信息流广告投放三要素
1. 标题：极强吸引力，欢迎"标题党"
2. 配图：精美+生活化，原生感强（不像广告）
3. 核心卖点：一条文案一个卖点

### 微信朋友圈广告选择
| 目标 | 选择 | 门槛 | 特点 |
|------|------|------|------|
| 线下门店引流 | LBS定投 | 日预算1000元起 | 3-5km精准，七分效果三分品牌 |
| 品牌大曝光 | 排期品牌广告 | 5万元起 | 视频/图文，七分品牌三分效果 |

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 广告消费高但转化低 | 关键词太宽泛/落地页不匹配 | 关键词精确匹配+落地页与广告文案一致 |
| CPA持续上涨 | 素材疲劳/竞争加剧 | 每周更换素材，监控竞品出价 |
| 三端数据对不上 | UTM参数丢失/跨域追踪失败 | 统一归因模型，每日核查数据一致性 |
| 信息流CTR高但转化低 | 标题党过度、用户预期不符 | 标题承诺必须在落地页首屏兑现 |
| 朋友圈广告ROI<1 | 定向过窄或过宽 | 用DMP排除已转化用户，精准定向核心人群 |

### 🚫 反模式
- ❌ 只看CTR不看后端转化 → 必须追踪到订单/成单
- ❌ 一套素材用一个月 → 信息流素材7天必换，SEM创意14天必换
- ❌ 投放与落地页团队割裂 → 投放和落地页优化必须同一团队负责
- ❌ 不做否定关键词 → SEM每周至少增加10个否定词

### ✅ 自检点
- [ ] 媒体端点击数与网站端UV差异是否<15%？
- [ ] 是否建立了从展现→点击→访问→留资→成单的完整漏斗？
- [ ] 过去7天是否更换过广告素材？
- [ ] 否定关键词列表是否≥100个？

## 八、落地页优化技能

### 六要素检查清单
1. [ ] 首屏是否有品牌+核心卖点（"我是谁，能提供什么"）？
2. [ ] 是否有口碑/从众信息（"已有×万人使用"）？
3. [ ] 是否从消费者益处出发（非规格参数）？
4. [ ] 是否有权威认证/社会证言？
5. [ ] 留资按钮是否在醒目位置（首屏+中部+底部多处）？
6. [ ] 索取信息是否最少化（仅手机号/微信）？

### 文案优化速查
| 错误做法 | 正确做法 |
|----------|----------|
| 客户视角（"简约时尚欧美风格橱柜"） | 用户视角（"我的厨房还能空出这么多空间"） |
| 罗列多个卖点 | 一条文案一个卖点 |
| 专业术语（"韩式半永久培训"） | 通俗易懂（"美容培训"） |
| 直接给答案（"不穿便宜货"） | 留悬念（"女人过了20岁这样穿"） |
| 情感绑架（店员跪地试鞋） | 激发主观动机（橱窗里写着你名字的鞋） |

### 神州车闪贷落地页迭代经验
- 第1版→第8版：电话咨询量增加330%。
- 关键改动：GIF跳动按钮、电话放主视觉上方、ICON化卖点、二维码位置提升。

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 高跳出率（>70%） | 首屏无核心信息/加载太慢 | 首屏3秒内展示品牌+核心利益点，页面<3秒加载 |
| 留资多但成单少 | 留资门槛太低导致无效线索 | 增加筛选条件（如预算/时间/需求） |
| A/B测试无结论 | 样本量不够/变量太多 | 每次只改一个变量，至少1000UV后再判断 |
| 广告承诺落地页不兑现 | 创意团队与落地页团队脱节 | 广告主推卖点=落地页首屏内容 |

### 🚫 反模式
- ❌ 落地页塞满所有卖点 → 每个落地页只聚焦1个核心转化目标
- ❌ 用PC端设计直接套移动端 → 移动端必须独立设计，按钮≥44px
- ❌ 落地页几个月不更新 → 每月至少做一次数据复盘+优化迭代
- ❌ 只优化页面不优化表单 → 表单每多一个字段，转化率降10-15%

### ✅ 自检点
- [ ] 落地页首屏加载时间是否<3秒？
- [ ] 转化按钮在不滚动的情况下是否可见？
- [ ] 过去30天是否做过至少1次A/B测试？
- [ ] 广告文案承诺是否在落地页首屏得到兑现？

### Benefit

### Technology
- [ ] 拍摄/灯光/连麦/导播设备是否就绪？
- [ ] 弹幕管理/浮层特效是否配置？

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 直播间人少 | 预热不足/时间选错 | 提前3天预热，选目标用户活跃时段 |
| 观看多但下单少 | 产品讲解不到位/价格无优势 | 每30分钟做一次限时秒杀+产品演示 |
| 主播翻车/口误 | 无脚本/无彩排 | 必须完整彩排1次+准备应急话术卡 |
| 直播后无长尾效果 | 无二次传播素材 | 直播切片3-5条短视频二次分发 |

### 🚫 反模式
- ❌ 纯靠明星流量无内容设计 → 直播本质是内容节目，不是明星见面会
- ❌ 只在直播间卖货 → 直播前中后需配合全域传播矩阵
- ❌ 不复盘就做下一场 → 每场直播后必须做数据复盘（峰值/低谷/转化节点）

### ✅ 自检点
- [ ] 是否有完整的脚本+时间轴（精确到分钟）？
- [ ] 直播切片素材是否提前规划好二次分发渠道？
- [ ] 是否做了至少1次完整彩排？

## 十、BD跨界合作技能

### 合作对象筛选
```
数据匹配：双方用户画像是否有重叠？
品牌匹配：调性是否一致？（门当户对）
场景匹配：是否有天然结合点？
跨度越大，化学反应越好。
```

### 自身可交换资源盘点
| 类型 | 资源位 |
|------|--------|
| 曝光类 | App banner、官网、微博、车身展示、车内iBeacon、户外广告 |
| 转化类 | 会员短信、微信推送、App push、裂变红包页面 |

### BD合作递进路径
```
联合创意 → 内容/平台合作 → 产品/技术合作 → 订单交易合作
（海报/活动）  （快闪店/API接入）  （预装App/联名产品）  （联名卡/采购收入）
```

### 两类关键BD资源
- **娱乐资源**：减少传播成本（IP授权、影视合作、明星资源置换）
- **金融资源**：增加销售收益（联名卡、银行采购、支付合作）

### ⚠️ 失败模式
| 症状 | 根因 | 修复 |
|------|------|------|
| 合作雷声大雨点小 | 只做PR不做效果追踪 | 合作方案必须有量化KPI和数据回收机制 |
| 合作方用户无转化 | 用户画像不匹配/合作浮于表面 | 做一次联合用户画像分析再启动合作 |
| BD周期太长错过窗口 | 决策链条过长/合同条款纠结 | 设30天BD推进deadline，超时换下一个 |
| 资源置换不对等 | 未提前盘点自身筹码 | 制作资源价值评估表，按曝光/转化/成本量化 |

### 🚫 反模式
- ❌ 只找大品牌不看匹配度 → 匹配度>品牌量级
- ❌ 合作只有PR稿无实际落地 → 每次合作至少有一个用户可感知的互动
- ❌ 不签效果协议 → 合作必须有明确的效果对赌或资源对等条款

### ✅ 自检点
- [ ] 合作双方的用户画像重叠度是否≥30%？
- [ ] 合作方案是否有明确的量化目标和数据回收计划？
- [ ] 是否盘点了己方可交换资源的市场价值？

## 附：跨场景常见陷阱速查

| 陷阱 | 出现场景 | 预警信号 | 应对 |
|------|----------|----------|------|
| 品效脱节 | 品牌+广告+裂变 | 品牌投入大但无法追踪转化 | 每个品牌动作必须埋转化钩子 |
| 数据孤岛 | 广告+落地页+CRM | 三端数据对不上 | 建立统一用户ID体系 |
| 玩法疲劳 | 裂变+微信+事件 | 同一玩法效果逐月衰减 | 每月迭代1次玩法，建立玩法库 |
| 资源分散 | 全场景 | 同时推进5个以上项目 | 每季度只聚焦2-3个核心场景 |
| 团队割裂 | 广告+落地页+CRM | 各部门各管各的数据 | 设立增长负责人统管全链路 |
