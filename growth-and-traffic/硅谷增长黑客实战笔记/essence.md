## 核心思想

增长不是零散技巧，而是一套**以数据为指引、以实验为方法**的体系，贯穿用户生命周期的获取→激活→留存→推荐→变现→回流，通过快速迭代实验在每个阶段寻找性价比最高的杠杆点。

增长三要素：**实验 + 流程 + 文化**。

## 第1章 增长黑客旅程

### 增长黑客的起源
- 2010年 Sean Ellis 在招聘继任者时发现传统市场VP不适用，创造了"增长黑客"一词
- 核心区别：早期创业公司只关注一件事——**增长**
- 增长产品经理 = 产品 × 营销的交界点，用产品手段达到营销目标

### 每周增长例会流程（1小时）
1. **15分钟** 看指标：北极星指标和聚焦领域指标变化趋势
2. **15分钟** 回顾上周实验进展，扫清障碍
3. **15分钟** 分析已结束的实验结果（最爱的部分）
4. **10分钟** 决定下周实验（目标每周上线≥3个实验）
5. **5分钟** 检查备选实验想法库（保持500+想法）

### 最初90天计划
- **第1个月**：熟悉工具数据、确定增长指标、找到聚焦领域、上线第一批实验
- **第2个月**：扩大实验规模、开始建立流程
- **第3个月**：完善流程、争取组建独立团队

### "增长第一炮"案例
- 问题：新用户"定期投资"功能渗透率低
- 实验1：文案A/B测试——量化积少成多效果（+40%）和社交证明（+60%）
- 实验2：改变功能位置（再翻倍）
- **一个月，两个实验，工程师不到两天，定期投资率提升数倍**

### 关键洞察
- "增长没有银弹"——即使是增长黑客之父也没有万能灵药
- "任何事情都是一个实验，你赢或者学到经验"
- 改善留存最有力的手段：对产品进行"疯狂"的改善

## 第2章 制定增长作战计划

### 增长作战计划四要素
1. **方向标**：北极星指标
2. **路线图**：增长模型
3. **仪表盘**：关键指标看板（定量）
4. **参考书**：用户心理决策地图（定性）

### 北极星指标（North Star Metric）
**定义北极星指标的6个标准**：
1. 反映产品核心价值——用户体验到了这种价值吗？
2. 反映用户活跃程度（非累积静态指标）
3. 指标变好 = 公司向好
4. 容易理解和交流（选绝对数，非百分比）
5. 先导指标优于滞后指标
6. 可操作

**案例对比**：
- MySpace 用"总注册用户数" → Facebook 用"月活跃用户数"
- "数据指标从来不只是指标，它代表管理层对用户价值的理解"

### 构建增长模型（3步）
1. **定义北极星指标**
2. **绘制用户旅程**：从一无所知到体验核心价值的步骤
3. **组装增长模型**：为每步找对应指标，代入公式

```
活跃用户 = 新增活跃用户 + 已有活跃用户
=（下载量×注册率×首次浏览比例×首次听歌比例）+（已有用户数×持续登录比例×持续听歌比例）
```

**增长模型的5大用途**：
1. 揭示所有输入变量，量化指导实验
2. 排序优先级，专注最有影响力的部分
3. 分解指标，定量预测增长趋势
4. 为进一步分析提供方向（分群分析）
5. 有效的管理决策沟通工具

### 关键行为漏斗
- 先追踪3个最重要的一级事件（不要贪多）
- 电商：产品页浏览 → 加入购物车 → 购买成功
- 聊天应用：注册 → 加好友 → 发信息

### 用户决策心理地图（6个阶段）
| 阶段 | 用户心理 | 增长重点 |
|------|---------|---------|
| 访问 | 会注意到吗？ | 冲击力设计+文案 |
| 转化 | 要不要试？ | 社交证明、稀缺性、紧迫感 |
| 激活 | 怎么用？得到价值了吗？ | 新用户引导、简化流程 |
| 留存 | 为什么要继续用？ | 习惯养成、里程碑、新功能 |
| 推荐 | 要介绍给别人吗？ | 情感决策+补贴激励 |
| 变现 | 值得付钱吗？ | 价值>定价 |

## 第3章 用户获取

### 永恒公式：LTV > CAC
- **CAC** =（营销总费用+销售总费用）/同期新增用户数
- **LTV** = 用户平均使用月数 × 每月平均收入
- 风投期望 LTV/CAC ≥ 3

### 打造1亿美元产品的四个契合
1. **市场—产品契合**：目标客户群需要此产品
2. **产品—渠道契合**：在特定渠道找到目标客户
3. **渠道—模型契合**：LTV能支撑渠道成本
4. **模型—市场契合**：目标客户愿意付费

### 五步选择用户获取渠道
1. 认识产品特点（面向对象、产品形态、盈利模式）
2. 了解用户群体（画像、习惯、在哪里）
3. 列出备选渠道（19种：付费、有机、其他）
4. 筛选初选渠道（大体量优先、免费优先、可追踪优先）
5. 运营、优化、拓展

**真正能做大的只有4类渠道**：付费增长、病毒传播、SEO、销售（都是反馈闭环+体量大）

### 用户推荐的6大类型
1. **口口相传**：产品体验极好
2. **展示相传**：好奇心驱动
3. **补贴推荐**：返利/代金券/产品功能（功能>钱）
4. **社交网络推荐**：邀请好友使双方体验更好
5. **病毒传播**：有趣新颖+视觉感染力+易分享
6. **产品内传播机制**：产品需求、内容分享、欢乐时刻、顺便接触

### 衡量用户推荐的万能公式
```
新用户推荐数 = 活跃用户基数 × 接触邀请机会比例 × 邀请转化率 × 平均邀请数 × 接受率 × 注册转化率
```

### 移动应用增长框架（Andy Carvell）
- 底层：技术
- 中层：分析和洞察
- 上层：用户获取、参与留存、变现
- **增长模型 + 增长框架图 = 完美暴击**

## 第4章 用户激活

### 为什么激活如此重要
- 绝大多数应用3天内流失超75%用户
- 新用户体验不好，其他功能做得再好也白做
- 激活率提升有**放大效应**——传递到整条留存曲线
- 提高市场预算回报率（更多用户成为长期用户）

### Aha时刻
定义公式：**（谁）在（多长时间内）完成（多少次）（什么行为）**

找到Aha时刻的4步：
1. 列出3~5个可能的关键行为
2. 数据分析：比较有/无行为用户的留存曲线差异
3. 定性调研：用户问卷、访谈确认
4. 确定关键行为+魔法数字+时间窗口

**注意**：关键行为与留存是相关性，需通过实验验证因果性

### 激动指数模型（Dropbox Darius Contractor）
- 0~100代表用户完成某事的动力
- 新用户激活 = 在激动指数降到0之前引导完成关键行为
- **每一步都可能增减激动指数**：注册障碍-10，社交证明+10，空白界面-20

### 新用户引导四大原则
1. **增强动力**：前后一致性、解释权限好处、社会信任、让用户参与
2. **减少障碍**：推迟注册、移除多余步骤、避免冷启动
3. **适时助推**：明确机会窗口、灵活UX模式、邮件/推送提醒
4. **私人订制**：用户自选偏好、根据历史推荐、按目的/群组分流

### 八大误区
1. 步骤太多流程太长
2. 没聚焦一个关键行为
3. 花太多时间教界面而非让用户使用
4. 让用户太快完成设置（缺乏教育）
5. 没设置必要障碍筛选不合格用户
6. 以"注册完成"而非"激活"衡量
7. 对每个用户统一对待
8. **完全照抄最佳实践而不A/B测试**

## 第5章 用户留存

### "不留存，就去死"
- BranchOut案例：病毒传播带来2500万用户，留存差→昙花一现
- Twitter案例：IPO后股价低迷，核心问题是留存率远低于Facebook
- **留存的复利效应**：80%月留存+500万新增 vs 95%月留存+250万新增，3年后后者反超

### 定义留存三要素
1. **回归关键行为**：不是登录，而是完成核心行为
2. **天然使用周期**：找到产品合适的留存频率（日/周/月）
3. **同期群分析**：追踪同一时期用户的留存曲线

### 留存曲线分析
- **横向**：好的留存曲线应趋于平缓（如果一直向下→可能未达PMF）
- **纵向**：后来的用户留存曲线应比之前更平缓（产品在改善）

### 留存四阶段
1. 新用户激活 → 帮助发现产品价值
2. 中期留存 → 形成使用习惯
3. 长期留存 → 持续感受核心价值
4. 流失用户 → 唤回

### 用户参与度阶梯
- **1-9-90规则**：1%超级用户、9%核心用户、90%消极用户
- 目标：把用户从低层级向高层级迁移
- Twitter的阶梯：消极（看推文）→ 核心（回复转发）→ 超级（搜索+粉丝群）

### 独角兽产品路线图（Sarah Tavel）
1. **增加参与用户数**（完成关键行为 = PMF）
2. **长期留住用户**（用得越多积累越多+离开损失越大）
3. **自我持续**（用户行为创造良性循环 → 获取新用户/改善体验/唤回老用户）
4. **独角兽**

### BJ Fogg行为模型：B = MAT
- **M**otivation（动力）：增强用户动机
- **A**bility（能力）：降低行动难度
- **T**rigger（触发）：适时提醒

### 习惯养成黄金公式
1. 给单次行为提供**不固定的奖励**（惊喜感）
2. 要求用户投入**一点努力**（增加沉没价值）
3. 外在触发 → 内在触发（从推送提醒到自动想起）
4. 打造参与闭环

### 参与闭环
- **单人模式**：Facebook"回忆"功能→一年后重新分享
- **多人模式**：LinkedIn"谁看了你的履历"→回到平台查看

### 提高留存8种武器
①产品改进 ②新用户引导教育 ③邮件 ④通知 ⑤客户服务 ⑥促销 ⑦忠诚客户计划 ⑧新产品
- 前两项回报最大
- 移动推送RRF模型：覆盖面(Reach) × 相关性(Relevance) × 频率(Frequency)

## 第6章 组建增长团队

### 为什么要增长团队
- Snapchat vs Instagram"故事"功能：几乎相同的产品，Instagram有增长团队→1年超越
- 聚焦 + 合作 + 体系化

### 增长 vs 产品 vs 营销 vs 运营
| | 核心目标 | 方法论 | 工具箱 |
|---|---------|--------|--------|
| 产品 | 增加产品价值 | 用户需求驱动 | 新功能开发 |
| 增长 | 让更多人体验价值 | 数据驱动+实验 | 产品内优化+外部渠道 |
| 营销 | 新用户获取 | 效果导向 | 外部渠道 |
| 运营 | 短期用户价值 | 活动/内容驱动 | 运营手段 |

### 增长团队四大核心职能
1. **战略**：建立增长模型，找杠杆，定方向
2. **执行**：全生命周期寻找增长机会
3. **流程**：高效A/B测试体系
4. **文化**：数据和实验驱动

### 团队配置
**基本配置**：1增长PM + 2~3工程师 + 1数据分析师 + 1设计师

**三种组织架构**：
1. **独立特区模式**（Facebook）：增长VP直报CEO，全栈团队
2. **功能主导模式**（LinkedIn）：归属产品部门
3. **内部咨询师模式**（Mozilla/Firefox）：提供增长咨询服务

Y Combinator调查：70%在产品部门，20%独立，10%在市场部门

### 不同阶段的增长团队
| 阶段 | 目标 | 建议 |
|------|------|------|
| 早期 | 判断是否需要 | 先确保PMF（看留存率） |
| <30人 | 最小化可行性团队 | CEO+1增长PM+1程序员+1设计师 |
| 30~200人 | 5人专门团队 | 找"两不管"地带下手，快速产生胜利 |
| 200人+ | 多个细分团队 | 按AARRR各阶段分团队 |

## 第7章 打造增长引擎

### 增长流程（以Canva为例）
1. **寻找机会**：聚焦"海报制作"功能（贡献20~30%新注册）
2. **收集数据**：可用性测试+用户问卷（磨刀不误砍柴工）
3. **形成假设**：个人化引导能提升激活率
4. **进行实验**：5%新用户看到新版本，95%对照
5. **分析结果**
6. **双倍下注**
7. **100%发布**

### 增长战略制定
1. **设定增长目标**
2. **指标分解**：大目标→多个可操作的小变量
3. **找到聚焦领域**：当前性价比最高的杠杆点

### 增长冲刺（Sprint）
1. 产生实验想法
2. 优先级排序（ICE评分：Impact影响力 × Confidence信心 × Ease容易度）
3. 设计和开发实验
4. 分析数据
5. 应用结果

### A/B测试要点
- "无所不在"——产品、渠道、文案、设计都可测试
- 注意统计显著性
- 实验结果 → 下一个实验的输入

### 高效能增长团队10个习惯
（包括但不限于）
- 每周固定增长例会
- 维护实验想法库
- 快速迭代（1~2周周期）
- 全员参与产生想法
- 记录和分享所有实验结果
- 指标透明公开
- 接受失败、从失败中学习
