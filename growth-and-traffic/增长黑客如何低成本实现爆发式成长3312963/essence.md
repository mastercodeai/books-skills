# 增长黑客：如何低成本实现爆发式成长

**作者**：肖恩·埃利斯 (Sean Ellis)、摩根·布朗 (Morgan Brown)  
**译者**：张溪梦  
**出版**：中信出版社  

---

## 核心论点

增长黑客不是找一个"独行侠"变魔术，而是一套**跨职能团队 + 数据驱动 + 快节奏试验**的系统方法论。它将精益创业的持续迭代理念应用于用户获取、激活、留存和变现的全生命周期，用低成本试验替代高额广告预算。

---

## 第一部分：方法论

### 一、搭建增长团队

**打破筒仓**：传统组织中市场、产品、工程、数据各自为战，增长团队将这些职能打通。

**团队构成**：
- **增长负责人**：确定试验流程、节奏，监测指标，主持周会
- **产品经理**：监督产品功能实现，作为"客户的诉求传达者"
- **软件工程师**：增长团队主力，用技术解决问题
- **营销专员**：内容/SEO/邮件等不同方向
- **数据分析师**：确保试验设计严密，从数据中提取洞察
- **产品设计师**：提升试验执行速度

**两种汇报结构**：
1. **产品部门主导模式**：团队向产品高管汇报，聚焦特定产品（Pinterest、LinkedIn、Twitter）
2. **独立模式**：向增长副总裁/CEO汇报，可跨产品线试验（Facebook、Uber）

**BitTorrent案例**：移动团队通过跨职能协作，一个"升级按钮"带来日收入92%增长；鼓励用户写好评使四五星评论增加900%；两年半内App下载量达1亿次，总收入增长300%。

---

### 二、好产品是增长的根本

**核心原则**：在确定产品"不可或缺"之前不要火力全开。

**"啊哈时刻"（Aha Moment）**：用户真正发现产品核心价值的瞬间。
- Yelp → 用户通过真实社区评论发现好商家
- Facebook → 前10天添加至少7位好友
- Twitter → 关注至少30个人，1/3回关
- Slack → 团队内发送2000条消息
- Dropbox → 便捷文件分享 + 无限存储

**不可或缺性调查**：
> "如果这个产品明天就无法使用了你会有多失望？"

- ≥40% 回答"非常失望" → 可以全力驱动增长
- 25%~40% → 需微调产品或调整描述语言
- <25% → 需要实质性产品开发或换目标市场

**BranchOut反面教训**：过早追求病毒增长，三个月用户从400万涨到2500万，但产品体验差，月流失率4%，最终以200万美元被收购。

**产品重新定位案例**：
- Burbn → 砍掉所有功能只留照片分享 → Instagram
- Tote → 用户不购买而是收藏 → Pinterest
- The Point → 优惠购买众筹最成功 → Groupon
- 视频约会网站 → 用户分享各类视频 → YouTube

---

### 三、确定增长杠杆

**增长等式**（因产品而异）：
- eBay：发布物品的卖家数 × 发布物品数量 × 买家数 × 成功交易数 = GMV
- Amazon：垂直扩张 × 产品库存 × 流量 × 转化率 × 客单价 × 复购率 = 收入增长

**北极星指标（North Star Metric）**：最能反映核心价值体验的单一指标。
- WhatsApp → 发送信息数（非DAU）
- Airbnb → 客房预订量
- eBay → 总商品量（GMV）
- Facebook → 月活跃用户数 → 日活跃用户数

**数据整合**：需要构建"数据湖泊"或"数据仓库"，整合网络分析、CRM、销售系统等多源数据。Facebook增长团队在2009年1月暂停所有试验，花整整一个月改进数据跟踪系统。

---

### 四、快节奏试验

**增长黑客循环**：分析 → 想法 → 排优先级 → 测试 → 回到分析

**ICE评分体系**：
- **Impact（影响力）**：对目标指标的预期提升程度
- **Confidence（信心）**：基于数据/经验的把握
- **Ease（简易性）**：所需时间和资源

**试验规则**：
- 采用99%置信水平
- 永远以对照组为依据（平局时对照组获胜）
- 缓慢起步，逐渐提速
- 每月5%的转化率改进 → 全年80%的提升

**增长会议**（每周一次，1小时）：
1. 15分钟：回顾指标并更新关注领域
2. 10分钟：回顾前一周测试工作
3. 15分钟：分析试验收获
4. 15分钟：选择下一周期试验
5. 5分钟：检查想法储备库

## 第二部分：实战

### 五、获客：优化成本，扩大规模

**三个前提匹配**：
1. **产品—市场匹配**（PMF）
2. **语言—市场匹配**：描述能否打动用户
3. **渠道—产品匹配**：营销渠道是否有效

**语言优化**：
- Tickle：将"在线存储照片"改为"在线分享照片" → 6个月增加5300万用户
- 约会App："寻找约会对象"改为"帮助他人寻找约会对象" → 8个月增加2900万用户
- Highrise："注册并试用"改为"查看购买方案与价格" → 注册量增加200%
- iPod："将1000首歌放在你的口袋里"

**渠道选择**：不要广撒网，找一两个最优渠道。三种类型：病毒/口碑、有机、付费。

**渠道排序六要素**：成本、定向、控制、时间投入、产出时间、规模

**病毒循环设计**：
- 病毒性 = 有效载荷 × 转化率 × 频率
- K因子 = 发出邀请数 × 接受率（K>1几乎不可能，不必执着）
- Dropbox推荐计划：双向奖励250MB存储空间，推荐注册量增加60%，最终从10万增至400万
- Airbnb推荐：双向25美元旅行信用，日注册量增加300%

---

### 六、激活：让潜在用户真正使用产品

**核心公式**：欲望 – 摩擦 = 转化

**三步法**：
1. 列出到达"啊哈时刻"前的所有步骤
2. 创建转化/流失漏斗报告
3. 对流失用户和坚持用户分别调研

**消除摩擦**：
- **单点登录**：Kissmetrics用谷歌账号登录 → 注册量增59.4%（但英曼取消后反而提高24.8%——必须测试！）
- **翻转漏斗**：先让用户体验产品再注册。HelloBar → 激活率提升52%；Stripe先给代码后要求注册

**积极摩擦**：
- 游戏化入门教程让用户做出承诺（Adobe图像处理闯关 → 购买量增加4倍）
- Pinterest：让用户选5个感兴趣的话题 → 激活率提高20%

**触发物**：
- 基于福格行为模型：动机 × 能力 → 行动
- 三种类型：协助型、信号型、刺激型
- 西奥迪尼六原则：互惠、承诺一致、社会认同、权威、喜好、稀缺
- 关键：触发物必须提醒用户"存在明显有价值的机会"，否则适得其反

---

### 七、留存：唤醒并留住用户

**三个阶段**：
1. **初期留存**：决定用户是继续使用还是"沉睡"（优化新用户体验、触发物）
2. **中期留存**：让使用成为习惯（参与回环、回报机制）
3. **长期留存**：持续为用户带来更大价值（新功能、个性化、持续引导）

**群组分析**：按获客时间/渠道/行为特征分组跟踪留存曲线

**习惯养成**：
- Amazon Prime：91%续订率，第三年达96%。免费配送 + 即刻满足 + 心理账户改变
- 储值效应：用户在产品中投入越多，忠诚度越高

**留存策略**：
- 品牌大使计划（Yelp精英队贡献44%总评论量）
- 认可用户成就（Fitbit祝贺万步、Medium祝贺文章被推荐）
- 个性化推荐（亚马逊、Netflix、Pinterest的Copytune）
- 控制节奏（HBO、Netflix"即将播出"策略降低流失率）
- 复活"僵尸用户"（印象笔记发现换手机后未重装是主因）

---

### 八、变现：提高每位用户带来的收益

**绘制变现漏斗**：找出高收益环节和夹点（损失潜在收益的地方）

**群组分析**：按收益贡献分群组，找出不同群体的变现特征
- HotelTonight：3G/4G用户订房率是Wi-Fi用户的两倍 → 只向非Wi-Fi用户投广告

**个性化推荐**：亚马逊推荐引擎、杰卡德系数计算商品相似度

**定价策略**：
- 心动价格：以9/99/95结尾可提高销售额24%
- 定价相对论：《经济学人》"烟幕弹套餐"让84%选最贵方案
- SmartShoot增加烟幕弹产品 → 转化率飙升233%
- Qualaroo反直觉提价400% → 打开大客户市场

**价值度量**三个问题：是否与用户看到的价值匹配？是否随使用量增加？是否易于理解？

**消费者心理学**（西奥迪尼六原则在变现中的应用）：
- 互惠：好市多样品演示提高购买率2000%
- 社会认同：英国税务总署"大多数人按时缴税" → 缴税率从57%升至86%
- 稀缺：Booking.com显示剩余房间数和正在浏览的人数

---

### 九、良性循环：维持并加速增长

**避免增长停滞**：
- 87%企业经历过增长停滞，前后10年市值平均缩水74%
- 原因：疏于监测用户满意度、过度依赖单一渠道、骄傲自满、人才流失

**GrowthHackers自身教训**：一个季度测试不到10次导致三个月增长停滞；恢复每周3次试验后，三个月增长等于前一年总和。

**关键行动**：
1. 像鲨鱼一样不停游动——持续试验
2. 不要低估决心——充分挖掘已成功渠道的潜力
3. 深挖数据金矿——投资数据分析能力
4. 发掘新渠道——避免单一渠道依赖
5. 引入全新视角——跨部门/外部专家集思广益
6. 超越局部最大值——定期进行大胆创新试验

| 阶段 | 核心问题 | 关键指标 |
