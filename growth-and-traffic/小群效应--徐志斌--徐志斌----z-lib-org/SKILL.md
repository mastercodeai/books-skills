# SKILL.md — 小群效应实战技能手册

> 基于徐志斌《小群效应》提炼的可操作方法论，聚焦社群增长、留存与变现。

---

## 场景路由：根据目标快速定位章节

```
你的核心目标是什么？
│
├─ 社群运营 / 社区建设 → 阅读「一 ~ 三、八、九」
│   关键路径：诊断健康度 → 找三近一反 → 培养连接者 → 选运营策略 → 建文化
│
├─ 社交电商 / 社群变现 → 阅读「四、六、十」
│   关键路径：设计工具性 → 搭互惠接口 → 建进阶机制 → 加入一反供需关系
│
├─ 病毒式传播 / 裂变增长 → 阅读「三、五、七」
│   关键路径：找连接者 → 密集度策略 → 影响身边人 → 魔术数字
│
└─ 快速诊断现有社群 → 直接跳到「社群诊断框架」+「失败模式」
```

## 一、社群诊断框架

### 1. 三个标准评估社群健康度
- **互相认识程度**：成员好友数（非平均数，而是大部分用户的实际值）
- **互相信赖程度**：是否有"三近"基础？是否曾互相帮助？
- **互动频次**：社群消耗用户时长是多少？

### 2. 社群类型快速判断
问自己三个问题：
- 成员之间是否互相认识？→ 熟人型/陌生人型
- 核心动作是浏览还是互动？→ 浏览型/互动型
- 关系是平等还是不平等？→ 平等型/不平等型

### 🔍 诊断检查点
| 指标 | 健康信号 | 危险信号 |
|------|----------|----------|
| 日活跃率 | >30% | <10% |
| 新人7日留存 | >40% | <15% |
| 人均好友数 | >5个群内好友 | 0-1个 |
| UGC产出比 | >5%成员产出内容 | <1% |
| 互助行为 | 每周有自发互助 | 全靠官方推动 |

## 二、三近一反：社群构建基础

### 运用清单
```
□ 找到"三近"：你的用户在地域/年龄/兴趣上有哪些相似性？
□ 找到"一反"：用户之间存在什么相互需求或冲突？
□ 三近帮助聚集和扩散，一反帮助黏合和变现
□ 行业社群(甲乙双方/供需双方)最快变现
```

### 检验方法
- 如果社群长期沉默 → 检查"一反"是否缺失
- 如果社群转化率低 → 检查是否存在供需关系
- 如果新人留存差 → 检查新人能否快速找到"三近"的小圈子

### 🔍 检查点
```
□ 是否已用一句话描述你的"三近"？（写不出来 = 还没想清楚）
□ 是否已用一句话描述你的"一反"？（没有一反 = 只能做内容社区，难变现）
□ 新成员加入后能否在3分钟内感受到"和我好像"？
```

## 三、找到你的连接者

### 四步发现法
1. **建立目标用户库**（500-5000人），添加为好友
2. **翻阅半年朋友圈/微博**，记录：
   - 关注了哪些账号/KOL
   - 经常讨论什么话题
   - 分享了什么链接（来自哪些APP/账号）
   - 语言风格、信息类型、发布时段
   - 参与过什么线上/线下活动
3. **制作工作表格**：连接者人群、KOL、目标合作APP、活跃时段、兴趣喜好等
4. **定期更新**，辅以大数据分析

### 关键认知
- 连接者不一定是大V，可能是普通人但关系链跨多个小群
- 0.8%的人可以带来46.25%的参与量
- 增加连接者数量 = 放大社群影响力

### 🔍 检查点
```
□ 是否已识别出至少5位连接者？
□ 连接者覆盖了几个不同的小群/圈子？
□ 是否有机制持续发现新连接者？（连接者会流失）
```

## 四、工具性设计：用户为什么加入？

### 核心问题
> "你解决了什么根本性问题，让用户第一眼看到就愿意加入？"

### 六个用户诉求（对应六种工具性用法）
| 诉求 | 工具性设计 |
|------|------------|
| 问答求助 | 快速获得专业回答（如跑步论坛新人问装备） |
| 炫耀 | 比拼/排名/勋章系统 |
| 寻找共鸣 | 情感话题、吐槽栏目 |
| 分享知识 | 深度文章、知识库 |
| 管理潜在关系 | 储备人脉、维系关系的功能 |
| 长期目标/价值观 | 社群文化、共同使命 |

### 工具性三重作用
1. **外部性**：建立潜在用户池（通过内容扩散）
2. **过滤器**：将新用户引导到"三近一反"的小圈子
3. **望远镜**：观察社交网络变迁，找到新切入点

### 🔍 检查点
```
□ 能否用一句话说清用户加入的理由？（不是"学习成长"这种空话）
□ 新用户首次接触时，是否在30秒内理解价值？
□ 工具性是否足够强，强到用户愿意主动推荐给朋友？
```

## 五、小池塘里的大鱼：荣誉驱动运用

### 三步设置法
```
1. 设定一个小池塘（和谁比？）
2. 让用户去比较，告诉比较结果（比什么？）
3. 让用户知道有人在和他比（感知竞争存在）
```

### 小池塘划分维度
- 和全部用户比（无关系链产品）
- 和好友比（关系链是天然小池塘）
- 和等级/战斗力相近用户比（动态进阶）
- 和相同兴趣小范围好友比
- 和自己过往表现比

### 动态进阶机制设计
- 排行榜永远以用户为中心，展示前后5名
- 用户处在"够够手就能超越"和"别人够够手就能超越自己"的状态
- 新人不要和顶级用户比较（会放弃）
- 难度略高于个人能力时，成就感最高

### 🔍 检查点
```
□ 用户是否能清晰感知自己在小池塘中的位置？
□ 排名更新频率是否合适？（太慢无感，太快焦虑）
□ 新用户是否被保护，避免与顶级用户直接比较？
```

## 六、互惠接口：利益驱动运用

### 六原则检查清单
```
□ 个性化：用户可以自行决定要不要、给谁
□ 参与简单：一键发起，无门槛
□ 零成本：免费获取，不需要大量人力维护
□ 双方利好：对企业核心诉求有帮助，社群也能受益
□ 可分享：利益可以被分享给好友
□ 可累积：持续投入可获得更大利益
```

### 长板效应
- 只需不断强化最擅长的领域
- 其他领域的强者会自动前来合作
- "被人使用得越多，价值就越大"

### 🔍 检查点
```
□ 互惠是否真正双向？（只有平台得利 = 假互惠）
□ 用户分享利益时是否感到"有面子"而非"像微商"？
□ 利益是否可累积？（一次性奖励 vs 长期回报）
```

## 七、三人成虎：关系驱动运用

### 影响决策的正确姿势
```
❌ 直接向目标用户推销
✅ 先影响他身边的2-3位好友
```

### 魔术数字参考
| 产品 | 魔术数字 | 效果 |
|------|----------|------|
| 捞月狗 | 3个好友 | 6-9个月活跃 |
| 朋友印象 | 10个好友 | 点赞/发帖/阅读跃升 |
| LinkedIn | 5个好友(首周) | 留存率提升 |

### 密集度策略
- 在特定人群中密集发声 → 病毒式扩散
- 影响15%人群即可影响整个大型社群
- 密集度 + 小池塘里的大鱼 = 绝配

### 🔍 检查点
```
□ 是否已识别本产品的"魔术数字"？（好友数量阈值）
□ 新用户加入首周是否能加上N个好友？
□ 是否在特定小圈子实现了密集渗透？（>15%）
```

## 八、运营策略选择

### 强运营适用场景
- 人群范围明确（如CMO、产品经理）
- 社群诞生早期
- 垂直领域/地域社群

### 弱运营适用场景
- 大型平台
- 成熟社群
- 覆盖范围广、新增人群不明确

### 闺蜜圈弱运营三板斧
1. **机制运营**：人群细分 → 内容分类 → 去中心化曝光
2. **标杆运营**：找到典型用户/内容/玩法，给予资源推荐
3. **氛围运营**：让用户在意"我属于这个圈子"

### 用户状态管理（豆瓣四型）
```
植物型(低频浏览) → 动物型(高频浏览) → 人类型(产出内容) → 大神型(产出影响力)
```
- 推动用户向上迁移
- 管理四种状态之间的比例
- 人类型比例升高+大神型降低 = 用户自说自话，缺乏互动
- 非人类型比例升高 = 用户正在逃离

### 🔍 检查点
```
□ 当前阶段该用强运营还是弱运营？（别选错）
□ 用户状态分布是否健康？（植物型>60%要警惕）
□ 大神型用户是否在流失？（流失 = 社群衰落信号）
```

## 九、社群文化建设

### 关键机制
| 机制 | 怎么做 |
|------|--------|
| 故事 | 可流传的历史和传奇（创始故事、大事件） |
| 仪式 | 入会仪式、统一前缀、固定格式 |
| 语言 | 独有术语/表情包/暗号 |
| 竞争 | 团队对抗、跨社群比拼 |
| 归属感 | 所有机制联合作用的结果 |
| 模拟关系 | 将陌生人关系模拟为家人/战友/师徒 |

### 模拟关系 + 事件驱动
- 告诉粉丝一个目标
- 粉丝将你的目标视为自己的目标
- 自觉分配时间、精力和资源去帮助实现

### 🔍 检查点
```
□ 社群是否有至少2个可流传的故事？
□ 是否有可识别的仪式/语言/符号？
□ 成员能否说出"我们社群和别的社群不一样的是……"？
```

## 十、用户进阶机制

### 三种进阶方向
1. **荣誉驱动进阶**：动态小池塘比较（等级排行榜、财富排行榜、技能排行榜）
2. **等级制进阶**：根据活跃度/积分划分等级（QQ星星月亮太阳、贴吧18级）
3. **利益驱动进阶**：根据贡献划分等级（大V店蜜蜂→凤凰妈妈，不同等级对应不同利益）

### 关键原则
- 用户希望不断进步
- 希望和比自己能力更强/地位更高的人在一起
- 到达最高点时，设置新的更高成就点

### 🔍 检查点
```
□ 进阶路径是否清晰可见？（用户知道自己在哪、下一步去哪）
□ 是否有"天花板"处理方案？（顶级用户无处可去会离开）
□ 荣誉、等级、利益三者是否平衡？（纯利益驱动会吸引薅羊毛）
```

## 失败模式：常见踩坑与应对

### 社群冷启动失败
| 症状 | 根因 | 应对 |
|------|------|------|
| 拉了500人但没人说话 | 没有"一反"，缺乏互动理由 | 先找10-20个种子用户，用"一反"黏合 |
| 种子用户活跃但扩展后沉默 | 种子用户之间三近强，但和新人无三近 | 按三近维度细分小群，不要混在一起 |
| 群建了3天就死 | 没有工具性设计，用户不知道来干嘛 | 回到第四章，回答"解决了什么根本问题" |

### 社群增长停滞
| 症状 | 根因 | 应对 |
|------|------|------|
| 靠拉新但留存差 | 没达到魔术数字（好友数不够） | 设计"加好友"机制，帮新用户快速建立关系链 |
| 分享出去但没人来 | 分享内容没有工具性价值 | 让分享的内容本身就是"工具"（如测试题、排行榜） |
| 连接者不活跃 | 连接者没有获得足够的荣誉/利益回报 | 给连接者专属身份+优先权益 |

### 社群变现失败
| 症状 | 根因 | 应对 |
|------|------|------|
| 推商品没人买 | 缺乏"一反"中的供需关系 | 重组社群结构，让供需双方共存 |
| 用户觉得"被割韭菜" | 利益单向流向平台 | 设计真正的双向互惠接口 |
| 只有头部用户买单 | 荣誉驱动缺失，只有利益驱动 | 加入"小池塘里的大鱼"机制 |

### 社群衰落信号
```
⚠️ 早期预警（还能救）：
  - 大神型用户开始减少发言
  - 新人7日留存率连续2周下降
  - 管理员/官方产出占比超过50%

🚨 紧急警报（需要大改）：
  - 日活跃率跌破10%
  - 核心成员开始建自己的群并转移
  - UGC产出完全依赖激励（去掉激励就停）
```

## 反模式：这些做法看似合理但会毁掉社群

### ❌ 反模式1：盲目追求人数
```
错误：群越大越好，疯狂拉人
后果：三近被稀释，互动率暴跌，变成广告群
正确：先做小而精（50-200人），验证三近一反后再扩展
```

### ❌ 反模式2：用红包代替机制
```
错误：社群不活跃就发红包
后果：吸引薅羊毛用户，红包停则群死
正确：用荣誉驱动（小池塘里的大鱼）替代金钱激励
```

### ❌ 反模式3：强推KOL运营
```
错误：花大价钱请KOL来带社群
后果：KOL走了社群就散，用户关系链在KOL身上而非社群
正确：培养普通连接者（0.8%人群），让关系链属于社群
```

### ❌ 反模式4：过早变现
```
错误：社群刚建立就开始卖货
后果：用户觉得被利用，信任崩塌
正确：先用工具性价值吸引 → 建立三近一反 → 自然产生变现场景
```

### ❌ 反模式5：忽视小圈子
```
错误：把所有人放在一个大群里统一运营
后果：陌生人之间无互动，信息过载，用户沉默
正确：按三近维度拆分小群，大群做分发，小群做互动
```

### ❌ 反模式6：排行榜只看Top N
```
错误：排行榜只展示前10名
后果：99%的用户永远上不了榜，失去动力
正确：以用户为中心展示前后5名，让每个人都有"够够手"的目标
```

## 快速行动清单

### 启动社群前
- [ ] 明确"三近"和"一反"
- [ ] 回答"你解决了什么根本性问题"
- [ ] 设计工具性用法（让用户一瞥就加入）
- [ ] 确认不用红包代替机制设计
- [ ] 确认不过早变现

### 社群增长
- [ ] 找到并培养连接者（0.8%关键人群）
- [ ] 设置互惠接口（利益可分享、可累积）
- [ ] 识别并运用"魔术数字"（好友数量阈值）
- [ ] 确认分享内容本身具有工具性价值

### 社群留存
- [ ] 设计动态小池塘和进阶机制
- [ ] 创造"够够手"的竞争体验
- [ ] 培养社群文化（故事+仪式+语言）
- [ ] 监控大神型用户活跃度

### 社群变现
- [ ] 加入"一反"因素（供需双方、甲乙双方）
- [ ] 运用荣誉驱动（比利益驱动更强）
- [ ] 模拟关系 + 事件驱动 → 粉丝自驱力
- [ ] 确认互惠是双向的

### 定期健康检查（每月）
- [ ] 日活跃率是否>30%？
