# SKILL.md — 私域亲密关系运营方法论

## Skill 名称
`private-domain-relationship-flywheel`

---

## 场景路由（Scene Routing）

根据业务场景快速定位适用模块：

| 场景 | 核心关切 | 优先阅读模块 | 推荐关系模型 |
|------|---------|-------------|-------------|
| **私域运营** | 用户留存、复购、LTV提升 | 核心框架 → 平辈/长辈实操 → 数据基准线 | 平辈或长辈 |
| **社群构建** | 活跃度、互动密度、裂变增长 | 裂变六字 → 长辈"百事通" → 关键时刻 | 长辈（社区团购）、平辈（品牌社群） |
| **超级用户培养** | 识别、激活、放大1%用户价值 | 识别超级用户 → 飞轮构建 → 晚辈"养成" | 晚辈（新品牌）、平辈（成熟品牌） |
| **DTC品牌建设** | 品牌忠诚度、用户共创 | 关系定位 → 晚辈实操 → 价值观对齐 | 晚辈（新锐品牌）、平辈（品质品牌） |
| **高客单价转化** | 信任建立、长周期决策、转介绍 | 平辈实操 → 信任账户 → 关键时刻 | 平辈 |
| **导购/KOC赋能** | 本地化运营、人情关系 | 长辈实操 → 地域要素 → 百事通模式 | 长辈 |

**路由决策树**：
1. 产品是否高价低频？ → **是** → 平辈关系 + 提频策略
2. 品牌是否为新创？ → **是** → 晚辈关系 + 养成机制
3. 是否涉及本地/社区场景？ → **是** → 长辈关系 + 百事通模式
4. 用户群体是否成熟忠诚？ → **是** → 平辈关系 + 信任账户

- 私域流量运营策略设计

- KOC/导购赋能体系建设

## 核心框架：关系飞轮模型

### 第一步：确定关系定位

根据企业类型选择模拟的亲密关系：

| 关系类型 | 关键词 | 适合企业 | 用户宽容度 |
|---------|--------|---------|-----------|
| 平辈（兄弟姐妹） | "我很可信" | 老牌大品牌、品质过硬的传统企业 | 低（信任崩塌不可逆） |
| 晚辈（子女） | "我很可爱" | 新创品牌、轻量级团队 | 最高（包容小错） |
| 长辈（父母） | "我很可亲" | 大企业、知名品牌、高利润行业 | 中等 |

### 第二步：识别超级用户

**超级用户 = 1%的用户群，贡献5倍以上价值**

识别指标（四高）：
1. **转化率高**：直接付费购买
2. **复购率高**：高频重复购买
3. **分享率高**：主动在社媒分享产品信息
4. **转介绍率高**：影响亲友购买决策

数据参考：
- 行业App平均分享率 < 3%，超级用户群体可达 60%+
- 超级用户阅读/消费贡献是普通用户的 5-10 倍
- 友盟+数据：超级用户占11%-28%，贡献时长42%-79%

### 第三步：构建新增长飞轮

```
密集分享 → 小圈子密集转化 → 亲密关系增强
    ↑                                    ↓
长期持续支持 ← 强力黏着复购 ← 超强转化
```

飞轮启动条件：
- 高质量产品/内容（先满足需求，再超越预期）
- 共同目标或事件驱动
- 用户参与感（参与即喜）
- 密集覆盖（同一圈子多人同时关注）

### 第四步：运用"裂变六字"与扩展

**基础六字**：拼、帮、砍、送、比、换

**扩展方向**：
- **争**：小团队间较量比拼（关系驱动 × 事件驱动）
- **约**：地域性强的共同活动（关系 × 事件 × 地域）

**底层逻辑**：任何增长策略的根本力量 = "让好友之间关系更紧密的互动行为"

## 实操方法

### 1. 平辈关系（"我很可信"）实操

**核心动作**：
- 产品品质第一，物美价廉，服务超预期
- 建立"信任账户"机制：每次推荐都为关系加分
- 售后服务不是终点，是建立长期关系的关键出口

**前置运营模式**：
1. 社群内需求搜集 → 立项研发
2. 可参与的感知活动（如"立硬币"展示静音技术）
3. 用户从价格思维转向价值思维
4. 预售定金模式（用户愿意等3-4个月）

**关键数据参考**：
- 估量定价法：先买先受益，保护忠诚用户
- 海尔顺逛微店主：1%超级微店主贡献40%+收入
- 信任崩塌 = 关系一去不复返，一次差评毁掉团长口碑

### 2. 晚辈关系（"我很可爱"）实操

**核心动作**：
- 让用户感知到企业在"时时成长"
- 展示成长过程（挫折、进步、迭代）
- 全员与用户互动，快速响应反馈

**"养成"运营机制**：
1. 用户参与产品测试，意见被采纳即产生欣喜
2. 专属服务群（如蔚来为每个小订用户建9人群）
3. "半成品"思维：让用户参与产品/营销改进
4. 正反馈循环：用户投入 → 品牌成长 → 用户更投入

**提频策略**（高价低频产品）：
- 生活方式社区（不只聊产品，聊生活）
- 轻互动广告（如"装有情书的信封"式投放）
- 中间态轻服务（9.9元体验课、199元轻咨询）
- 内容提频（用户成长视频分享率100%）

**风险提示**：
- 用户过度维护可能"好心办坏事"（如蔚来车主联合声明事件）
- 爱无限，但对物质索取应有限度

### 3. 长辈关系（"我很可亲"）实操

**核心动作**：
- 时时陪伴，主动洞察需求
- 制造感动和惊喜瞬间
- 售后服务是关系构建的最佳出口

**"五步法"（借鉴保健品行业正面经验）**：
1. **进入用户熟悉场景**：公园、菜市场等，不制造陌生感
2. **制造惊喜瞬间**：天气不好时送菜上门、海外带回专业书籍
3. **时时陪伴**：定期拜访、嘘寒问暖、拉家常
4. **激发成就感**：让顾客产生指导欲（青涩 > 套路化）
5. **发起大事件**：到店体验 → 大促转化

**"百事通"模式**（社区团购）：
- 团长不仅卖货，还提供生活服务（修水管、开锁等联系方式）
- 增进邻里关系 → 复购从6-8次/月升至8-12次/月
- 退货率从3%降至万分之二

## 关键时刻运营

| 时刻类型 | 触发条件 | 运营动作 |
|---------|---------|---------|
| 美好时刻 | 企业有好事（融资、获奖、突破） | 顺势推出活动，多开用户参与接口 |
| 荣光时刻 | 新技术、新突破、社会责任 | 让用户脸上有光，推荐时给信任账户加分 |
| 啊哈时刻 | 产品使用中的惊喜体验 | 在关键节点设计超预期体验 |
| 感动时刻 | 雪中送炭（如疫情期间送口罩） | 主动关心用户需求，不求回报 |

## 关系建立六大要素清单

- [ ] **事件**：是否有随时可触发的参与事件？
- [ ] **场景**：是否有可互动的社群/社区场景？
- [ ] **角色**：用户是否清楚自己不可或缺？
- [ ] **地域/兴趣**：是否利用了地域和兴趣相近性？
- [ ] **进阶**：是否有关系持续升级的路径（会员体系/积分）？
- [ ] **价值观**：品牌和用户价值观是否一致？

## 私域的天花板

**用户亲密关系数量 = 私域天花板**

- 71.88%用户只接受不超过5个商家进入私域
- 邓巴发现：用户60%时间给15个最亲密的人
- 品牌必须成为用户的15个亲密好友之一

**结论**：私域或很短，亲密关系所主导的红利却很长。改变商业的不是私域，是企业和用户之间形成的亲密关系。

## 失败模式（Failure Modes）

### FM-1：关系定位错配
- **症状**：用户互动冷淡、社群活跃度持续走低、活动参与率 < 5%
- **根因**：新品牌强行用平辈（高期待低容忍），或成熟品牌用晚辈（用户觉得做作）
- **诊断**：用户是否在主动传播品牌？NPS中推荐分是否显著高于满意度？
- **修复**：重新评估品牌成熟度和用户心理预期，切换关系模型

### FM-2：飞轮空转
- **症状**：社群人数增长但转化率不升，分享行为多但转化少
- **根因**：缺乏"密集覆盖"——同一圈子内品牌触达人数不足，无法形成社交验证
- **诊断**：同一用户圈层内品牌触达率是否 > 30%？分享后是否有二次互动？
- **修复**：聚焦小圈子渗透而非泛人群覆盖，先做深再做广

### FM-3：信任账户透支
- **症状**：用户沉默退出、负面口碑传播、退群率突增
- **根因**：过度营销/频繁促销消耗了建立的信任存量
- **诊断**：社群消息中营销内容占比是否 > 30%？用户主动发言率是否下降？
- **修复**：立即停止促销密集期，转入价值输出模式（内容/服务/互动），重建信任

### FM-4：超级用户枯竭
- **症状**：核心用户流失、转介绍断崖式下降、KOC活跃度骤降
- **根因**：缺乏超级用户维护体系，核心用户未被识别/未获差异化对待
- **诊断**：是否有超级用户专属通道？Top 1%用户近30天是否无互动？
- **修复**：建立超级用户分层识别机制，提供专属权益和一对一服务

### FM-5：本地化失灵
- **症状**：社区团购/本地社群活跃度低、复购率不达标
- **根因**：外部团队运营本地社群，缺乏在地信任基础
- **诊断**：社群运营者是否为本地人？是否有线下互动触点？
- **修复**："本地人做本地事"，培养本地KOC/团长，增加线下场景触点

### FM-6：养成疲劳
- **症状**：用户参与热情从高潮回落、共创活动参与率递减
- **根因**：缺乏阶段性里程碑和正反馈，用户感觉投入无回报
- **诊断**：用户建议采纳率？是否有可见的品牌成长反馈？
- **修复**：设置明确里程碑、公开致谢贡献者、让成果可视化

## 反模式（Anti-Patterns）

### AP-1：流量思维入侵私域
- **表现**：把私域当广告渠道，群发促销信息，用ROI衡量一切
- **为什么错**：私域本质是关系而非渠道，流量思维会快速透支信任
- **正确做法**：以关系深度（复购率、转介绍率、参与度）作为核心指标

### AP-2：标准化服务替代个性化
- **表现**：对所有用户同一话术、同一活动、同一权益
- **为什么错**：亲密关系的核心是"被看见"和"被特殊对待"
- **正确做法**：基于用户分层（超级用户/活跃/沉默）提供差异化服务

### AP-3：过度依赖裂变工具
- **表现**：疯狂使用拼团、砍价工具，忽视关系质量
- **为什么错**：裂变六字的底层是关系紧密化，工具只是载体
- **正确做法**：先确保用户间有真实关系基础，再用工具放大

### AP-4：高管缺席用户互动
- **表现**：所有用户互动由客服/运营团队完成，管理层从不露面
- **为什么错**：晚辈关系需要"全员互动"，长辈关系需要"真诚陪伴"，层级隔离破坏亲密感
- **正确做法**：创始人/高管定期出现在社群、直播、线下活动中

### AP-5：追求规模牺牲密度
- **表现**：疯狂拉新但单个社群/圈层渗透率低
- **为什么错**：飞轮启动需要"密集覆盖"，广而不深无法触发社交验证
- **正确做法**：先在10个种子圈层做到高密度渗透，再复制扩展

### AP-6：信任崩塌后的强行修复
- **表现**：平辈关系中出现品质事故后，用道歉信/补偿活动试图挽回
- **为什么错**：平辈信任崩塌不可逆，修复尝试反而加深负面印象
- **正确做法**：预防为主；一旦发生，坦诚面对后果而非试图修复关系

## 检查点（Checkpoints）

### 🟢 启动期检查点（第1-4周）

- [ ] **关系定位确认**：是否已明确选择平辈/晚辈/长辈模型？团队是否达成共识？
- [ ] **种子用户识别**：是否已从现有用户中筛选出Top 1%超级用户？
- [ ] **场景搭建**：是否已建立至少1个可互动的用户社群/社区？
- [ ] **团队共识**：一线运营人员是否理解关系模型并知道如何执行？
- [ ] **价值内容储备**：是否有至少2周的非营销内容（故事/知识/互动）？

### 🟡 成长期检查点（第5-12周）

- [ ] **互动密度**：社群日活跃发言用户占比是否 > 15%？
- [ ] **飞轮启动信号**：是否有用户主动发起分享/拼团/推荐（非激励驱动）？
- [ ] **超级用户活跃**：识别的超级用户近7天是否有互动行为？
- [ ] **裂变质量**：新用户来源中关系渠道（转介绍）占比是否在上升？
- [ ] **信任账户正向**：用户反馈中正面评价占比 > 80%？
- [ ] **关系要素覆盖**：六大要素清单中是否至少满足4项？

### 🔴 成熟期检查点（第13周+）

- [ ] **飞轮自转**：是否出现用户自发驱动的增长（无需官方发起活动）？
- [ ] **数据达标**：分享率是否 > 10%？转介绍率是否 > 20%？
- [ ] **LTV提升**：亲密关系用户LTV是否为普通用户的3倍以上？
- [ ] **天花板感知**：是否已接近单用户可接受的私域商家数上限？
- [ ] **抗风险能力**：是否有超级用户流失的预警机制和应对方案？
- [ ] **关系可复制**：运营模式是否可复制到新城市/新品类/新圈层？

### ⚠️ 危机检查点（触发条件：以下任一指标异常）

触发条件 → 立即检查 → 应对动作：

| 触发信号 | 检查项 | 紧急动作 |
|---------|--------|---------|
| 社群退群率 > 5%/周 | 是否出现信任透支？营销内容占比？ | 暂停所有促销，转入价值输出 |
| 超级用户30天无互动 | 是否缺乏专属维护？是否需求未被满足？ | 启动一对一触达，了解原因 |
| 负面口碑传播 | 哪个环节出了问题？是否为品质事故？ | 快速响应，透明沟通，不掩盖 |
| 分享率持续下降 | 内容质量是否下降？分享动机是否枯竭？ | 更新分享素材，创造新的分享场景 |
| 转介绍质量下降 | 推荐来的用户是否匹配？是否激励过度？ | 调整激励机制，强调质量而非数量 |

---

## 禁忌事项

1. ❌ 把用户当"流量"或"韭菜"收割
2. ❌ 信任崩塌后试图修复（平辈关系不可逆）
3. ❌ 群发促销广告代替个性化服务
4. ❌ 过度消费粉丝善意（饭圈教训）
5. ❌ 外部团队运营本地社群（"本地人做本地事"）
6. ❌ 忽视售后服务这个关系构建的关键出口
7. ❌ 销售技巧过于纯熟（套路化引发不信任，青涩感反而更好）
