# SKILL.md — 用户拉新 (Traction)

## 技能定义

系统化获取新用户的能力。通过"靶心"框架在19种运营渠道中找到核心增长渠道，并用数据驱动的测试方法持续优化，实现可复制的用户增长。

---

## 场景路由 (Scene Routing)

根据当前阶段和目标，直接跳转到对应执行路径：

### 路由A：渠道选择（我要找增长渠道）
**触发条件**：没有核心渠道 / 产品刚上线 / 增长停滞
**跳转至**：`执行框架：靶心` → 步骤1（外环）
**核心动作**：扫19种渠道 → 选3-5种候选 → 进入路由B

### 路由B：渠道测试（我要验证渠道可行性）
**触发条件**：有3-5个候选渠道待验证
**跳转至**：`执行框架：靶心` → 步骤2（中间环）
**核心动作**：低成本测试（≤1000美元/渠道，≤1个月）→ 选出1个核心 → 进入路由C

### 路由C：增长实验（我要优化核心渠道）
**触发条件**：已确定核心渠道，需要持续放大
**跳转至**：`执行框架：靶心` → 步骤3（内环）
**核心动作**：A/B测试 → 策略迭代 → 渠道饱和时回到路由A

### 路由D：增长目标设定（我要定目标和里程碑）
**触发条件**：需要向团队/投资人对齐增长方向
**跳转至**：`增长目标设定`

### 路由决策速查

| 状态 | 路由 | 优先级 |
|------|------|--------|
| 0用户，产品未验证 | 路由A（从直接推销、软广告开始） | ★★★ |
| 有用户但增长慢 | 路由A（完整走靶心流程） | ★★★ |
| 有候选渠道未验证 | 路由B | ★★☆ |
| 核心渠道在增长 | 路由C | ★★☆ |
| 核心渠道已饱和 | 路由A（重走靶心） | ★★★ |
| 需要定目标/对齐 | 路由D | ★☆☆ |

---

## 适用场景

- 产品发布前/后需要获取首批用户
- 增长停滞，需要找到新的增长渠道
- 资源有限，需要找到最高效的获客方式
- 从0到1建立用户增长体系
- 向投资人证明增长势能

---

## 执行框架：靶心 (Bullseye)

### 步骤1：外环 — 头脑风暴所有渠道

**目标**：对19种渠道逐一设想，消除偏见

**操作**：
1. 列出19种渠道的清单
2. 对每种渠道写出一个具体的执行策略
3. 问自己：如果这种渠道成功了，会是什么样子？
4. 记录在电子表格中

**关键原则**：
- 不要跳过任何渠道，即使它"看起来不适合"
- 最有效的渠道往往是利用率最低的
- 了解竞争对手在用什么渠道——然后考虑避开它

**✅ 检查点**：
- [ ] 19种渠道是否全部过了一遍？
- [ ] 每种渠道是否写出了具体执行策略（不是泛泛而谈）？
- [ ] 是否标记了3-5种最有潜力的渠道进入中间环？

---

### 步骤2：中间环 — 低成本快速测试

**目标**：用最少资源验证3-5种最有潜力的渠道

**操作**：
1. 从外环选出最让你兴奋、看起来最有潜力的3-5种渠道
2. 为每种渠道设计一个低成本测试（≤1000美元，≤1个月）
3. 每个测试必须回答：
   - 获客成本(CPA)是多少？
   - 能获得多少用户？
   - 这些用户是目标用户吗？
4. 同时运行多个测试，快速获取数据

**测试示例**：
| 渠道 | 测试方法 | 预算 |
|------|---------|------|
| 软广告 | 联系10家细分平台，争取评测文章 | $0-500 |
| SEM | 在AdWords投4个广告 | $250 |
| 内容营销 | 每周发1篇文章，坚持1个月 | $0 |
| 病毒裂变 | 建立推荐机制，计算病毒系数 | $0 |
| 社交广告 | 在Facebook/Twitter投放2个细分广告 | $200 |
| 直接推销 | 列出20个潜在客户，面对面推销 | $0 |

**注意**：不要在这个阶段深入优化，只验证渠道是否"值得全力投入"

**✅ 检查点**：
- [ ] 是否为每种渠道设定了明确的成功标准（CPA上限、最低用户数）？
- [ ] 测试预算是否控制在总预算的10-20%以内？
- [ ] 是否同时运行而非串行（避免浪费时间）？
- [ ] 测试结果是否用数据而非直觉来判断？

---

### 步骤3：内环 — 全力投入核心渠道

**目标**：压榨核心渠道的所有潜力

**操作**：
1. 选定效果最佳的1种渠道作为核心
2. 将全部精力和资源投入这个渠道
3. 持续测试和优化（每周A/B测试）
4. 探索核心渠道内的新策略和新方法
5. 当渠道饱和时，回到步骤1重走靶心流程

**优化循环**：
```
发现策略 → 小规模测试 → 衡量结果 → 优化或放弃 → 下一个策略
```

**核心渠道可以"借用"其他渠道**：
- 核心是SEO → 用大众媒体获取链接
- 核心是内容营销 → 用社交广告推广最佳内容
- 核心是病毒裂变 → 用推送/平台作为传播载体

**✅ 检查点**：
- [ ] 是否真的只聚焦1种核心渠道（而非2-3种）？
- [ ] 每周是否在做A/B测试？
- [ ] CPA是否在持续下降？
- [ ] 是否在探索核心渠道内的新子策略？
- [ ] 渠道饱和信号出现时（边际CPA上升>30%），是否回到步骤1？

## 增长目标设定

### 制定增长目标

1. 明确最终目标（用户数、收入、市场份额）
2. 拆解为可量化的阶段性里程碑
3. 在日历上标记所有增长相关的活动
4. 确保至少50%的时间用于增长

### 关键的道路

列出实现目标的**必要**步骤（里程碑），排除一切不必要的工作：

```
增长目标 → 必要里程碑1 → 必要里程碑2 → ... → 达成
```

每个里程碑完成后重新评估：路径是否还正确？

**✅ 检查点**：
- [ ] 目标是否量化（具体数字+截止日期）？
- [ ] 里程碑是否可衡量（完成/未完成，非模糊状态）？
- [ ] 是否排除了与增长无关的"忙碌工作"？
- [ ] 50%时间是否真的用在增长上（而非假设）？

## 19种渠道速查

### A. 精准触达
1. **软广告** — 博客/社区投稿、赞助小型平台
2. **大众媒体** — 从媒体链底层向上渗透，与记者建立关系
3. **创意运营** — 制造话题（城市改名、冰块抗议）+ 客户赞赏（手写卡片、礼品）
4. **搜索引擎营销** — AdWords测试产品方向，长尾关键词，质量评分优化

### B. 内容驱动
5. **线上推广** — 社交媒体广告（非直接反应策略），推广已有火花的内容
6. **线下推广** — 广告牌、广播、平面广告，剩余广告空间降低成本
7. **搜索引擎优化** — 大头策略(高竞争高流量) vs 长尾策略(低成本高转化)
8. **内容营销** — 成为行业领导者，信息图分享率是文章的20倍

### C. 用户运营
9. **推送(邮件)** — 生命周期邮件、迷你课程、个性化触发
10. **病毒裂变** — K = i × 转化率，缩短病毒周期，寻找"病毒口袋"
11. **小程序/小工具** — 创建免费工具作为持续获客资产

### D. 合作与分销
12. **商务拓展** — 标准伙伴、合资、特许、分销、供给伙伴关系
13. **直接推销** — SPIN问题模型（情景→困境→影响→解决）
14. **分销/代理** — CJ、ClickBank等网络，按效果付费
15. **现有平台** — 应用商店排名、浏览器扩展、填补社交平台空缺

### E. 线下与社群
16. **会展** — 提前安排会面，走进来+走出去策略
17. **线下活动** — 会议、聚会、派对
18. **演讲** — 从本地小活动开始，录像扩大影响
19. **社群建设** — 任务驱动、质量标准、赋予推销者权利

## 关键指标

| 指标 | 含义 | 目标 |
|------|------|------|
| CPA (每次获取成本) | 获得一个用户的成本 | < 用户终身价值 |
| 转化率 | 访客→用户的比例 | 因渠道而异 |
| 病毒系数 K | 每用户带来的新用户数 | > 1 = 指数增长 |
| 病毒周期 | 完成一次病毒循环的时间 | 越短越好 |
| 质量评分 | 广告相关性和用户体验 | 尽量高 |
| 留存率 | 用户持续使用的比例 | 决定增长是否可持续 |

## 失败模式 (Failure Modes)

| 失败模式 | 表现 | 根因 | 解法 |
|----------|------|------|------|
| **渠道偏见** | 只用熟悉的2-3种渠道 | 舒适区效应 | 强制走靶心外环，19种全过 |
| **并行贪多** | 同时优化5+渠道，都没做好 | 资源分散 | 一次只聚焦1种核心渠道 |
| **测试未完成就加大投入** | CPA很高但已投入大量预算 | 沉没成本谬误 | 设定止损线，测试阶段预算≤20% |
| **指标虚荣** | 追求注册量，忽略留存 | 用错误指标衡量成功 | CPA必须结合留存率看 |
| **产品漏水桶** | 渠道拉来人但留不住 | 产品-市场匹配未验证 | 先确保留存>40%再加大拉新 |
| **渠道疲劳** | 核心渠道效果持续下降 | 策略固化不迭代 | 每周A/B测试，饱和时回到靶心 |
| **测试太保守** | 预算太少/时间太短，得出错误结论 | 害怕失败 | 每个测试至少$200+2周 |
| **忽略"无聊"渠道** | 不考虑BD、会展、分销 | 认为这些"不酷" | 对每种渠道一视同仁走靶心 |

## 反模式 (Anti-Patterns)

### ❌ 反模式1：先做产品，再想增长
**为什么错**：产品做完才发现没有获客渠道，浪费数月
**正确做法**：从第一天起50%时间用于增长，产品和渠道并行验证

### ❌ 反模式2：复制竞争对手的渠道
**为什么错**：竞争对手的渠道可能已经饱和，CPA被抬高
**正确做法**：了解竞品渠道后考虑避开，寻找未被开发的渠道

### ❌ 反模式3：凭直觉选渠道
**为什么错**：创始人的直觉往往有偏见，喜欢技术的选SEO，喜欢社交的选裂变
**正确做法**：用靶心框架强制全面评估，用数据而非直觉决策

### ❌ 反模式4：测试成功后不优化
**为什么错**：渠道测试通过 ≠ 渠道已最大化，还有大量增长空间
**正确做法**：进入内环后持续A/B测试，每周迭代策略

### ❌ 反模式5：只看CPA不看留存
**为什么错**：低CPA但用户秒流失 = 烧钱
**正确做法**：CPA × 留存率才是真正的获客效率指标

### ❌ 反模式6：渠道饱和后死守
**为什么错**：边际成本递增，ROI持续下降
**正确做法**：当CPA上升>30%且无法优化时，果断重走靶心找新渠道

## 决策树

```
你的产品有用户吗？
├── 否 → 从"无法规模化"的方法开始（直接推销、软广告、演讲）
│         同时开发产品，验证产品-市场匹配
├── 是，但增长缓慢
│   ├── 你有明确的增长目标吗？
│   │   ├── 否 → 先设定具体数字目标（路由D）
│   │   └── 是 → 走靶心流程，测试3-5种新渠道（路由A→B）
│   └── 你测试了多少种渠道？
│       ├── < 5种 → 继续测试，消除偏见
│       └── ≥ 5种都没有效果 → 产品可能需要打磨（桶还在漏水）
└── 是，且有核心渠道
    ├── 核心渠道还在增长 → 继续压榨潜力（路由C）
    └── 核心渠道饱和 → 重走靶心流程找下一个核心渠道（路由A）
```

## 执行检查清单 (Master Checklist)

### Phase 1: 靶心外环（1-2天）
- [ ] 19种渠道全部列出
- [ ] 每种渠道写出具体执行策略
- [ ] 标记3-5种最有潜力的渠道

### Phase 2: 靶心中间环（2-4周）
- [ ] 为每种候选渠道设计低成本测试
- [ ] 设定每种渠道的成功标准（CPA上限、最低用户数）
- [ ] 同时运行测试（非串行）
- [ ] 收集数据，用数据选出核心渠道

### Phase 3: 靶心内环（持续）
- [ ] 只聚焦1种核心渠道
- [ ] 每周做A/B测试
- [ ] CPA持续下降
- [ ] 探索核心渠道内新子策略
- [ ] 监控饱和信号（边际CPA上升>30%）

### 持续监控
- [ ] 留存率 > 40%（否则先修产品）
- [ ] CPA < 用户LTV
- [ ] 每月重新评估增长路径
- [ ] 50%时间用于增长
